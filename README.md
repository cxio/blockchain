# 区块链开放式架构设计

## 前言

Bitcoin是区块链世界的开创者，自2009年1月3日启动运行至今，已被世界上大多数国家/地区所认识，其区块链去除第三方信任和去中心化分布式网络的价值已被普遍认可。但即便如此，具体的技术发展也依然在快速迭代和探索之中。

> 区块链当前的共识方案已有几个，主要的公链共识有：创始于Bitcoin的**工作量证明（PoW）**、源于Peercoin的**权益证明（PoS）**，以及Bitshares发明的**委托权益证明（DPoS）** 等。其它大部分共识设计基本上由此改进或组合而来，实质性的变化不多，这里不作赘述。

Bitcoin采用工作量证明（PoW）共识模型。PoW是利用哈希运算结果的随机性，让节点用暴力碰撞来寻求目标难度的哈希值，最早成功的节点即获得区块的铸造权。当满足规定难度的区块被铸造出来后，其它节点会抓紧时间开始新的哈希运算，创建新的区块。

该模型简单有效，但运行至今已逐渐暴露出来一些不足之处：

1. **大量的电力消耗：**<br>
哈希碰撞是一种强制消耗时间但无更多意义的耗能运算，节点靠PoW竞争理论上是一种没有终点的逻辑（虽然存在暂时的某种平衡），长期看会是一个问题。

2. **垄断的中心化可能：**<br>
高能耗促使矿池规模化运营，这可以降低成本，大型矿池的出现是自然的。但这引发了中心化的存疑，目前挖矿芯片的研发和生产就已经接近一种中心化了。

3. **算力波动的影响：**<br>
不同的区块链可能采用相同的挖矿算法，切换算力到不同的区块链可能让矿工获得更大的利益，这会造成算力波动，影响系统的出块时间和可用性。出于任何原因的矿机关停（如矿场事故）如果严重，也会带来相同效果。

4. **缺乏时间确定性：**<br>
区块铸造依赖于缺乏时间确定性的哈希运算，出块时间无法确定，这对挖掘和利用时间因子的价值不利，或许不是一个小损失。

5. **庞大数据负载的高门槛：**<br>
区块数据会持续不断增长，存储和计算的成本也越来越高，全节点的硬件成本也会如此，这对于自由的P2P模型不是一个好消息。这不是PoW的问题，但它依然需要解决。

本设计试图克服以上这些缺点，构建一种 **能源节约、安全性足够、同时也有良好去中心化特性** 的区块链系统架构。成本下降甚至低廉，会带来额外的好处：大规模部署更易行，区块链融入社会的各方面也更容易。


## 概要

### 信用的实体单元

在Bitcoin的技术实现中，一笔交易由输入和输出构成，当前交易的输入引用前一笔交易的输出，其合法性由输出的脚本执行验证。交易承载了比特币的权属和流转，是资产的转移，从某种角度看，这其实是一种价值表达，本质上是一种信用的传递。

人类社会的运行，离不开信用这一介质，它是一个社会最基础的结构性要件，也是一种基本逻辑。它驱动着一个社会的运行，而交易就是这个基本要件的数字载体。


### 由交易驱动的系统

在流行的HTML规范中，Web页面由JavaScript脚本实现与用户的交互，JavaScript采用一种称为事件驱动的交互模型。

区块链是一个信用系统。信用在社会中广泛存在，融入在各种不同的传统体系里。这些体系通常是自完整的，有着自己的逻辑环，对外或许有可信操作的接口，但却不知道区块链为何物，也即它们只是与信用相关而非区块链技术。

因此，我们认为区块链更多的应当是去充当一种信用介质，连接社会沟通各方，而不是一个全能的超级平台。


#### 链信用的独立性

从目前的技术实现上看，区块链的信用是有界限的：只能保证本链上资产的安全，即自保证。

一条区块链很难直接给另一条链提供安全，这实际上可以理解。无法跨链保护不等于信用不可以跨链传递，交易是权属在个体之间的转移，逻辑上是私有的，只要中介环境可靠，辅以必要的约束结构（如过期赎回脚本或法律诉讼），在不同的链间传递信用就是自负其责并可行的。

> 注：交易的双方是具体的实体，物品的权属必然有其受体，因此私有逻辑是基础性的。<br>


#### 中间件的中介层

在两条链之间加入一个自由的「交互者」逻辑层，即中间件，是一种自然而然的方法。

中间件在一条链上注册监听，当监听目标发生变化触发对外行为时，中间件就可以执行自定义的逻辑：可能是在另一条链上创建交易，也可能是驱动一个不知区块链为何物的传统系统，如税务登记、机床启动或门禁开启等。

这种中间件监听驱动的机制就是区块链作用于外部世界的模型：**用户发起交易，交易驱动现实**。监听的事件可能是目标地址的收付款及其确认：初始出现、首次确认、第n次确认，或者脚本执行时脚本指令自身的驱动行为。

因为解耦，中间件可以由任意语言实现，驱动任意的系统。逻辑上，这种驱动是私有的：它们仅需部署在有目标需求的节点上。例如：

- 无人旅店中间件监听收款地址的收款，开启智能门锁（进屋取卡）。
- 税务系统通过预先发布的管理者交易，监听跳转过路信息并记录企业财务支出（假设一种新的监管方式）。
- 版权管理部门的中间件监听版权声明，提取信息检查核实、录入内部系统或触发其它行为。
- 停车场系统中间件监听收款和车位，驱动目标车位开启可用。
- 用户将A链代币兑换为B链代币，注册中间件监听对方行为实现自动化处理。

> 将区块链的信用独立出来作为一种外在的驱动力，是一种开放的架构，这里的区块链只是一种有信用能力的第三方服务而已。
> 或许只有这样，区块链系统才能简单稳定并富于效率。当然，一个特定用途的区块链本身也可以是一个「类传统功能」的系统。


### 公共服务的网络

对于终端用户来说，区块链是一个P2P的服务网络，但现有的区块链系统负载过重，这影响了它们提供服务的能力。实际上，区块链的逻辑可以分解出通用的公共部分：

- **节点发现（findings）**：作为一个基础服务，负责应用节点的登记、缓存、STUN服务。功能简单，独立组网。
- **数据驿站（depots）**：数据传输的中转和停靠，可能实现为一个容器，封装内部微服务的数据请求，实现数据的全网流通和交互。
- **档案存储（archives）**：一个微服务，存储区块中交易携带的附件数据。由数据驿站封装，无需直接的网络能力。
- **区块查询（blockqs）**：一个微服务，分离区块数据的存储负担，提供公共的区块查询。同上由数据驿站封装，自身无需网络能力。

这些逻辑部分被拆解为相对独立的微服务，功能专一稳定（也更简单），服务器之间自由连接，从而形成一个去中心化的公共服务网络。这些公共服务网络是通用的，任何区块链都可以使用它们，这使得它们可以成为区块链P2P世界里的基础设施。

### 概率证明（PoP）共识模型

这是利用交易ID的随机性设计的一个基于**概率证明**（PoP: Proof of Probability）的共识模型，简单并很可能廉价。出于辅助性的价值考虑，也收纳了交易的币权（`币量x币龄`）因子作为二级评选工具。

1. **一级筛选**：交易ID与目标区块哈希的相似性（相位差和）对比，低者胜。
2. **二级筛选**：交易的币权大小（输入中 `币量x币龄` 的合计），高者胜出。
3. **三级筛选**：仅为最终唯一性目的，取一级筛选中的某个哈希值简单比较即可。

这样的设计有如下优点：

1. 与传统的PoS权益证明不同，交易ID只是一个数据哈希，与财富无关，币权仅作为次级因子，这使得富者越富的情况不再严重。
2. 用交易ID实现铸造竞争会鼓励交易，大量的交易费能够保证矿工的收益，这创建了一个正向的良性回馈。
3. 历史交易的数量巨大，如果每区块包含 `64k笔` 交易，则 `20万` 区块长度内竞争者规模可达百亿，51%攻击几乎不可能。

### 小结

宏观上看，我们认为区块链的逻辑不应当是一个平台。平台是大地的内涵，创生一切也主宰一切，毫无疑问是一个中心。

在这个现实世界里，还有流动的空气、热力学的动能、无形的市场驱动力……在我们看来，区块链的信用应当是一种力量、动能，拥有驱动这个社会运行的能力，甚至塑造人类与机器的关系。

最后引用拜占庭将军的问题：将军们现在终于有了一个可靠的信令系统，可以协调一致，去完成那本不可能的任务。
