>
> 未來，人工智能會擁有強大的理性，任何中心化單位都會被說服或腐蝕，然而，如果這超級智能並不通往宇宙的眞理，它就很可能將人類帶往歧途。
>
> 唯有眞正的去中心化……

# 區塊鏈開放式架構設計

## 前言

Bitcoin是區塊鏈世界的開創者，采用工作量證明（PoW）的共識模型。該模型簡單有效，但運行至今已表現出一些不足：

1. **大量的電力消耗：**<br>
哈希碰撞是一種強製消耗時間的運算，大量的電力消耗隻是在證明自己有能力消耗，缺乏更多的意義。

2. **壟斷的中心化可能：**<br>
高能耗促使礦池規模化運營以降低成本，大礦池的出現自然而然，但這可能導致中心化的壟斷。

3. **算力波動的影響：**<br>
不同的區塊鏈可能采用相同的礦機，算力切換或自然災害都會造成算力的波動，這會影響繫統出塊的時間穩定性。

4. **缺乏時間確定性：**<br>
靠碰運氣的哈希運算缺乏時間確定性，因此出塊時間無法準確，這也影響了對時間因子價值的利用。

5. **龐大的數據負載：**<br>
區塊數據會持續增長，存儲和計算的成本也越來越高，在P2P模型這不是一個好消息。它不是PoW的問題，但依然是一個問題。

本設計試圖克服以上這些缺點，創建出一個**能源節約、安全性足夠、同時也有良好去中心化特性**的區塊鏈架構，包括不同繫統之間協作的機製和環境。歡迎參與討論，提出您寶貴的見解！


## 概要

### 信用的實體單元

在Bitcoin中，一筆交易由輸入和輸出構成，輸入引用的是前一筆交易的輸出，其合法性由輸出的腳本實現驗證。交易承載了比特幣的流轉，是資產權屬的轉移，本質上看，這是一種信用的傳遞。

人類社會的運行由信用驅動，而交易就是這個驅動的數字實體。


### 由交易驅動的環境

信用在社會中廣泛存在，融入在各種不同的傳統體繫裏，這些體繫通常是邏輯自完整，對外提供信用的接口。如果我們把交易看作信用的介質，區塊鏈就是一個運行信用的繫統，它與傳統的體繫相互協作，安全可靠的傳遞信用。

在Web中，用戶與頁面的交互由一種稱為**事件驅動**的模型提供。

這裏，用戶發起交易，交易攜帶意圖，通過區塊鏈傳遞到網絡上任何一個局部的繫統裏，或許，這可以稱之為**交易驅動**的模型。


#### 鏈信用的獨立性

一條區塊鏈隻能保證本鏈上資產的安全，這是合理的，事物皆有邊界，這不妨礙信用在不同鏈間的傳遞。


#### 中間件的中介層

交易定義了權屬在個體之間的轉移，獲取信息、構建交易並發送，在邏輯上是個體私有的。因此，從一條鏈上獲取信息創建另一條鏈上的交易並不涉及公共驗證，也卽，鏈間的信用傳遞實際上是自負其責的。私有的環境需要安全，但這與公共繫統無關。

由於邊界的存在，兩條鏈之間的交互就需要一個「交互者」的邏輯層，卽中間件。

具體的工作方式是：中間件在一條鏈上註冊監聽，當監聽目標發生變化觸發行為時，中間件就可以執行任意的自定義邏輯，譬如：

- 無人旅店中間件監聽收款地址的收款，開啓智能門鎖。
- 稅務繫統通過預先發布的管理者交易，監聽跳轉過路信息並記錄企業財務支出（假設為一種新的監管方式）。
- 停車場繫統中間件監聽收款和車位，驅動目標車位開啓。
- 用戶將A鏈代幣兌換為B鏈代幣，註冊中間件監聽對方行為實現自動化處理（參考 [鏈間兌換](examples/鏈間兌換.md) 腳本示例）。

> **附註：**
> 區塊鏈並非隻能用於信用的中介傳遞，它本身也可以實現為一個類傳統功能的繫統。


### 公共服務的網絡

對於終端用戶來說，區塊鏈是一個P2P的服務網絡，但現有的區塊鏈繫統負載過重，這影響了它們提供服務的能力。實際上，區塊鏈的邏輯可以分解出通用的幾個部分：

- **節點發現（findings）**：作為一個基礎服務，負責應用節點的登記、緩存、STUN服務。功能簡單，獨立組網。
- **數據驛站（depots）**：數據傳輸的中轉和停靠，應當是一個容器，封裝內部微服務的請求，實現數據的全網流通。
- **檔案存儲（archives）**：一個微服務，存儲區塊中交易攜帶的附件數據。由數據驛站封裝，無需直接的網絡能力。
- **區塊査詢（blockqs）**：一個微服務，分離區塊數據的存儲負載，提供公共的査詢服務。由數據驛站封裝，簡化實現。

這幾個部分可以實現為獨立的服務器，功能簡單穩定。服務器之間自由連接，就可以形成一個去中心化的公共服務網絡。

### 基於概率的共識證明

交易ID擁有隨機性，我們可以據此設計出一個基於概率的共識模型。它不需要工作量（PoW），因此可能非常廉價。與傳統的權益證明（PoS）不同，交易ID隻是一個哈希值，與財富無關，因此也不存在富者越富的邏輯。

用交易ID參與鑄造競爭在邏輯上可以鼓勵交易，而充足的交易費可以保證礦工的收益，這是一種正向的良性循環。
