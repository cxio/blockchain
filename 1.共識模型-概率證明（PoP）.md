區塊鏈是區塊的集合，區塊包含了當前合法並可納入區塊的不定數量的交易。如果有一筆交易被納入了區塊，就相當於該交易被記賬了。

區塊可以被任意的端點構造出來，其它端點驗證其中的交易，如果完全合法，理論上這一區塊就可以成為區塊鏈上的新區塊。但正因為任意端點都可以創建區塊，所以就需要一種機製來決定：誰的區塊可以進入主鏈，也卽：誰擁有區塊的鑄造權。


## 基於概率的證明（PoP: Proof of Probability）

交易ID是交易數據的哈希摘要，擁有無法預測的隨機性，因此可以把它用作區塊鑄造的評選因子。交易的幣權（幣齡x幣量）也很有價值，因此增加了二級篩選。評選需要最終唯一性，因此還有第三級篩選。

1. **一級篩選**：交易ID與目標區塊哈希的相似性（相位差和）對比，低者勝。註：目標區塊取動態的末端 `-11號` 區塊。
2. **二級篩選**：交易的幣權大小（輸入中 `幣量x幣齡` 的合計），高者勝出。
3. **三級篩選**：僅為最終唯一性目的，取一級篩選中的某個哈希值簡單比較卽可。

這樣的設計有如下優點：

1. 與傳統的PoS權益證明不同，交易ID隻是一個數據哈希，與財富無關，幣權僅作為次級因子，這使得富者越富的情況不再嚴重。
2. 用交易ID實現鑄造競爭會鼓勵交易，大量的交易費能夠保證礦工的收益，這創建了一個正向的良性迴饋。
3. 歷史交易的數量鉅大，如果每區塊包含 `64k筆` 交易，則 `20萬` 區塊長度內競爭者規模可達百億，51%攻擊幾乎不可能。

基本上，這是借助於交易ID和區塊哈希的隨機性來評選鑄造者，因重在隨機性，故名為概率證明。


### 對鑄造者的約束

通常情況下，交易是自然需求的產物，用戶僅在需要實際支付時才會創建交易。但交易可以參與鑄造競爭，這為單純獲取競爭機會就創建交易提供了理由。另外，鑄造是可以委托的（見後），應當讓委托可以失效、現狀可變，還用戶用腳投票的機會。因此設計如下規則：

1. 參與鑄造競爭的交易必須處於區塊鏈末端20萬區塊高度以內，取值範圍：`[-11, -200000]`。負號表示從末端倒數。
2. 交易包含的幣權需要滿足一個最低值（如 `0.1幣時`），這可以提高構建大量小微交易的成本。綜合兼顧，這個值不應太高。
3. 交易需要是有效綁定本鏈的交易，歷史標記（見後）為空或錯誤的交易無權參與鑄造競爭。

為便於描述，這些有資格參與鑄造競爭的交易稱為「鑄憑交易」，由其交易ID演生出來用於最終對比的哈希稱為「鑄憑哈希」。


### 一級篩選：哈希相似性

如果將兩個哈希序列描繪成兩條曲綫，兩條曲綫的相似度稱為哈希相似性。進行對比的兩個哈希一是鑄憑哈希，二是由目標區塊ID（`-11號`）演算而來的「基準哈希」。相似度采用「相位差和」（可想象成縱坐標上的點距之和）計算，小者勝出。算法如下。

> **註：**<br>
> 哈希運算采用 `512位` 算法，相位差和最大可達 `640億`（16x40億），相比於百億級的競爭池規模，這可能是一個合適的值。<br>

鑄憑哈希素材（偽代碼）：

```go
// 鑄造者簽名目標。
var hashData []byte = Hash256( 交易ID + 目標區塊ID )

// 鑄造者簽名數據。
var signData []byte = Sign( hashData )

// 註記：
// 鑄造者需要提供公鈅，因此可以驗證鑄造者地址和此處的數據。
// 簽名算法應當擁有確定性（同一私鈅和消息簽出確定的結果），少量的可變性影響不大。
```

哈希相似性算法（偽代碼）：

```go
// 鑄憑哈希：
// 用鑄造者的簽名數據作為哈希素材，
// 這樣可以隱藏潛在的高權重的未參與者：除非他們自己計算並聲明，否則外部無法知道。
// 卽：攻擊者無法主動尋找他們。
var h1 []byte = Hash512( signData )

// 基準哈希：
// 加入鑄憑哈希（h1）獲得關聯性約束。
// 加入交易ID與前面形成交錯再約束。
var h0 []byte = Hash512( 目標區塊ID + h1 + 交易ID )

// 相位差和計算（哈希空間壓縮）。
// 按4字節分段X坐標，4字節整數值為Y坐標，計算兩條綫Y坐標差距累積。
var sum int64
for n:=0; n<16; n++ {
    i := n * 4
    v1 := Uint32( h1[i : i+4] )  // Y值：大端序轉換
    v0 := Uint32( h0[i : i+4] )  // Y值：...
    sum += | v1-v0 |
}
return sum  // 相位差和
```
返迴的**相位差和**卽是鑄造競爭的一級評選因子，也卽**哈希相似性**的權重。


### 二級篩選：幣權

交易包含的每一筆輸入的幣量乘以幣齡的總和，卽是交易的幣權。它作為第二級篩選因子參與評選，高者勝出。這是在第一級篩選中評出的 `2名` 獲勝者基礎上進行的，因為評選的目標範圍受限極大，所以純PoS中富者越富的問題應該可以忽略。

幣權是一個有價值的篩選因子，它可能促使用戶優化自己交易的幣權分配，比如儘量把古老的餘額花掉（得到高的幣權），以及儘量保留新的收款以存儲幣齡。這對繫統的數據管理以及整體上的運行效率有益。

幣權值的計算可采用「聰秒」精度，降低2名獲勝者幣權相等的概率。


### 三級篩選：最終唯一性

如果二級幣權篩選不足以決出勝負（極為罕見），則簡單地取一級篩選中的 `h0` 哈希序列按字節對比，小者勝出。這隻是一個最終唯一性保證。


### 設計參數

> 鑄憑交易的有效範圍：`[-11 ~ -200,000]號` 區塊。<br>
> 鑄憑交易的幣權最低值限定為 `0.1幣時`（如：0.1個幣存1小時）。<br>
> 哈希相似性對比取 `512位` 哈希長度（64字節），相位值取 `32位` 長度（4字節）。<br>
> 一級哈希相似性篩選取名次 `前2名` 為獲勝者進行二級幣權篩選。<br>
> 二級幣權篩選值大者勝出，幣權值精確到 `「聰秒」` 以儘早分出勝負。<br>
> 最終唯一性篩選很簡單，取一級評選算法中的 `h0` 對比，值小者勝出。<br>



## 鑄造委托

用交易ID作為鑄造評選因子有廣泛的普適性，概率基數龐大，但這要求無專業技能的普通用戶參與進來。顯然，專業技能和時間的花費是一個不小的壁壘。沒有普通用戶的參與，龐大的概率基數沒有意義，因此讓普通用戶委托專業的鑄造者行使鑄造權十分必要。

為此，設計在交易的數據結構中添加了「鑄造委托」的定義。結構為：**鑄造地址(32) + 分成定義(1) + 收益地址(32)**，長度65字節。其中：

- 鑄造地址：定義簽名區塊的主人。這可能是用戶自己的地址，也可能是專業鑄造服務商的地址。
- 分成定義：定義鑄造地址與收益地址的分成比例。
- 收益地址：接收鑄造的收益分成。這使得鑄造可以與收益相分離，也創建了一個外部委托鑄造的機製。可選。

根據分成定義值的不同，收益分成實際上有三種模式：

1. 值 `0`：    收益地址無收益，全部收益支付到鑄造地址。因此收益地址可省略，節省32字節的空間。
2. 值 `255`：  鑄造收益全部支付到收益地址，鑄造者無收入。可實現鑄造與收益的完全分離，卽冷挖礦。
3. 值 `1-254`：按 `n/255` 的比例支付到收益地址，剩餘的支付到鑄造地址。這可實現外部委托鑄造。


### 交易頭

類似於區塊頭代表了區塊數據，這裏設計了交易頭結構。鑄造委托和歷史標記被定義在交易頭中，便於鑄造者資格的驗證。交易中的輸入和輸出序列被分別計算哈希，然後合並計算交易體的哈希，最後嵌入在交易頭裏。交易ID就是交易頭數據的哈希摘要。

交易頭偽代碼（合計 `97+32` 字節）：
```go
TxHeader {
    Version    int32     // 版本
    Timestamp  int64     // 交易時間戳。可設定為未來時間
    BlockClue  [20]byte  // 歷史標記
    Minter     Address   // 鑄造地址，32字節
    Scale      uint8     // 收益地址分成（x/255）
    Staker     Address   // 收益地址，32字節，可選
    HashBody   Hash256   // 交易數據體哈希，由輸入序列和輸出序列合並而來
}
```

#### 附：交易的末端區塊與歷史標記

本設計中無需工作量邏輯，區塊的出塊時間間隔為固定的值（如：6分鈡）。交易的**末端區塊**指交易時間戳所在時間的區塊鏈最新已確認區塊，`高度 = (交易時間戳 - 創始塊時間戳) / 6分鈡`。

在交易中嵌入主鏈上特定歷史區塊的標記，可以實現交易者對主鏈的認可。**歷史標記**綁定的目標就是從交易的末端區塊算起的 `-11號` 區塊，值為目標區塊哈希的前 `20字節`。

> **註：**<br>
> 交易的末端區塊與區塊鏈的末端區塊是兩個不同的概念，後者是一個不斷更新的值。<br>


### 安全性

鑄造委托創建了鑄造市場化運作的機製：不同的錢包服務商彼此競爭，創建專業的服務，獲得用戶的委托。委托是自由的，新的委托不斷加入，過期的委托不斷失效，這形成了流動性，也形成了市場的活力。

服務良好的錢包服務商會擁有大量的用戶委托，這可能帶來壟斷。但委托存在有效期，用戶可以選擇用腳投票，改變新交易的委托目標。同時其它錢包服務商也是一個競爭威脅，因此知名服務商串謀攻擊的可能性應該不高。

> **註：**<br>
> 錢包服務商和接收鑄造委托的服務商其實並沒有必然的連繫，作為一種友好性，錢包服務商其實可以讓用戶指定委托目標。


### 設計參數

> 委托定義在每筆交易裏增加了 `33+32字節` 的開銷。<br>
> 分成比例的不同，創建了3種完全不同的鑄造安全模式：`0`、`255`、`1-254`。<br>
> 委托鑄造讓普通用戶參與鑄造競爭，最大限度拓展了競爭池。<br>
> 區塊出塊時間間隔設計為 `6分鈡`。沒有工作量的不確定性，因此這是一個恆定值。<br>
> 歷史標記取從交易末端區塊算起的 `-11號` 區塊的哈希的前 `20字節` 序列。<br>
> 交易ID為交易頭數據的哈希摘要，長 `32字節`，可能由兩種不同的哈希算法嵌套計算而來。<br>



## 鑄造者的預選與同步

因為哈希相似性對比是針對區塊鏈末端 `-11號` 區塊的哈希，所以一個鑄造者可以提前 `10個` 區塊時段得知對比目標。如果一個節點卽時評估自己的鑄憑交易並廣播，則全網有充足的時間進行預先溝通。儘量多的參與者加入，可以使得儘可能優質的鑄憑交易被發掘出來。


### 擇優池

在鑄造者的預先溝通中，各個節點會收集被廣播出來的鑄憑交易，構造出一個 `100` 容量的擇優緩存池。當一個新的鑄憑交易抵達時，計算它的哈希相似性，如果比池中最差的一個更優，則存儲並轉播，否則忽略。

廣播的鑄憑交易需要被驗證，這由鑄造者對目標區塊（`-11號`）哈希的簽名實現（同時也證明在綫）。當出塊時間到達時，擇優池中位於前端的鑄造者就可以鑄造區塊並廣播了。

擇優池的設計是一個必要的策略，用於發掘鑄造者並有序化競爭，同時也約束全網廣播的區塊數量，避免造成網絡擁堵。

> **註：**<br>
> 鑄憑交易的競爭力包含兩個因素：哈希相似性和幣權，這裏僅需采用哈希相似性因子。<br>
> 如果出現名次並列，簡單地用交易ID排序（小者靠前），定出嚴格的單體順序卽可（確定性）。<br>


#### 附：鑄造者定位

對鑄造者的檢索驗證通過兩個信息實現：

1. 鑄憑交易所在眞實區塊的高度，4字節。這可以簡單判斷出交易是否滿足範圍限定。
2. 交易ID，32字節。用於在區塊中檢索目標交易頭信息，驗證鑄造者定義。


### 避免分區

P2P網絡是自由的，任何潛在的鑄造者都可能離綫，一個實際上擁有最優鑄憑交易的用戶也可能剛剛上綫。如果剛剛上綫的鑄造者鑄造了一個最優區塊，但卻因為處於出塊時間邊界而使得區塊未能廣泛送達，則全網範圍內最新區塊的評選就可能不一致，這會導致分區情況的發生（末端分叉）。或者，一個擁有最優鑄憑交易的攻擊者選擇有意地延遲，也可能造成這種情況發生。

Bitcoin繫統中是用最長鏈作為約束，但這裏是固定的出塊時間間隔，所有分叉（如果有）長度都一樣。因此需要一種機製來約束主鏈的成長、競爭和選擇，這就是下面的擇優池的確定性同步，以及後面的競爭支鏈的縱向評估等規則。


### 擇優池同步

為避免剛上綫的優質競爭者的擾亂，各節點擇優池中的鑄造候選者應當被先期確定下來，而之後的鑄造者就不再被認可。這需要在新區塊創建時間之前對擇優池進行同步。每一個區塊都有此邏輯，針對相應的-11號區塊有着自己的候選者擇優池。

擇優池中的成員需要不斷收集、淘汰和更新，並在適當的時間結束更新並啓動同步。時間規劃如下：

1. 在鑄造對比的目標區塊被創建後，到它成為 `-10號` 區塊前（9個區塊時段，54分鈡）：與它做相似性匹配的鑄憑交易可以被任意廣播和收集。
2. 在目標區塊成為 `-10號` 區塊時，擇優池的收集和更新結束，進入內部候選者的同步階段。該階段為 `1個` 區塊時段（6分鈡），至它成為 `-11號` 區塊時結束。
3. 當目標區塊成為 `-11號` 區塊後，它卽正式成為當前區塊時段將創建區塊的評比參考目標。鑄造者收集、驗證、打包交易，在預定的時間創建區塊並廣播。

> **註：**<br>
> 區塊鏈初始11個區塊的創建沒有-11號區塊作為參考，這是一個特殊區段。它們的創建遵循初創規則。<br>

擇優池的同步由擇優池中的候選者簽名並發起，出於利益無關性考慮，這僅由擇優池中權重排序低於32位的成員執行，卽第33-100名共 `68位` 候選者有權發起同步。這樣設計的原因是：

**在P2P環境下，時間是一個無法準確的值，繫統需要容忍這種模餬性。如果沒有這種限製，高權重的遲到競爭者可以隨時把自己加入擇優池然後啓動同步，因為其高權重很可能最終獲選，於是就會影響到最終的結果，讓預先同步失去了意義。同時，這也會讓同步工作很難正常結束。**

有了這一限製後，一位後來的中低權重的池外競爭者沒有動力去做這樣的事（卽便做了也無多少影響），而池內有權同步的候選者也沒有動力去把那位高權重的後來者補上（註：同步的權利隻有一次）。

擇優池中權重低於32位是一個模餬的概念，因為擇優池本身尚未同步，內部成員的序位並不確定。但這沒有關繫，同步是一個成員補充的邏輯，節點隻需檢査對方的擇優池以及它自己的序位是否合法，然後合並按 `100` 的容量清理卽可：**如果對方集優，合並後肯定合法，如果己方集優，合並後隻是淘汰掉質差的而已，不造成實質影響**。一個同步者隻有一次機會，隨着越多同步的合並，擇優池會越來越穩定。

> #### 註：擇優池同步的公益性質
>
> 擇優池中基本上沒有中選機會的候選者需要負責擇優池的同步工作，是一種類似義工的公益行為（應當讚美）。這是化解分區風險的一種最簡單做法，非此則可能十分復雜。
>
> 如果網絡是暢通的，不同節點的擇優池實際上差別不大，卽便隻有少量候選者（一位）實施了同步，擇優池就有了確定性。因此這種公益要求是可行的。


### 設計參數

> 鑄造者被定義在交易頭裏，對他的定位需要兩個值：目標區塊高度、交易ID所在區塊裏的序位。<br>
> 擇優池容量為 `100名` 候選者。<br>
> 擇優池中權重排序在 `32名` 之後的 `68位` 候選者有權簽名和發起擇優池同步。這是一個利益無關的設計約束。<br>
> 有權同步的候選者僅能發起 `1次` 簽名同步。<br>
> 擇優池同步結束包含2種情況：1. 全部同步者實施完成；2.到達規定的時間點結束。<br>



## 鑄造者的最終評選

當出塊時間將要到達時，理論上擇優池中的全部候選者都有權出塊，但因為篩選的條件很明確，通常隻需要處於前端的候選者出塊就可以了。但實際上，這裏還有一些潛在的隱患。

### 哈希塑造

哈希相似性要求鑄造者用自己掌控的交易ID與目標區塊的哈希進行對比篩選，這裏存在兩種攻擊的可能（假設攻擊者擁有極大的哈希算力）：

1. 攻擊者可以通過在交易裏添加隨機數據來塑造交易的ID，使它與目標區塊的哈希儘量相似（依照算法）。
2. 對於已經能夠創建最優區塊的攻擊者，可以通過變更交易ID來塑造當前區塊的哈希，使得該哈希與自己擁有交易的ID有更好的相似性，從而為自己下一次的競爭獲得優勢。

這就是**哈希塑造**，一種不可接受的作弊行為。如果誘惑力足夠，出現專業的作弊礦機是可以預期的。

**解決辦法：**

對於第一種交易ID的塑造，僅僅適用於末端 `10個` 區塊裏的交易，所以簡單的排除這10個區塊參與鑄造競爭卽可。

對於第二種反向匹配交易ID的**區塊哈希**塑造，雖然它的發生受限於 **隻能是那個已經勝出的區塊的鑄造者**，但如果攻擊者的哈希算力足夠，他就可能從基礎層面獲得優勢，持續勝出甚至鎖定勝負。

因此需要設計更細致的規則來製約：

1. **最低交易數量**：區塊包含的交易數量必須占到可進入區塊總交易數的 `70%` 以上。這可以壓縮攻擊者塑造區塊哈希的可用時間。
2. **動態性和模餬性**：擇優池前 `3位` 不參與鑄造競爭。攻擊者會因此而失去塑造的趨近判斷（不是最近就好），而擇優池是動態的。
3. **確定性的消減**：在一級哈希相似性篩選中增加一個子級篩選：卽時哈希。消減哈希相似性的「勝選確定性」，消磨作弊者的動力（見後）。

> #### 附：一種負向迴饋的可能
>
> 擇優池中前3名被取消競爭資格是一個重要設計，但這可能帶來某種心理上的不悅：最優的權重反而沒有收益。如果這種心理普遍存在，誰都不願把高權重的鑄憑交易發布出來，就會帶來一種負向的遞進迴饋，顯然，這是災難性的。
>
> 保留高權重鑄憑交易不發布並不隱含利益（參考區塊已變），而僅僅是不服氣的心理驅使而已。P2P是個體們的世界，而您是守護者！


### 擇優池憑證

擇優池是一種局部的邏輯，為了可驗證，可用擇優池中 `前3名` 為代表，用它們的定位信息（`3x36=108字節`）作為擇優池憑證。鑄造者和該3位的簽名信息也會包含在 `Coinbase` 交易中，這大概會增加約300字節的數據量。

這樣的設計可以提高攻擊者的門檻：如果攻擊者要構造一條競爭支鏈，除了自身需要有足夠權重的哈希相似性外，還需要對每一個區塊尋找3名權重更高的鑄造候選者為憑證，而這顯然是一個很高的門檻。

> **註：**<br>
> 對Coinbase交易的合法性驗證多了一些工作量，但這些工作量其實在預選時已經完成，故影響不大。<br>
> 與普通交易類似，此處的簽名數據可能並不需要參與計算交易ID，因為古老的鏈段並不考慮分叉攻擊（簽名數據可以刪除）。<br>


### 微調評選規則

1. 擇優池中 `前3名` 不參與鑄造競爭，對於之後的 `97位` 候選者，取當前活躍的 `前4名` 作為初選結果。
2. 這4名初選者按「卽時哈希」進一步篩選，取值最小的 `2位` 為決勝者進入幣權的二級評選，規則同前。

卽時哈希算法偽代碼：

```go
// 當前確定的108字節。
var base = 擇優池憑證

// 活躍的前4名是平等的，更優的哈希相似性沒有意義。
var h1 []byte = Hash256( base + 活躍者1交易ID )
var h2 []byte = Hash256( base + 活躍者2交易ID )
var h3 []byte = Hash256( base + 活躍者3交易ID )
var h4 []byte = Hash256( base + 活躍者4交易ID )

queue := sort(h1, h2, h3, h4)

return queue[0], queue[1] // 選取兩名值小者獲勝
```

> **註：**<br>
> 設計為4位可能是一個恰當的值，更大的值會更多的消磨作弊者的動力，但也會更多地影響到主鏈的競爭力（拉低權重）。

#### 附：設計註記

> 實際上，消減區塊哈希塑造的動力 **動態性和模餬性** 已經足夠。但單純依靠擇優池排序有一個優勝者需要義務保護的瑕疵（低幣權者需要發布區塊來保證高幣權者勝出）。這裏的卽時哈希實際上有一種 **改變勝利者** 的破壞性力量，從人性心理上看，可能比單純的公益保護更能觸動人心。


### 末端活躍區塊

區塊不能收錄未來的交易（按時間戳計算），這是一個基本設計。鑄造者沒有理由等待，新區塊通常會在預定的時間（6分鈡的間隔時間點）開始創建。區塊的創建、廣播和驗證都需要時間，這裏作如下規定：**新區塊的創建和廣播可延長至下一區塊時段內的前 `3分鈡`**，之後節點不再接受新的區塊。

在擇優池中排前的候選者不一定持續活躍，雖然概率不高，但邏輯上這是動態的，因此新區塊並不能在準確的時間點確定。隨着變化，優勝者可能改變，新區塊也會隨之變更，這種動態的變化可稱之為「活躍」。

區塊的廣播抵達和區塊數據的同步完成不是同一個概念，隻要可驗證的區塊證明抵達全網，區塊的數據同步可延續到下一區塊時段完整的6分鈡。考慮可靠性冗餘，同步的區塊可以不止一個，候選的次優區塊也應當伴隨，如果最終發現最優區塊裏有無效交易，次優區塊就可以被考慮了。


### 設計參數

> 為避免交易ID的塑造，末端 `10個` 區塊內的交易不參與鑄造競爭。<br>
> 為壓縮區塊哈希塑造的可用時間，區塊內包含交易的數量必須占到可進入區塊交易總數的 `70%` 以上。<br>
> 擇優池中前 `3名` 候選者不參與鑄造競爭，這種不參與是一種增強的安全措施，其作用體現在三個方面。<br>
> 擇優池中第3名之後 `97名` 候選者參與鑄造競爭，取當前活躍的 `前8名` 進行卽時哈希評選，然後取 `2名` 進入二級幣權競爭。<br>
> 末端活躍區塊有 `3分鈡` 的創建和廣播時間，剩下的3分鈡用於釋放被臨時阻塞的落後錯時交易。<br>
> 區塊數據的同步時間要求稍微寬鬆一些，可以占用下一區塊完整的時段。同步還應當考慮最優區塊失效的冗餘性保障問題。<br>



## 優勢與不足

- 基於歷史交易ID和區塊的哈希隨機性，用概率的方式評選區塊，沒有工作量證明（`PoW`）的能源浪費。
- 次級的幣權篩選因子受到很大的限定，可以認為基本上不存在權益證明（`PoS`，由Peercoin發明）模式下富者越富的問題。
- 自由的直接篩選，不需要委托權益證明（`DPoS`，由BitShares發明） 的選舉流程，更接近自然的去中心化形式。
- 固定的出塊時間間隔使得區塊擁有可預測性，時間因子的價值更容易被開發利用。
- 每一筆交易中需要包含鑄造委托定義和主鏈綁定標記，會多出 `53+32` 字節與交易邏輯無關的額外數據。
