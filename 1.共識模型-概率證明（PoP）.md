區塊鏈是區塊的集合，區塊包含了當前合法並可納入區塊的不定數量的交易。如果有一筆交易被納入了區塊，就相當於該交易被記賬了。

區塊可以被任意的端點構造出來，其它端點驗證其中的交易，如果完全合法，理論上這一區塊就可以成為區塊鏈上的新區塊。但正因為任意端點都可以創建區塊，所以就需要一種機製來決定：誰的區塊可以進入主鏈，也卽：誰擁有區塊的鑄造權。


## 基於概率的證明（PoP: Proof of Probability）

交易ID是交易數據的哈希摘要，擁有無法預測的隨機性，因此我們實際上可以把它用作一種評選因子：評選出哪一位用戶可以鑄造區塊。交易是任何一位使用區塊鏈的用戶都會創建的，擁有普適性，同時它也是一種歷史性數據，無法更改，所以十分地適合這種用途。

另外，與傳統的PoS權益證明不同，用交易ID本身與財富無關，不存在權益證明中富者越富的邏輯。唯一增加獲選機會的方式是創建更多的交易，而這與財富數量的關聯性並不高（較少的財富也可以創建大量的交易）。

同時，用交易ID來評選鑄造權可以鼓勵交易，大量的交易費能夠保證礦工的收益，這是一個正向迴饋的良性邏輯。


### 規則

- 用交易ID與目標區塊的ID進行 **哈希相似性** 計算對比，相似程度越高，則權重越高。目標區塊取區塊鏈末端 `-11號` 區塊。
- 參與評選的交易需位於目標區塊之前以避免哈希塑造，理論上在 `-10號` 區塊之前卽可。但考慮到遲到2天的交易會被收錄，因此簡單的限定應當是 `-(480+10)號` 區塊。另外，合法的範圍應當提供一種淘汰機製：更古老的交易被排除，讓人們可以用腳投票來離開擁有交易主導權的錢包服務商。
- 因此這裏設計為 `-490號` 區塊之前 `10萬` 區塊高度的限製，卽 `[-491 ~ -100,000]` 的範圍（含邊界）。
- 交易頭中有一個 **歷史標記** 用於綁定本鏈，參與評選的交易也需要是綁定本鏈的才行。這一要求源於分叉可合並的設計，避免分叉上的交易擁有鑄造權。

為便於描述，有資格參與鑄造評選的交易可以稱為「鑄憑交易」，由其交易ID衍生計算出來與目標區塊做哈希相似性對比的哈希稱為「鑄憑哈希」，而作為評比參考的 `-11號` 目標區塊下面也可以稱為「評考區塊」。

> **名詞：哈希塑造**
> 通過向源數據中加入隨機無用的數據來塑造其哈希摘要，以使其趨近於想要的結果。這需要大量的計算，會導致類似於PoW的競爭。


### 哈希相似性

如果將兩個數據的哈希字節序列想象成兩條曲綫，兩條曲綫的相似度可稱為哈希相似性。

交易ID與評考區塊ID隻是哈希相似性對比的兩個源數據，最終用於對比的兩條哈希曲綫需要經過計算，加入一些安全性上的約束。


#### 鑄憑哈希

由交易ID衍生計算出來，需要加入卽時的動態性，以及在評考區塊創建時無法預知，但在鑄造評選時已知的信息。這可以避免鑄造者對區塊哈希進行塑造（以趨近於自己已有交易ID）的作弊行為。這樣的信息可以從評考區塊的下一個區塊（**後繼塊**）中提取，此處暫時稱之為 **X信息**。

**X信息** 需要擁有確定性，不能隻是簡單的取用後繼區塊的哈希，否則就隻是把哈希塑造的機會轉給了後繼塊而已。

另外，鑄憑哈希實際上可以用簽名數據計算，這樣就可以隱藏潛在的高權重但未參與競爭的交易ID，使得攻擊者無法主動尋找他們（收集並賄賂他們參與攻擊）。

> **註：**
> 以下及之後的代碼借鑑了Go語言語法，但皆為偽代碼，僅用於表意。

```go
// 簽名源數據
var hashData []byte = Hash256( 交易ID + 後繼塊X信息 )

// 鑄造者簽名
var signData []byte = Sign( hashData )

// 鑄憑哈希（64字節）
var hashUser []byte = Hash512( signData )
```


#### 基準哈希

卽由評考區塊ID衍生計算出來的哈希，用於與用戶的鑄憑哈希進行哈希相似性對比。

```go
// 加入鑄憑哈希產生聯動性約束。
var hashBase []byte = Hash512( 評考區塊ID + hashUser )
```


#### 哈希相似性算法

在哈希序列中按4字節為單位取點，計算兩條哈希曲綫的差值。最終取差值絕對值的和為對比。

```go
// X軸：4字節分段取點
// Y軸：4字節整數取值
// 相位差：兩條曲綫Y坐標差距取絕對值
var sum int64
for n:=0; n<16; n++ {
    i := n * 4
    v0 := Uint32( hashBase[i : i+4] )
    v1 := Uint32( hashUser[i : i+4] )
    sum += | v1-v0 |
}
return sum  // 相位差和
```

返迴的 **相位差和** 卽用於鑄造評選，值小者勝出。


### 最終唯一性

理論上，不同的競爭者可能出現相位差和相等的情形，此時就需要最終唯一性來決出勝負。

這裏設計為簡單地取鑄憑哈希本身 **逐字節** 對比卽可，值小者勝出。


### 設計參數

> 鑄憑交易的有效範圍：`[-491 ~ -100,000]號` 區塊。含邊界區塊本身。<br>
> 哈希相似性對比取 `512位` 哈希長度（64字節），相位值取 `32位` 長度（4字節）。<br>
> 最終唯一性保證采用鑄憑哈希本身（`hashUser`）逐字節對比，值小者勝出。<br>



## 鑄造委托

用交易ID作為鑄造評選因子有廣泛的普適性，概率基數龐大，但這要求無專業技能的普通用戶參與進來。顯然，專業技能和時間的花費是一個不小的壁壘。沒有普通用戶的參與，龐大的概率基數沒有意義，因此讓普通用戶委托專業的鑄造者行使鑄造權十分必要。

為此，在交易的數據結構中添加了「鑄造委托」的定義。結構為：**鑄造地址(20) + 分成定義(1) + 收益地址(20)**，長度41字節。其中：

- 鑄造地址：定義鑄造者，負責相關的簽名邏輯。這可以是用戶自己的地址，也可以是第三方專業鑄造服務商提供的地址。
- 分成定義：定義鑄造地址與收益地址的分成比例。有效值為：`[0 - 199]` 和 `255`，前者為一個範圍。
- 收益地址：接收鑄造的收益分成。如果委托第三方鑄造，這是一個很有用的值，用戶可直接接收到鑄造商的收益支付。

根據分成值的不同，實際上出現了三種模式：

1. 值 `0`：    收益地址無收益，全部收益支付到鑄造地址。此時收益地址可省略，節省32字節的空間。
2. 值 `255`：  鑄造收益全部支付到收益地址，鑄造者無收入。可實現鑄造與收益的完全分離，卽通常所說的 **冷挖礦**。
3. 值 `1-199`：按 `n/200` 的比例支付到收益地址，剩餘的支付到鑄造地址。這就是外部委托鑄造的情形。


### 交易頭

類似於區塊頭抽象了區塊數據，這裏設計了交易頭結構，鑄造委托和歷史標記就被定義在這裏。

交易中的輸入和輸出部分被分別計算哈希摘要，然後兩者合並計算交易體的哈希存入交易頭裏。交易ID就是交易頭數據的哈希摘要。

交易頭結構（`85+20` 字節）：
```go
// 偽代碼
TxHeader {
    Version    int32     // 版本
    Timestamp  int64     // 交易時間戳。設定為未來時間可處於鏈外遊離等待
    BlockClue  [20]byte  // 歷史標記
    Minter     Address   // 鑄造地址，20字節
    Scale      uint8     // 收益地址分成（x/200|255）
    Staker     Address   // 收益地址，20字節，可選
    HashBody   Hash256   // 交易數據體哈希，32字節
}
```

#### 附：交易的末端區塊與歷史標記

本設計中無需工作量邏輯，區塊的出塊時間間隔為固定的值（如：6分鈡）。交易的 **末端區塊** 指交易時間戳時間所在區塊鏈的最新已確認區塊（尚未被收錄，卽將被收錄），`高度 = (交易時間戳 - 創始塊時間戳) / 6分鈡`。

在交易中嵌入主鏈上特定歷史區塊的標記，實現了交易者對主鏈的認可，這是一種安全性設計，用於排除攻擊分叉上的交易的鑄造權利。**歷史標記** 綁定的目標就是從交易的末端區塊算起的 `-11號` 區塊，值為區塊哈希的前 `20字節`。

> **註：**<br>
> 交易的末端區塊與區塊鏈的末端區塊是兩個不同的概念，後者是一個不斷更新的值。<br>
> 因為交易的歷史標記會綁定主鏈，所以實際上就避免了私自挖礦的攻擊，因為交易是公開的。<br>


### 安全性

鑄造委托創建了鑄造市場化運行的機製，不同的錢包服務商彼此競爭，提供專業的服務，獲得用戶的委托。委托是自由的，新的委托不斷加入，過期的委托不斷失效，這就帶來了流動性。用戶可以選擇用腳投票，改變新交易的委托目標，從而形成對大型服務商壟斷市場的一種約束。

實際上，錢包服務商和接收鑄造委托沒有必然的關聯，作為一種友好，錢包服務商甚至可以讓用戶自己指定委托目標（鑄造服務商）。


### 設計參數

> 委托定義在每筆交易裏增加了 `20+1字節` 的固定開銷和 `20字節` 的可選開銷。<br>
> 分成比例的不同，創建了3種完全不同的鑄造安全模式：`0`、`255`、`1-199`。<br>
> 區塊出塊時間間隔設計為 `6分鈡`。沒有工作量的不確定性，因此這是一個恆定值。<br>
> 歷史標記截取相對於交易時間戳末端 `-11號` 區塊的哈希值的前 `20字節` 序列。<br>



## 鑄造者的預選與同步

因為哈希相似性的對比是針對區塊鏈末端 `-11號` 評考區塊，所以一個鑄造者可以提前 `10-1個` 區塊時段得知對比的目標。如果一個節點及時評估並發布自己的鑄憑交易（廣播），則全網有充足的時間進行預先溝通。儘量多的參與者加入，可以使得儘可能優質的鑄憑交易被發掘出來。


### 擇優池

在鑄造候選者的預先溝通中，各個節點會收集廣播出來的鑄憑交易，按權重排序放入一個緩存池，這個緩存池稱為 **擇優池**，容量為 `32`。新的鑄憑交易加入擇優池的過程很簡單：計算它的權重，如果比池中最差的一個更優，則有序插入並轉播，否則忽略。

廣播的鑄憑交易需要被驗證，這由鑄造者對鑄憑哈希的簽名實現（如前）。當出塊時間到達時，擇優池中位於前端的鑄造者就可以鑄造區塊並廣播了。

> **註記：**<br>
> 擇優池的設計是一個必要的策略，可用於發掘鑄造者並有序化競爭，同時也約束全網廣播的區塊數量。


#### 附：交易的理想塊和鑄造者定位

交易的 **理想塊** 指交易的時間戳對應的最早可收錄入區塊（卽時間戳對應該區塊的創建時段），其高度值為交易的 `末端區塊 + 1`。這是一個確定的值，僅僅從交易的時間戳就可以計算而來，通常可借用它來設計交易數據的存儲繫統。

對鑄造者的檢索驗證通過下面的信息實現：

1. 交易ID和其時間戳。用於定位交易在理想塊存儲繫統中的交易頭信息。
2. 鑄造者的公鈅和簽名數據。用於驗證鑄造者地址、簽名和鑄憑哈希的計算。

> **註記：**<br>
> 如果檢索交易頭信息由外部的第三方服務提供，可通過計算交易頭的哈希（等於交易ID）來驗證數據的眞實性。<br>
> 交易所在區塊號（理想塊）的計算簡單采用時間戳卽可，合法範圍排除了遲到交易帶來的安全性隱患問題。<br>


### 避免分區

P2P網絡是自由的，任何潛在的鑄造者都可能離綫，一個實際上擁有最優鑄憑交易的用戶也可能剛剛上綫。如果剛剛上綫的鑄造者鑄造了一個最優區塊，但卻因為處於出塊時間邊界而使得區塊未能廣泛送達，則全網範圍內最新區塊的評選就可能不一致，這會導致分區情況的發生（分叉）。或者，一個擁有最優鑄憑交易的攻擊者選擇有意地延遲，也可能導致這種情況。

Bitcoin繫統中使用最長鏈作為約束，但本設計是固定的出塊時間，所有分叉（如果有）長度都一樣，因此需要一種機製來約束主鏈的成長、競爭和選擇。這就是下面擇優池的確定性同步設計，以及後面競爭支鏈部分的縱向評估規則設計。


### 擇優池同步

擇優池中的成員是動態更新的，為避免剛上綫的優質競爭者帶來的擾亂，各節點的擇優池應當被提前確定下來（獲得確定性），這就需要對擇優池進行同步。

每一個擇優池都是相對於鑄造評比的目標區塊的，在該區塊成為 `-11號` 區塊前的各個時段，有如下規劃：

1. **廣播收集**：目標區塊被創建後，到它成為 `-9號` 區塊前（7個區塊時段，42分鈡），與它做相似性對比的鑄憑交易可以被廣播和收集。
2. **一級同步**：當該區塊成為 `-9號` 區塊後，擇優池的收集和更新結束，進入一級同步階段。此階段為補齊不同節點間擇優池的差距，為1個區塊時段。
3. **二級同步**：當該區塊成為 `-10號` 區塊後，執行二級同步。這是一種確定性同步，由授權的節點發送自身的擇優池信息。用時1個區塊時段。
4. **抵達就位**：當該區塊成為 `-11號` 區塊後，卽成為當前時段待創建新區塊的評比參考目標。

> **註記：**<br>
> 廣播收集階段隻有7個區塊時段，因為實際上需要等待後繼塊出來之後才能計算鑄憑哈希。<br>
> 區塊鏈初始11個區塊的創建沒有-11號區塊作為參考，這是一個特殊區段。它們有專屬的創建規則。<br>


#### 授權節點

對擇優池的同步僅由當前擇優池排序中 `後端24名` 成員發起，這基於利益無關性考慮，原因為：

**在P2P環境下，時間是一個無法準確的值，繫統需要容忍這種模餬性。如果沒有限製，高權重的遲到競爭者可以隨時把自己加入擇優池然後發起同步，因為其高權重，它有可能最終勝選，於是就幹擾到了最終的結果，讓預先同步沒有意義。另外，這也讓同步工作難以正常結束。**

有了這一限製後，排序靠後的有權同步者沒有動力去把遲到的高權重者補入。

> **註：**
> 在兩級同步中，同步者都隻有一次同步的權利。


#### 一級同步

這一階段是為了充分補充不同節點收集到的擇優池成員，儘可能將所有節點的擇優池同化為相同內容。

發起同步的節點是否有權，由接收同步的節點根據自己的擇優池情況判斷：如果對方為自己擇優池中 **後24名** 成員，則認可並合並對方提供的數據，否則忽略。


#### 二級同步

通過一級同步之後，全網並不能確保所有節點的擇優池數據都相同，也卽此時的擇優池並沒有完全可靠的 **確定性**，因此需要二次同步。

與一級同步不同，此時不再有合並擇優池數據的邏輯，而是緩存各個同步者的同步數據，然後比較並選擇一個最優集（相差應當很小）。每個同步者的數據都是獨立的，最優集隻是某個同步者自己的集合，因此此時的擇優池就有了確定性。

授權檢査與一級同步相同，同步者需要是接收節點自己擇優池中的後24名成員之一。最優集衡量可能用集合中全部成員的權重之和來實現。


#### 可靠性

鑄憑交易廣播和擇優池的收集更新時間有8個區塊時段，已足夠長。在經歷了一級同步之後，各個節點的擇優池狀況應當已經十分相似。確定性同步是由24個節點發起，隻要其中有一個是工作並誠實的，就可以讓擇優池擁有正確的確定性。


### 設計參數

> 擇優池容量為 `32名` 候選者。<br>
> 擇優池中權重排序在 `8名` 之後的 `24位` 候選者有權簽名和發起擇優池同步。<br>
> 有權同步的候選者在兩級同步中都隻有 `1次` 同步的權利。<br>



## 區塊鑄造

當出塊時間將要到達時，理論上擇優池中的全部候選者都有權出塊，但因為篩選的條件很明確，通常隻需要處於前端的候選者出塊就可以了。

擇優池是一種臨時性的集合，主要是為了挖掘出潛在的高權重鑄造者。鑄造者的權重證明會被保存在當前區塊的 `Coinbase` 交易中，當出現分叉時，就可以驗證競爭支鏈上的區塊的鑄造者及其權重是否眞實。

正因為擇優池的臨時性，當區塊已經完成，擇優池就消失了。一個攻擊者如果運氣足夠好，他可能能夠找到足夠權重的鑄憑交易來實施攻擊（分叉競爭）。為了降低攻擊者獲得足夠權重的能力，我們可以利用擇優池眞實匯集鑄造者的現實，增加 `2-3位` 高權重者的權重證明，這就是 **擇優憑證**。


### 擇優憑證

區塊鑄造的權重證明包含了如下幾項信息：

- 交易ID和時間戳。
- 鑄造者公鈅。
- 鑄造者對 `hashData` 的簽名（如前 `signData`）。

毫無疑問，應當選取擇優池中前幾名的權重證明來保存。它們已經存在於擇優池中，無需再請求，所以發布者當前是否在綫並無影響。具體的設計是這樣：

- Coinbase交易中會固定保存擇優池 `前2名` 候選者的權重證明，通常他們就是實際的區塊鑄造者。
- 如果前2名候選者都異常離綫不能打包區塊，則按擇優池內的順序，選擇權重最高候選者打包的區塊。此時就需要包含實際鑄造者的權重證明。

另外，這擇優憑證中的參與者的哈希相似性對比的「相位差和」的平均值會作為 **擇優權重** 保存在區塊頭內，用來評估分叉鏈段的競爭力。

> **註：**
> 與普通交易類似，簽名數據可作為附屬數據不參與計算交易ID，因此古老的區塊的簽名數據可以刪除。


### 鑄造者和鑄造冗餘

#### 鑄造者

明確前面鑄憑哈希計算裏的 **X信息**，我們可以選擇存放在區塊頭內的 **擇優權重** 來參與計算，同時再加入評考區塊ID本身來獲得一種交叉關聯，並增加數據的隨機性。

```go
var hashData []byte = Hash256( 交易ID + 後繼塊擇優權重 + 評考區塊ID )
var signData []byte = Sign( hashData )

// 鑄憑哈希
var hashUser []byte = Hash512( signData )
// 基準哈希
var hashBase []byte = Hash512( 評考區塊ID + hashUser )
```

鑄憑哈希與基準哈希的相位差和卽用於權重計算，評估出當前區塊的鑄造者。


#### 鑄造冗餘

區塊的鑄造者通常就是擇優池中的第一名，但網絡世界可能存在任何異常，如果第一名候選者正好離綫了，區塊鑄造應當正常進行。

所以在現實的運行中，作為一種約定，擇優池中的 `前4名` 都會鑄造區塊並發布，為可能的異常留下冗餘保證。如果最終不是擇優池前2名候選者簽發了區塊，則Coinbase交易中還需包含該實際簽發者的權重證明（但不作為擇優憑證）。

區塊不能收錄未來的交易（按時間戳計算，包含出塊時間點），這是一個基本設計，因此鑄造者沒有理由延時等待更多的交易抵達，新區塊通常會在預定的時間開始創建。但區塊的創建、廣播和驗證都需要時間，這裏作如下規定：

**新區塊的創建和廣播可延長至下一區塊時段內的前 `3分鈡`，之後節點不再接受新的區塊。**

區塊的廣播抵達和區塊數據的同步完成不是同一個概念，隻要可驗證的區塊證明抵達全網，區塊的數據同步可延續到下一區塊時段完整的6分鈡。


### 設計參數

> 為了提昇抵抗攻擊的能力，擇優池中 `前2名` 候選者的權重證明被保存在Coinbase交易中。<br>
> 評考區塊之後的後繼塊裏的 `擇優權重` 被用於鑄憑哈希源數據，提供未知性以避免區塊哈希塑造的作弊問題<br>
> 鑄造者按擇優池中的排名決定，但為預防勝選者異常離綫，擇優池中 `前4名` 候選者都會打包簽發區塊。這是一種冗餘性安全保證。<br>
> 為適應網絡可能的擁堵，新區塊的廣播和驗證維持適當寬鬆的約束：下一區塊時段內前 `3分鈡` 都可用<br>
