### 基礎數據結構

這裏的數據結構主要表達成員、類型和存儲，不作為程序代碼的實現。

#### 區塊頭

區塊數據的指引表達，也是區塊哈希的數據源。長度：`90(+32)` 字節。

```go
type BlockHeader struct {
    // 協議版本
    Version     uint32

    // 分支標識
    //   15] 分支位，置1表示分支，置0則後續位忽略。
    // 14-5] 待定。
    //  4-0] 週期數，每週期7天（1680塊），最多支持15個週期。
    Flag        uint16

    // 前一區塊哈希
    // 算法：sha3.Sum256(...)
    PrevBlock   Hash256

    // 前一年塊哈希
    // 從起始0號區塊開始，每87661個區塊（恆星年）記錄一次。
    // 該序列並非每塊都有，而是每年才有一塊有該記錄（多32字節）。
    // 卽：'Height % 87661 == 0' 的塊有該記錄。
    // 主要用於附生側鏈對主鏈的高效引用（年塊頭鏈+年內區塊頭鏈）。
    YearBlock   Hash256

    // 區塊高度
    // 由創始塊的時間戳按固定間隔計算而來。
    // 註記：非時間戳，更簡單和節約。
    Height      uint32

    // 校驗根。
    // 由交易哈希樹根、UTXO指紋、鑄造簽名合並計算而來。
    // 註：類似Bitcoin裏的MerkleRoot。
    CheckRoot   Hash256

    // 鏈段競爭因子，重要性由上至下遞減。
    // 主要用於評估鏈段競爭力，故部分因子可容忍溢出。
    /////////////////////////////////////////////////

    // 相位差和
    // 值越低品質越優，實際應當是一個比較低的值。
    // 理論上值會溢出，溢出時取最大值。
    SumVal      uint32

    // 交易費
    // 銷毀前（如果有）的交易費合計。單位聰（<42.9幣），溢出時取最大值。
    // 不包含分叉合並迴收來的部分。
    // 註記：
    // 約43個幣的封頂額度已經足夠，異常的鉅額交易費也不會造成幹擾。
    // 如果需要精確的統計，應當通過Coinbase交易獲取。
    Fees        uint32

    // 歷史標記數
    // 僅指包含有效歷史標記的交易數。無效標記通常由分叉合並而來。
    // 交易費因子已藴含鏈段競爭力，故這隻是一個輔助競爭因子。更多是一種統計價值。
    TxClues     uint32

    // 幣權銷毀總量
    // 單位幣天，包含分叉合並的部分。
    // 主要用於統計，後期權益金的通脹計算就參考於此。
    Stakes      uint32
}
```


#### 交易頭

交易數據的頭部指引。長度：`93(+32)` 字節。

```go
type TxHeader struct {
    // 版本。
    Version     int32

    // 交易時間戳。
    // 設定為未來的時間戳卽可模擬 LockTime 效果。
    Timestamp   int64

    // 歷史標記。
    // 按時間戳計算，綁定-11號區塊的哈希前16字節。
    // 註：無目標綁定的未來交易此位應當置零。
    BlockClue   [16]byte

    // 鑄造地址，32字節
    Minter      Address

    // 收益地址分成（x/255）
    Scale       uint8

    // 收益地址，32字節，可選
    Staker      Address

    // 交易數據體哈希
    // 由交易的輸入序列和輸出序列分別計算而來。
    // Hash( Hash(TxIns) + Hash(TxOuts) )
    HashBody    Hash256
}
```


#### 交易數據體

```go
type TxBody struct {
    In      []*TxIn     // 無長度限製
    Out     []*TxOut    // 最大長度64k
}
```


#### 交易輸入

```go
type TxIn struct {
    Height  uint32      // 理想塊高度
    TxID    *[32]byte   // 交易ID
    Index   uint16      // 輸出下標
}
```


#### 交易輸出

```go
type TxOut struct {
    Flag        byte        // 輸出標記
    Receiver    []byte      // 接收者
    Amount      uint64      // 幣金數量
    Description []byte      // 憑信描述
    Count       uint32      // 憑信轉移計數
    Title       []byte      // 證據標題
    Content     []byte      // 證據內容
    Attachment  []byte      // 附件（ID）
}
```


#### 鑄幣交易

```go
type Coinbase struct {
    Version   int32         // 交易數據格式版本
    Timestamp uint64        // 交易時間戳。固定地與區塊創建時間相同。
    Minter    [40]byte      // 鑄造憑據。結構：4+4+32（高度, 序位，交易ID）

    // 收益、激勵相關字段
    Coins     uint64        // 收益總額（鑄幣、交易費、權益金、兌奬截留）。
    CoinSelf  [2]*TxOut     // 鑄造收益（鑄造者，分成者）。約束：總額的50%。
    AwardTo   [3]*AwardOut  // 外部奬勵（findings, blockqs, archives）。約束：總額的50%。
    DrawSlot  [90]byte      // 兌奬槽（3*240 = 720位）
}
```



### 區塊存儲 ------------------------------------------------------------------

#### 交易索引

```go
type TxIndex struct {
    ID        *[32]byte  // 交易ID
    Timestamp uint64     // 微秒級時間戳
    Prev      *TxIndex   // 前向指針
    Next      *TxIndex   // 後向指針
}
```


#### 四元鏈段

```go
type List4TH struct {
    Segmer  [4]*TxIndex  // 四個交易索引單元
    Next    *List4TH     // 下一個鏈段指針
}
```


#### 區塊體

存儲交易索引（32+8）的有序序列，首個成員為Coinbase交易索引。

```go
type BlockBody struct {
    Coinbase *TxIndex    // 零號下標成員
    Queue    []*List4TH  // 四元鏈段集
}
```
