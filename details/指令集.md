### 標準模塊指令

模塊成員為標準的名稱，書寫在花括號內。附參通常僅為1個字節，用於索引成員函數。

通常，模塊如果需要創建一個表示自身的對象，由首個成員 Create 實施（而不是一個單獨的函數指令）。
作為約定，如果其它成員需要使用該對象，該對象會作為成員函數的首個參數傳入。

**值區間**：`[512-1023]`, 512個。

```go
指令值  指令字          說明
-----------------------------------------------------------
512     MO_RE{}(1)      正則表達式模塊。
        // 處理正則表達式對象，對象作為首個參數傳入。
        // 成員：
        // -
        // 註：
        // 基礎指令集中的 REGEXP 為創建該對象的指令。

513     MO_TIME{}(1)    關於時間對象的操作。
        // 這是一個單獨的模塊，用於某些特別的需求。
        // 通常情況下，使用繫統內置的腳本級全局時間對象已足夠。
        // 成員：
        // - Create     創建一個時間對象。
        //              實參：UNIX時間戳（微秒）或RFC3339格式字符串。
        // -

514     MO_MATH{}(1)    數學運算模塊。
        // 例：
        // MO_MATH{Abs} 計算實參的絕對值後入棧。
        // 成員：
        // - Abs    計算絕對值，|-4| => 4
        // - Pow    計算冪乘值，x**y
        // - Pow10  計算10的冪乘，10**n
        // ...

515     MO_CRYPT{}(1)   加密運算模塊。
        // 對稱加密：
        // - AES16GCM-ENC   AES-128 + GCM
        // - AES24GCM-ENC   AES-192 + GCM
        // - AES32GCM-ENC   AES-256 + GCM
        //   實參1：key 密鈅，分別對應 16，24，32 字節。
        //   實參2：nonce 隨機數，12 字節。
        //   實參3：plainText 待加密明文。
        //
        // - AES16GCM-DEC   AES-128 + GCM
        // - AES24GCM-DEC   AES-192 + GCM
        // - AES32GCM-DEC   AES-256 + GCM
        //   實參1：key 同上密鈅。
        //   實參2：nonce 同上隨機數。
        //   實參3：cipherText 已加密密文。
        //

...1023 （待定）
```
