### 标准模块指令

模块成员为标准的名称，书写在花括号内。附参通常仅为1个字节，用于索引成员函数。

通常，模块如果需要创建一个表示自身的对象，由首个成员 Create 实施（而不是一个单独的函数指令）。
作为约定，如果其它成员需要使用该对象，该对象会作为成员函数的首个参数传入。

**值区间**：`[512-1023]`, 512个。

```go
指令值  指令字          说明
-----------------------------------------------------------
512     MO_RE{}(1)      正则表达式模块。
        // 处理正则表达式对象，对象作为首个参数传入。
        // 成员：
        // -
        // 注：
        // 基础指令集中的 REGEXP 为创建该对象的指令。

513     MO_TIME{}(1)    关于时间对象的操作。
        // 这是一个单独的模块，用于某些特别的需求。
        // 通常情况下，使用系统内置的脚本级全局时间对象已足够。
        // 成员：
        // - Create     创建一个时间对象。
        //              实参：UNIX时间戳（微秒）或RFC3339格式字符串。
        // -

514     MO_MATH{}(1)    数学运算模块。
        // 例：
        // MO_MATH{Abs} 计算实参的绝对值后入栈。
        // 成员：
        // - Abs    计算绝对值，|-4| => 4
        // - Pow    计算幂乘值，x**y
        // - Pow10  计算10的幂乘，10**n
        // ...

515     MO_CRYPT{}(1)   加密运算模块。
        // 对称加密：
        // - AES16GCM-ENC   AES-128 + GCM
        // - AES24GCM-ENC   AES-192 + GCM
        // - AES32GCM-ENC   AES-256 + GCM
        //   实参1：key 密钥，分别对应 16，24，32 字节。
        //   实参2：nonce 随机数，12 字节。
        //   实参3：plainText 待加密明文。
        //
        // - AES16GCM-DEC   AES-128 + GCM
        // - AES24GCM-DEC   AES-192 + GCM
        // - AES32GCM-DEC   AES-256 + GCM
        //   实参1：key 同上密钥。
        //   实参2：nonce 同上随机数。
        //   实参3：cipherText 已加密密文。
        //

...1023 （待定）
```
