# 模块指令

模块用于引入分门别类的功能板块。

执行模块指令，就会调用其构造方法，创建一个模块对象。然后就可以由通用的 `CALL` 指令调用其开放的方法。

可以向模块构造函数传递实参，就像其它普通指令一样。

> **约定：**
> 模块成员函数返回多个值时，会被封装为一个切片。无自动展开能力。


## 定义

指令码区间：`[225-250]`，26个。


### 基础模块

```go
码值    指令字          说明
-----------------------------------------------------------
225     MO_MATH         数学运算类
        // 汇聚专用于数学运算的高阶操作。
        // 实参：无。
        // 说明：
        // 这是一个模块对象，仅能用 CALL 指令调用。
        // 对象内会定义必要方法，同时这也是约束方法集的方式。
        // 成员：
        // - Abs(x float64) float64
        // - Ceil(x float64) float64
        // - Floor(x float64) float64
        // - Pow(x, y float64) float64
        // - Max(x, y float64) float64
        // - Min(x, y float64) float64
        // - Mod(x, y float64) float64
        // - ....
        // 例：
        // MO_MATH "Abs" {-100} CALL[1]     //=> 100.0
        // MO_MATH "Max" {-1} {99} CALL[2]  //=> 99.0

226     MO_FMT          数据格式化（Go:fmt）
        // 汇聚基本的格式化方法。
        // 主要是对 go.fmt 标准库的局部成员封装。
        // - Sprint(a ...any) string
        // - Sprintf(format string, a ...any) string
        // - Sprintln(a ...any) string

227-249 （未用）
```


### 标准模块包

```go
码值    指令字          说明
-----------------------------------------------------------
250     MO_XX(1){}      标准模块引用
        // 引用目标位置的子模块。
        // 附参：目标索引。
        // 友好：
        // 子模块名称可书写在花括号内。
        // 例：
        // MO_XX{crypto}    引入crypto标准模块
        //
        // 设计：
        // 256个标准模块的空间应已足够，
        // 但如需要更多支持，可使用扩展指令 EXT_MO，空间巨大。
```
