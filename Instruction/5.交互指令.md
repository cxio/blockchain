# 交互指令

用于脚本与外部环境的交互，通过两个各自独立的「缓存区」实现。

缓存区是一个先进先出（FIFO）的队列，从头部取数据，从末尾添加数据。INPUT 和 OUTPUT 的缓存区并不互通。


## 定义

指令码区间：`[46-50]`，5个。

```go
码值    指令字      说明
-----------------------------------------------------------
46      INPUT(1)    导入
        // 从 INPUT 缓存区导入数据（阻塞）。
        // 如果缓存区无值或数量不足，中断退出。
        // 附参：读取的条目数，0值表示全部。
        // 实参：无。
        // 返回值：不定数量。自动展开到目标位置。
        // 例：
        // @INPUT       导入缓存区全部数据展开到实参区。
        // INPUT[3]     导入缓存取头部3项数据，展开入栈。
        // @INPUT NOP   清空缓存区。
        //
        // 设计：
        // 导入失败按正常退出对待，不影响PASS通关状态。
        // 因此前面若已正常通关，此处的中断退出并不影响通过性。
        //
        // 对于公共验证节点来说，这是一种隐式的 END。
        // 因为公共验证节点不会导入任何数据。
        //
        // 对于需要 INPUT 的私有节点来说，应当先注入数据，然后再执行脚本。

47      OUTPUT      导出
        // 把实参区的数据导出到 OUTPUT 缓存区，维持原始顺序。
        // 非阻塞操作。
        // 多次 OUTPUT 会累积数据到缓存区，直到遇到 BUFDUMP。
        // 实参：不定数量。
        // 例：
        // @POPS OUTPUT 数据栈全部内容导出
        // ... OUTPUT   若实参区为空，导出0项数据

48      BUFDUMP(1)  数据转出
        // 将OUTPUT的缓存区内容转出（清空），无阻塞操作。
        // 附参：标识值。
        // 标识由用户定义，可用于标记数据转出点的位置。
        // 接收者可能需要这样的信息。
        // 实参：无。
        // 例：
        // @POPS[3] OUTPUT BUFDUMP
        // 弹出栈顶3项数据到实参区 => 导出到 OUTPUT 缓存区 => 转出清空
        //
        // 注：
        // 外部需要注册处理器。
        // 处理器是第三方私有逻辑，只有需要的节点才用部署。

49      （未用）

50      PRINT       打印（控制台）
        // 输出实参区的内容到控制台。
        // 实参：不定数量。
        // 例：
        // @TOP PRINT       打印栈顶项
        // @POPS[3] PRINT   打印栈顶3项数据
        // 即调用 fmt.Println()，用默认的格式打印。
```


## 附：与 GOTO 的外部脚本的交互问题

INPUT/OUTPUT 两个缓存区不互通，因此不能用于调用者与 GOTO 外部脚本的交互。取而代之的是：

- `Caller ==> GOTO-script`: 通过 GOTO 的实参向外部脚本注入初始栈数据。
- `GOTO-script ==> Caller`: 因为 GOTO-script 事先已知，所以 Caller 需要的数据可以直接集成。GOTO-script 的 PASS/CHECK 结果会直接反映到 Caller 的通关状态上。

> **注：**
> 若用 INPUT/OUTPUT 支持与 GOTO-script 交互，会极大复杂化脚本逻辑。
