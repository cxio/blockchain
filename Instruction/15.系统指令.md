# 系统指令

少量执行系统级功能的指令。至此，指令名已有前缀开头，此为 `SYS_`。


## 定义

指令码区间：`[164-169]`，6个。

```go
码值    指令字          说明
-----------------------------------------------------------
164     SYS_TIME(1){}   全局时间取值
        // 全局时间对象是客户端真实的当前时间。
        // 附参：条目标识。
        // 实参：无。
        // 返回值：目标条目的值。
        // 条目：
        // - 默认           当前时间（Time）
        // - Stamp          时间戳（Unix 毫秒数）
        // - Year           年次（4位）
        // - Month          月次/年（1-12）
        // - YearDay        日次/年（1-365|366）
        // - Day            日次/月（1-31）
        // - WeekDay        日次/周（0-6）
        // - Hour           时数/日（0-23）
        // - Minute         分钟数/时（0-59）
        // - Second         秒数/分钟（0-59）
        // - Millisecond    毫秒数/秒（0-999）
        // - Microsecond    微秒数/秒（0-999999）
        // 注：
        // 返回的数值统一为Int类型。
        // 例：
        // SYS_TIME{Stamp}  获取系统当前的时间戳。
        // SYS_TIME[0]      获取当前时间对象（Time）。
        // SYS_TIME         同上
        //
        // 设计：
        // 节点时间是一个不确定性质的值，不能使用在共识场景。
        // 因此该指令只能在明确的END或隐性END（INPUT）之后才能执行。
        // 这是一个特例。
        // 当前时间在私有场景下是有用的，因此提供。
        //
        // 在END之前使用该指令，脚本会被视为错误而失败。

165     SYS_AWARD(1,4)  兑奖验算
        // 检查兑奖槽情况并输出兑奖比例（0|50|100）。
        // 主要是核查区块高度和兑奖槽确认数，按规则返回可兑奖比例。
        // 附参1：脚本所在输出序位（向内传参）
        // 附参2：区块高度。
        // 实参：无。
        // 返回值：布尔值（是否成功）。
        // 注意：
        // 仅可使用在 Coinbase 交易的输出脚本中，否则脚本会失败。
        // 规则：
        // - 附参1：脚本所在输出序位必须匹配真实位置。
        // - 附参2：区块高度为当前 Coinbase 交易所在区块高度。
        // 此规则在 Coinbase 交易合法性检查时核实。
        //
        // 设计：
        // 该指令会检查附参2（区块高度）之后的链段，包括高度和兑奖槽状况。
        // 该指令会激活系统内币金平衡再计算：
        // - 传参输出序位：检索接收者币金数量。
        // - 传递兑奖比例：计算真实可用金额（金额*比例），向下取整。
        // 友好：
        // 从附参2可以快速推断后续链段是否满足兑奖规则（+31 ~ 48）。
        // 实现：
        // 如果100%兑奖，实际上可以跳过币金平衡再计算。
        // 如果币金平衡没问题（总输出不大于总输入），即便0兑换比例也返回真。
        //
        // 用户：
        // 花费前应当先自行检查可兑换金额，以及区块高度是否合法。
        // 然后构造总输出金额不大于总输入金额的交易。

166     SYS_CHKPASS     系统验证
        // 即内置的地址核实&签名验证，签名包含单签和多签。
        // 本指令也是通关指令（如PASS），验证失败会退出。
        // 注意接收者地址必须是标准格式。
        // 实参：无。
        // 返回值：无。
        // 注意：
        // 币金和凭信遵循不同的验证逻辑，但二者都由本指令启动执行。
        // - 币金：需提取币金数量。
        // - 凭信：一次性消费，脚本验证通过即可。
        // 币金与凭信的区分从输出项配置字节里的（类型说明）识别。
        //
        // 参考：
        // 验证函数声明：
        // SYS_CHKPASS( 接收者, 验证类别, 签名类型, 授权标记, []签名, []公钥, []初级哈希 )
        // 验证类别：
        // 从输出项配置字节的低四位获取：
        // - 1: 币金
        // - 2: 凭信
        // 签名类型：
        // 本实参及后续实参从环境提取而非数据栈。这是一个特例。
        // - 1: 单签
        // - 2: 多签
        // 末尾的“[]初级哈希”仅在多签时有用。
        //
        // 设计：
        // 让解锁数据中的签名相关数据剥离，不参与交易ID计算，可方便后期数据剪枝。
        // 交易的安全性已转由链式哈希来保证（多链耦合）。
        //
        // 作为单独的指令存在，可方便用户灵活指定系统验证启动的位置。
        // 甚至可存在于 GOTO/EMBED/SCRIPT 等指令引入的外部脚本中，没有限制。
        //
        // 另外，单签/多签实参在脚本之外，也强化了隐私性。
        //
        // 实现：
        // 验证成功后，会在系统层标注验证结果（true|false），可供需 SIGNED 指令查询。
        // 会先核实接收者公钥哈希（地址）是否匹配，然后才进行签名验证。

167-168 （未用）

169     SYS_NULL        源码零点标识
        // 在脚本的当前位置设置一个NULL点，作为片段提取时的起点。
        // 主要对 SOURCE 指令有用。
        // 例：
        // SYS_NULL     // 设置起点
        // "Hai"        // 入栈字符串 Hai
        // ...          // 除了 SYS_NULL 的任意指令序列
        // SOURCE[2]    // 提取从 Hai 开始到本指令的源码（包含）
        //
        // 设计：
        // 可用于解锁脚本段，以提供「解锁<=>锁定」的衔接灵活性。
        // 这是一个特例。
```
