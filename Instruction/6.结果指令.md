# 结果指令

本类指令会产生决定性的结果，比如*通关*检查、逻辑*跳转*、函数*返回*或脚本结束等。

> **注：**
> 初始通过状态为true，即：如果不设置通关检查，默认通过。


## 定义

指令码区间：`[51-57]`，7个。

```go
码值    指令字      说明
----------------------------------------------------------
51      PASS        通关（安检）
        // 实参为true时通过，设置通关状态，继续。
        // 实参为false时通关失败，结束脚本。
        // 实参：布尔值。
        // 返回值：无。
        // 提示：
        // 多个PASS类似于“And" 的逻辑，每一个都需成功。
        //
        // 实现：
        // 通关失败可能触发某种通知机制，以便外部了解情况。

52      CHECK       检查（安检）
        // 实参为true时通过，为false时标记失败。
        // 与PASS相同的通关检查，但失败不会退出。
        // 实参：布尔值。
        // 返回值：无。
        // 注意：
        // 通关状态会被后续的CHECK覆盖，因此实际上取决于最后一个CHECK。
        // 这有点类似于“Or”的逻辑（但不同）。
        //
        // 设计：
        // 在正常结束脚本时，系统会检查通关状态来判定是否通过。

53      GOTO(~,48,~)    跳转（独立）
        // 跳转到某个交易的输出脚本。
        // - 实参区数据传递到新脚本作为其数据栈初始内容。
        // - 新脚本延续当前的校验，但有其自身的环境（数据栈、实参区、全局变量区）。
        // - 新脚本中的通关状态（PASS/CHECK）也即是初始顶层脚本的通关状态。
        //   注：跳转到的目标脚本中也可以包含GOTO或EMBED。
        // - 私有环境的残留信息在脚本退出后会全部被丢弃。
        // 附参1：交易年度（按交易时间戳计算）。
        // 附参2：交易ID。
        // 附参3：输出项序位（从0开始）。
        // 实参：不定数量。
        // 返回值：无。
        // 例：
        //      @POPS[5] GOTO[2026, 0xab93cf..., 1]
        // 提取栈顶5项数据，
        // 跳转到2026年的 0xab93cf...交易的第2项输出项的脚本，
        // 传递实参为其数据栈初始内容。
        //
        // 设计：
        // 外部脚本对调用者的回馈仅限于通关状态（如 PASS/CHECK）。
        // 不支持数据栈残留项返回到调用者，这容易带来安全问题（不可控数据注入）。
        //
        // 实现：
        // 本指令需要验证节点实时查询网络以载入目标脚本，
        // 因此考虑效率，支持用户自己提供目标脚本的验证上下文。

54      EMBED(~,48,~)   嵌入（合并）
        // 嵌入某个交易的输出脚本。
        // 目标脚本为合并逻辑，共享当前的脚本域（数据栈、实参区、全局变量区）。
        // 注意目标脚本直接执行，会向当前数据栈中添加内容。
        // 附参1：交易年度（按交易时间戳计算）。
        // 附参2：交易ID。
        // 附参3：输出项序位（从0开始）。
        // 实参：无。
        // 返回值：无。
        // 例：
        //      EMBED[2026, 0xab93cf..., 0] FN_HASH256 DATA{...} ...
        // 提取2026年某交易的首个输出项脚本，嵌入并执行，
        // 之后取栈顶值计算哈希摘要（FN_HASH256）……
        //
        // 实现：
        // 与GOTO类似，支持用户提供目标脚本的验证上下文以提升效率。

55      EXIT        结束脚本
        // 脚本至此被终止执行并退出。
        // 退出前，系统会检查通关状态以判断验证是否通过。
        // 实参：任意单值，可选。
        // 返回值：即实参值。
        // 如果实参区有值，返回该值（到系统，可能会被处理）。
        // 如果实参区无值，则仅简单结束。
        // 注：
        // 本指令通常出现在IF语句块内，但在顶层也可使用。

56      RETURN      函数返回
        // 从函数子块中退出，并返回一个值。
        // 仅用于函数式语句块 MAP{} 和 FILTER{} 中，退出当前迭代。
        // 实参：任意单值，可选。
        // 返回值：即实参值，无实参时返回nil。

57      END         公共验证结束
        // 公共验证逻辑结束，脚本退出。
        // 仅限于公共验证节点，其它私有用途的节点会忽略它。
        // 注：
        // 对公共验证节点来说，该指令类似于EXIT。
        //
        // 例：定向加密：
        // - 交易者A先发布自己的公钥到链上。
        // - 交易者B用A的公钥创建共享密钥，加密数据后连同自己的公钥一并发布。
        // - 交易者A用B的公钥重构共享密钥，解密数据。
        // 解释：
        // 这种数据仅对交易者A有意义，因此需提前结束公共验证。
        //
        // 设计：
        // 私有监听需要借助于开放式的公共发布触达私有监听节点。
        // 私有处理通常需要包含公共验证的逻辑（部分）。
```
