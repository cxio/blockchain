# 环境指令

脚本的运行处在一个逐层嵌套的环境：`系统域 > 交易域 > 校验域 > 脚本域 > 块域`，其中前者是后者的父域。

1. **系统域**。节点的运行时环境。包含节点能提供的一些系统级信息。
2. **交易域**。一笔交易可包含多个输入和输出，以及输入的来源输出部分。
3. **校验域**。当前输出的校验环境，包含 `GOTO` 跳转引入的脚本的验证。
4. **脚本域**。脚本的环境。包含顶层脚本和 `GOTO` 跳转或 `SCRIPT` 引入的子脚本等。
5. **块  域**。即语法块的局部域，包含顶层和嵌套子块。局部域无法互访。

该类指令大多不需要实参，除了设置脚本全局变量的 `SETVAR`。

> **附注：**
> 数据栈和实参空间属于脚本域。脚本域可以嵌套，因此子脚本也有自己的数据栈和实参区。


## 定义

指令码区间：`[128-137]`，10个。

```go
码值    指令字      说明
-----------------------------------------------------------
128     ENV(1){}    环境变量取值
        // 附参：目标标识。
        // 返回值：目标条目的值。
        // 提示：
        // 以下定义的名称并不全面，详情请参考具体实现。
        //
        // 系统域：
        // 即整个区块链环境。
        // - BlockTime  当前区块的时间戳（Unix 时间戳，毫秒）
        //              当前区块：打包交易即将进入的区块，尚未创建。
        // - Height     当前区块高度（与 BlockTime 严格对应）
        //              TipHeight + 1 = Height
        // - TipHeight  区块链末端高度（交易验证时，最新已确认区块号）
        // 例：
        // ENV{Height} {9999} GT PASS
        // 如果当前区块高度已经高于9999，即放行。
        //
        // 交易域：
        // - TxID       当前交易ID
        // - TxTime     当前交易的时间戳（Unix，毫秒）
        //
        // 校验域：
        // 特定于脚本当前的校验。
        // - Passed     通关状态（true/false）
        // - InGoto     是否在 GOTO 的脚本中
        // - InEmbed    是否在 EMBED 的脚本中
        // - InScript   是否在 SCRIPT 引入的脚本中
        // - InCode     是否在 CODE 定义的脚本中
        // - Gotos      截至当前 GOTO 跳转的深度
        // - Source     当前脚本全部源码（含解锁脚本）
        // - Source0    当前锁定脚本源码（不含解锁脚本）
        // - Source1    当前脚本中首个嵌入脚本源码（EMBED，下同）
        // - Source2    当前脚本中第2个嵌入脚本源码
        // - Source3    当前脚本中第3个嵌入脚本源码
        // - ...        以此类推，
        // - Source9    当前脚本中第9个嵌入脚本源码

129     IN(1){}     输入项取值（校验域）
        // 包括输入项的来源相关信息。
        // 站在锁定脚本源的位置，即未来被花费时输入部分的信息。
        // 附参:目标标识。
        // 返回值：目标条目的值。
        // 条目：
        // - Timestamp  输入源交易的时间戳（Unix，毫秒）
        // - TxID       输入源交易ID
        // - Height     输入源交易所在区块的高度
        // - Amount     输入的币金数量
        // - Receiver   输入源接收者（公钥哈希）
        // - Remarks    输入账户关联的附言
        // - Locked     锁定脚本序列
        // - Unlock     解锁脚本序列（不含见证信息）
        // - Source     脚本源码（解锁+锁定，不含见证信息）
        // - Index      本输入在输入集内的序位
        // 以下为凭信特有：
        // - Creator        凭信创建者
        // - Title          凭信标题
        // - Description    凭信描述
        // - Attachment     附件ID
        // 注：
        // Locked/Unlock/Source 可方便模式验证（如果需要）。

130     OUT(~,1){}  输出项取值（交易域）
        // 对当前正在验证交易的输出集。
        // 即锁定脚本可影响未来花费者的转出（如遗嘱——定向约束）。
        // 附参1：输出项序位（起始值0）。
        // 附参2：目标标识。
        // 返回值：目标条目的值。
        // 条目：
        // - Amount         币金数量
        // - Receiver       接收者
        // - Remarks        付款附言
        // - Creator        凭信/存证创建者
        // - Title          凭信/存证标题
        // - Description    凭信描述
        // - Content        存证内容
        // - Attachment     附件ID
        // - Locked         输出脚本（Bytes）
        // 例：
        // OUT{0, Amount}   当前交易首项输出的币金数量

131     INOUT(~,1){}    源输出项取值（交易域）
        // 输入项来源交易的输出集（兄弟条目）成员取值。
        // 附参1：输出项序位（起始值0）。
        // 附参2：目标标识。
        // 返回值：目标条目的值。
        // 条目：
        // - Amount         币金数量
        // - Receiver       接收者
        // - Remarks        付款附言
        // - Creator        凭信/存证创建者
        // - Title          凭信/存证标题
        // - Description    凭信描述
        // - Content        存证内容
        // - Attachment     附件ID
        // - Locked         输出脚本（Bytes）
        //
        // 实现：
        // 如果实时查询网络，会影响节点效率，
        // 因此支持用户自己提供输出项数据以及验证上下文。

132     （未用）

133     XFROM(1){}      获取源交易信息（交易域）
        // 外部脚本对跳转来的当前验证交易的信息检索。
        // 比如GOTO的目标脚本需要获取上级跳转来的源交易中的信息。
        // 附参：目标标识。
        // 返回值：目标条目的值。
        // 条目：
        // - InSize     源交易输入集大小（项数）
        // - InAmount   源交易输入总金额
        // - OutSize    源交易输出集大小（项数）
        // - OutAmount  源交易输出总金额
        // - Account    源脚本账户（公钥哈希）
        // - Amount     源脚本花费币金数量
        // - PayType    源脚本花费类型（币金|凭信|存证）
        // - Locked     源脚本的锁定脚本（Bytes）
        // - Source     源脚本指令序列（Locked 从开头到跳转点）
        // - Timestamp  源交易的创建时间戳
        // 注意：
        // 源交易指花费输出时，当前正在验证的交易，
        // 因此该指令只能在 GOTO 或 EMBED 的目标脚本中使用。

134     SIGNED(1)   签名序位核实（校验域）
        // 检查目标序位的公钥是否参与签名（签名已验证）。
        // 附参：目标序位。
        // 多重签名公钥哈希清单中的序位，从0开始。
        // 如果是单签，只有0值合法。
        // 返回值：布尔值。
        // - 若目标位置公钥已签名且验证合法，返回 `true`。
        // - 若目标位置公钥未参与签名或签名验证非法，返回 `false`。
        // 用途：
        // 区分签名者，然后执行条件分支或判断。主要用于多签场景。
        // 也可用在定制验证场景，检查签名是否已经验证成功：
        // 比如 FN_CHECKSIG 后并不急于 PASS 时，后期可以读取验证结果。
        // 例：
        // SIGNED[1] PASS   // 如果多重签名中第2位已签名，通过。
        // 例：
        // 确认3个参与者中至少有2个签名（另类用法）。
        // SIGNED[0] SIGNED[1] SIGNED[2] SHIFT[3] SOME[2] PASS
        // 提示：
        // 本指令属于校验域，在GOTO到的目标脚本中也可见。
        //
        // 参考：
        // FN_CHECKSIG、FN_MCHECKSIG、SYS_CHKPASS 在验证后，
        // 会在系统层标注状态（true|false），记录签名验证成功与否。
        // 但 SYS_CHKPASS 只会在成功时标注，因为失败会结束脚本。

135     VAR(1)      全局变量取值（脚本域）
        // 附参：目标变量位置 [0-255]。
        // 返回值：目标置位的值。
        // 注意：
        // 与局部域不同，此为变量逻辑，可被赋值覆盖。
        // 例：
        // @VAR[0]  取全局变量集0号位置的值，添加到实参区。
        // VAR[10]  取全局变量集10号位置的值入栈。
        // 提示：
        // 本指令属于脚本域，不跨GOTO。
        // 但EMBED嵌入的脚本共享相同环境，因此会相互影响。

136     SETVAR(1)   全局变量赋值（脚本域）
        // 对当前脚本域的全局变量区设置值，配合 VAR 使用。
        // 附参：变量位置 [0-255]。
        // 实参：任意类型，单值。
        // 返回值：无。
        // 例：
        // {123} SETVAR[1]      将栈顶的值 123 存入全局变量域的1号位置。
        // {1.5} SETVAR[255]    将浮点数 1.5 存入全局变量域的255号位置。
        // $VAR[1]              读取上面存入的 123 添加到当前局部域。

137     SOURCE(1)   脚本源码提取（脚本域）
        // 根据附参标识，获取脚本源码序列的不同片段。
        // 附参高位表达是否跨块域提取。
        // 附参：标识值。
        // 返回值：字节序列（Bytes，副本）。
        // 注意：这里的源码指脚本编译后的指令序列，而不是文本形式的脚本代码。
        // 标识：
        // 以下为块内，前阶提取含 SOURCE，后阶提取不含。
        // 0000 0000    当前块完整代码。（0x00）
        // 0000 0001    当前块前阶已执行代码段。（0x01）
        // 0000 0010    同上，但从最近的 NULL 点开始。（0x02）
        // 0000 0100    后续代码段直到块末尾。（0x04）
        // 以下跨块域：
        // 1000 0000    完整脚本代码，含解锁部分。（0x80, 128）
        // 1000 0001    前阶已执行代码段，含解锁脚本。（0x81, 129）
        // 1000 0010    同上，但从最近的 NULL 点开始。（0x82, 130）
        // 1000 0100    后续代码段直到脚本末尾。（0x84, 132）
        // 提示：
        // - 脚本顶层的块域包含解锁脚本。
        // - 外部脚本中的 SOURCE 指令只针对其脚本自身，与上级引入者无关。
        // - NULL 点由系统指令 SYS_NULL 标记。
        // - 任何情况下，取码时 NULL 点都不会被包含。
        // 例：
        //  DATA{556e6c6f636b}  // 假设为全部解锁数据
        //  SYS_NULL            // NULL.1
        //  ENV{Height} {<Height>}
        //  GT
        //  IF{
        //      SIGNED[1]
        //      SYS_NULL        // NULL.2
        //      PASS
        //      <Source.1>      // 取点.1：示例解说
        //      NOP
        //      EXIT
        //  }
        //  <Source.2>          // 取点.2
        //  SIGNED[0] CHECK END
        //
        // 解说：
        // <Source.1>
        // - SOURCE             // SIGNED ~ EXIT
        // - SOURCE[0]          // 同上，完整块
        // - SOURCE[1]          // SIGNED ~ <Source.1>
        // - SOURCE[2]          // PASS ~ <Source.1> 不含 NULL.2
        // - SOURCE[4]          // NOP ~ EXIT
        // - SOURCE[0x80]       // DATA ~ END 完整脚本序列
        // - SOURCE[0x81]       // DATA ~ <Source.1>
        // - SOURCE[0x82]       // ENV ~ <Source.1> 不含 NULL.1
        // - SOURCE[0x84]       // NOP ~ END
        //
        // <Source.2>
        // - SOURCE[0]          // DATA ~ END 完整脚本序列，块逻辑
        // - SOURCE[1]          // DATA ~ <Source.2>
        // - SOURCE[2]          // ENV ~ <Source.2>
        // - SOURCE[0x82]       // PASS ~ <Source.2> 块被解剖，取局部
        //
        // 设计：
        // 不支持以 NULL 点为终点的后阶未执行代码提取，
        // 因为需要额外的提前扫描。
        //
        // 实现：
        // 应当有两个 NULL 点游标：
        // - 块域局部：存在于每一个块域内，块结束即丢弃。
        // - 脚本全局：从解锁脚本头部开始，记录并更新NULL点位置。
```


## 实现参考

重要的环境信息应当对中间件开放，让它们可以感知系统状态，获取确定的值。如：`ENV{}`、`IN{}`、`OUT{}`、`INOUT{}` 中部分的成员值信息。
