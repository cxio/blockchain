# 脚本指令集-基本约束

脚本在执行过程中是只读的，脚本的源码本身不可被修改！
因此本基础指令集中，没有可修改原始脚本内容的指令，未来的指令定义也需遵循这一原则。


## 概念澄清

- **数据栈**：存放脚本执行过程中产生的各种数据的栈结构。
- **实参区**：用于存放指令调用时所需的实参的区域，但可能为空（无值）。
- **局部域**：存放当前语法块局部私有的值，一个静态堆叠空间。无上下级继承关系。
- **全局域**：整个脚本执行环境的共享空间，存放自定义的全局变量值。
- **指令码**：每个指令的唯一标识值，仅用1字节表达。
- **附参**：指令的硬参数（编译时固化），通常用于表达指令本身的属性。
- **关联数据**：指令携带的数据，也简称**数据**。主要用于表达较大的数据块，如字符串或字节序列。
- **实参**：指令调用时需要传入的运行时值，可能在实参区（否则从数据栈中获取）。


### 脚本行为

- **结束**：脚本执行退出，PASS通关状态决定验证是否通过。
- **失败**：会让通关检查的PASS无效，验证不通过。错误会导致脚本失败。
- **中断退出**：与脚本的正常结束稍有差别，但逻辑类似（不影响PASS通关状态）。主要适用 `INPUT` 无数据时。


## 指令的结构

每个指令都由 `指令码+附参+关联数据` 构成，其中除了指令码是必需的，其它都视指令不同而可选。

实参是指令执行时产生的值（提取或指令调用返回），是运行时的值，不属于指令本身。


## 指令说明的格式

指令的说明被放在一个代码块内（go），但并非Go代码。而是为了高亮友好展示，便于阅读。

参考示例：

```go
码值    指令字      说明
-----------------------------------------------------------
0       NIL         空：nil
        // 通用的nil项，表示无值。
        // 指令码本身即表达该值（无需附参和数据）。

10      DATA{}(~)   数据：[]byte
        // 类型：Bytes
        // 附参：数据的长度（字节数），变长整数。
        // 数据：十六进制表达（无前置0x）。
        // 支持明确指定长度的写法，多出的空位用零值填充。
        // 例：
        // DATA{46af3fb4}       自动计算为4字节数据。
        // DATA[8]{46af3fb4}    明确8字节长，后续空位默认0值。
        // DATA{414243}         内容实为 "ABC"，3字节16进制表示。
        // 提示：
        // `DATA{414243}` 编译后的值序列为：
        // 0x0A 0x03 0x41 0x42 0x43
        // 其中：
        // - 0x0A 表示指令码（10）。
        // - 0x03 附参，表示数据长度为3字节。
        // - 后续的 0x41、0x42、0x43 为实际数据内容。

16      CODE{}(~)   代码/指令序列
        // 类型：Script
        // 附参：变长整数，指定后续代码块的字节数（varint）。
        // 数据：指令序列，包含在 {} 内，长度由附参指定。
        // 例：
        // CODE{
        //      TOP FN_HASH256 DATA{<hashResult>} EQUAL PASS
        // }
        // 花括号内为指令序列（正常书写格式）。
        //
        // 提示：
        // 这一段指令码编译后的值序列为：
        // - 首字节：0x10 表示 CODE 的指令码（16）。
        // - 后续跟随：附参，一变长整数，指定代码块序列的长度。
        // - 附参之后：即代码块的指令序列（已编译）。

18      VALUE(1){}      获取某值
        // 类型：Any
        // 通用的取值指令。
        // 附参：值索引，占1字节。
        // 可用于某些环境信息（与 ENV 有重叠）或其它信息的获取。
        // 支持方括号用值索引（附参用法，如 VALUE[0]），
        // 也可以用花括号引用值的名称，但名称只是值的别名。
        // 如：VALUE{someName}，someName 只是 VALUE[0] 的别名。
        // 提示：
        // 花括号紧跟在字母名称之后才是表示指令有关联数据。
        // 此处是 VALUE(1){} 而非 VALUE{}(1)，花括号仅表示可使用别名。

53      GOTO(~,48,~)    跳转（独立）
        // 附参1：交易年度（按交易时间戳计算）。
        // 附参2：交易ID。48字节长。
        // 附参3：输出项序位（从0开始），变长整数表达。
        // 例：
        // GOTO[2026, 0xabcdef..., 1]
        // 引用2026年度，ID为 0xabcdef... 的交易的第2个输出项的脚本。
        // 该脚本将在一个独立子环境中执行（继承部分调用环境）。
        //
        // 提示：多个附参依次列出，用逗号分隔。
```

每条指令的说明包含以下部分：

- **指令码**：固定占用1字节，唯一标识该指令。
- **附参**：可选。由小括号和其中的数字表达，数字表示附参所占字节数（`~` 表示变长整数）。指令可以有多个附参，以逗号分隔依次列出。
- **关联数据**：简称「数据」，可选。若指令有此数据，会在说明栏说明。携带数据的指令会跟随一对花括号（`{}`），表示有内容。
- **实参**：可选。指令无实参时则无需说明。
- **说明**：指令的详细说明被放在指令行的下面（悬挂缩进），以注释形式（`//` 开头）书写。

> **提示：**
> 如果附参需要引用值名称，指令名后通常也会包含 `(n){}`，小括号表达附参字节数（`n`）。
