# 函数指令

在语言基本语法逻辑之外，看着像函数的指令。命名包含前缀（`FN_`），与基本语法指令有所区分。

大部分函数指令没有附参，由动态的实参提供信息输入。


## 定义

本函数集包含两个部分：*基础函数集*和*标准函数集*（单指令扩展包）。

指令码区间：`[170-224]`，55个。


### 基础函数集

数量不多的一些常用的基础性函数。

> **注：**
> 这里的书写已略作调整，按函数定义的形式表达。实参可从括号内的声明体现出来。

```go
码值    函数名（参数序列） => 返回值类型
-----------------------------------------------------------
170     FN_BASE58( data Bytes|String ) => String|Bytes
        // 将目标字节序列为编码为 Base58 字符串，或反向解码。
        // 实参：字节序列或字符串。
        // 返回值：字符串或字节序列。
        // 提示：
        // 执行解码或编码，视实参类型而定。

171     FN_BASE32( data Bytes|String ) => String|Bytes
        // 将目标字节序列编码为 Base32 字符串，或反向解码。
        // 实参：字节序列或字符串。
        // 返回值：字符串或字节序列。
        // 规范：
        // RFC4648：大写 + 234567。
        // 提示：
        // 编码或解码时实参类型而定，字符串没有填充字符。

172     FN_BASE64( data Bytes|String ) => String|Bytes
        // 将目标字节序列编码为 Base64 字符串，或反向解码。
        // 实参：字节序列或字符串。
        // 返回值：字符串或字节序列。
        // 提示：
        // 编码或解码由实参类型自动决定。
        // 无填充字符，增补字符采用URL友好类型（-_）。

173     FN_ADDRESS( data Bytes|String, prefix String ) => String|Bytes
        // 将公钥哈希编码为账户地址（Base58），或反向解码。
        // 实参1：公钥哈希或账户地址。
        // 实参2：识别前缀。
        // 返回值：账户地址或公钥哈希。
        // 提示：
        // 编码或解码由实参类型自动判断。
        // 文本形式的账户地址主要用于UI展示和视觉识别。
        //
        // 编码流程（输入Bytes → 输出String）：
        // 0. 公钥哈希添加识别前缀：[前缀 || 公钥哈希]。
        // 1. 对结果执行2次哈希运算，取末尾4字节为校验码。
        // 2. 公钥哈希附上校验码：[公钥哈希 || 校验码]。
        // 3. 将结果编码为文本，附上前缀：[前缀 || 编码文本] >> 账户地址
        //
        // 解码流程（输入String → 输出Bytes）：
        // 0. 分离识别前缀（长度已知）和编码文本。
        // 1. 将编码文本解码为字节序列并切分：[公钥哈希 || 校验码.1]。
        // 2. 公钥哈希前置识别前缀：[前缀 || 公钥哈希]，
        // 3. 对结果执行2次哈希运算，取末尾4字节为[校验码.2]。
        // 4. 两个校验码对比，相同即为合法，返回公钥哈希。
        //
        // 参考：
        // 文本形式的账户地址仅用于人类可读展示，
        // 以及当用户构造交易解码地址时，验证输入的的地址是否合法。

174     FN_PUBHASH( data Bytes ) => Bytes
        // 从公钥创建单签名公钥哈希。
        // 实参：公钥（字节序列）。
        // 返回值：公钥哈希。
        // 提示：
        // 这是单纯的编码（哈希）操作，从公钥生成对应的公钥哈希。
        // 若需将账户地址解码为公钥哈希，请使用 FN_ADDRESS 函数的反向操作。

175     FN_MPUBHASH( baseHash []Bytes, m, n byte ) => Bytes
        // 创建多重签名的总公钥哈希。
        // 实参1：所有参与者公钥的初级哈希（BLAKE3-256）。
        // 实参2：m/N 配比中的 m。
        // 实参3：m/N 配比中的 N（m ≤ N）。
        // 返回值：复合公钥哈希。
        // 规则：
        // - 各公钥哈希有序串连（确定性）。
        // - 前置 `m/N` 配比（共2字节）。
        // - 对串连的字节序列执行哈希运算，结果即为复合哈希。
        // 提示：
        // 与 FN_PUBHASH 稍有不同，此处传递的是各个公钥的初级哈希。

176     FN_CHECKSIG( chk_type, auth_flag byte, sig, pubKey Bytes ) => Bool
        // 单签名验证。
        // 实参1：验证类型：1-币金；2-凭信。
        // 实参2：授权标识，配置签名消息的约束。
        // 实参3：签名数据，可由解锁脚本作为普通数据提供。
        // 实参4：签名用公钥。
        // 返回值：成功与否。
        // 实现：
        // 验证成功后，会在系统层标注验证结果（true|false）。
        //
        // 设计：
        // 验证类别（chk_type）仅分辨两种情况：币金花费、凭信转移。
        // 因此该 byte 类型实参的高位可能用于未来其它用途。

177     FN_MCHECKSIG( chk_type, auth_flag byte, sigs, pubKeys, baseHashs []Bytes ) => Bool
        // 多重签名验证。
        // 实参1：验证类型：1-币金；2-凭信。
        // 实参2：授权标识，配置签名消息的约束。
        // 实参3：签名数据集，与公钥集成员一一对应。
        // 实参4：签名公钥集。
        // 实参5：剩余公钥的初级哈希集。
        // 返回值：成功与否。
        // 提示：
        // 验证方式请查看上级目录下「附.交易.md」内多重签名章节。
        // 签名数据可由解锁脚本作为普通数据提供，不依赖见证信息。
        // 实现：
        // 验证成功后，会在系统层标注验证结果（true|false）。
        //
        // 设计：考虑同上 FN_CHECKSIG 指令说明。

178     FN_HASH224(1)( data Bytes ) => Bytes
        // 对实参数据执行哈希运算，224位长。
        // 附参：算法标识。
        // - 0: BLAKE3，默认
        // - 1: BLAKE2，实为 blake2b 算法
        // - 2: SHA2
        // - 3: SHA3
        // 如用 {name} 表达，算法名称用小写形式。下同。
        // 实参：任意字节序列。
        // 返回值：字节序列（28字节）。
        //
        // 设计：
        // 支持算法标识可能是一个必要的机制，更灵活。

179     FN_HASH256(1)( data Bytes ) => Bytes
        // 256位哈希运算。
        // 附参：算法标识。
        // - 0: BLAKE3，默认
        // - 1: BLAKE2
        // - 2: SHA2
        // - 3: SHA3
        // 实参：任意字节序列。
        // 返回值：字节序列（32字节）。
        // 例：
        // DATA{...} FN_HASH256{blake3} // 采用 blake3.Sum256(...)
        // DATA{...} FN_HASH256[0]      // 同上，默认算法
        // DATA{...} FN_HASH256         // 同上，极简表达
        //
        // 参考：
        // blake2b 在 64位 x86 和 ARM 架构上比 SHA2/SHA3 快。

180     FN_HASH384(1)( data Bytes ) => Bytes
        // 384位哈希运算。
        // 附参：算法标识。
        // - 0: BLAKE3，默认
        // - 1: BLAKE2
        // - 2: SHA2
        // - 3: SHA3
        // 实参：任意字节序列。
        // 返回值：字节序列（48字节）。

181     FN_HASH512(1)( data Bytes ) => Bytes
        // 512位哈希运算。
        // 附参：算法标识。
        // - 0: BLAKE3，默认
        // - 1: BLAKE2
        // - 2: SHA2
        // - 3: SHA3
        // 实参：任意字节序列。
        // 返回值：字节序列（64字节）。

182-222 （未用）

223     FN_PRINTF( fmt string, val ...any ) => nil
        // 向控制台打印消息，即 fmt.Printf。
        // 实参1：格式字符串。
        // 实参n：不定数量目标变量。
        // 提示：
        // 注意与标准打印指令 PRINT 有所不同。
```


### 标准函数集

```go
码值    指令字          说明
-----------------------------------------------------------
224     FN_X(1){}       标准函数引用
        // 调用目标位置的标准函数。
        // 附参：目标索引。
        // 实参：……
        // 实参视具体情况而不同，由目标指令及其配置决定。
        // 友好：
        // 花括号内可用目标名称书写（附参的别名）。
        //
        // 设计：
        // 标准函数集仅支持1字节空间，最多容纳256个函数指令。
        // 它们与普通的函数指令一样，需要配置参数数量，直接调用。
        //
        // 这通常仅由官方定义和发布。
```
