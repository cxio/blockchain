## 基础指令集

> **注**：下面的代码并非Go语言代码，仅为显示友好。


### 1. 值指令

在脚本中作为一个实际的值存在，自动入栈。通常称它们为实体值。
注：显示时省略中括号包围的附参值。

```go
指令值  指令字      说明
-----------------------------------------------------------
0       NIL         空值
        // 即Go语言中的 nil 值。

1       TRUE        真值（True）
2       FALSE       假值（False）
        // 布尔值，Bool 类型。

3       {}(1)       字节整数（byte/uint8），1字节：[0, 255]
4       {}(2)       小整数（int16），2字节：[-0x8000, 0x7fff]
5       {}(4)       中整数（int32），4字节：[-0x80000000, 0x7fffffff]
6       {}(~)       变长整数，Varint：[-1<<63, 1<<63-1]
        // 整数，Int 类型（int64）。
        // 附参值即为指令的关联数据，应当在{}内表达。
        // 从脚本的文本解析为指令序列时，值的大小自动决定指令的值（3-6）。
        // 值存储采用大端序结构（big-endian unsigned integer）。
        // 例：
        // {12756} 入栈正整数 12756，指令值应该为4（2字节）。
        // 注：
        // 4字节的中整数和变长整数有一些重合，优先采用4字节记录。
        // 内部运算时可能会统一转换到int64类型。

7       {}(1+n)     大整数（BigInt），变长字节。
        // 首字节标记实际数据占用的字节数，结构类似后面的 DATA{}(1)。
        // 值存储采用大端序（big-endian）结构，同上。
        // 注：
        // Go语言中即为math/big包里的Int类型。

8       {}(4)       浮点数（4字节，float32）
9       {}(8)       浮点数（8字节，float64）
        // 浮点数，Float 类型。
        // 附参即为关联的数据值，表达在{}内。
        // 作为文本显示时必须包含小数点，支持科学记数法。
        // 从脚本文本解析到指令序列时，值的大小自动决定指令值（8-9）。
        // 例：
        // {3.14159} 入栈浮点数 3.14159，指令值为8。
        // {2.1606e108} 超出float32容量，指令值为9。
        // 注：
        // 内部计算时会统一转换到 float64 类型。

10      NL          换行符。可用于批量解锁参数的分隔。
        // 注记：
        // 非直接查找方式切分，数据仍需解析。
        // 因为解析可跳过指令关联数据的检查，所以效率也不低。

11      DATA{}(1)   字节序列数据。后续1字节定义再之后的数据长度。
12      DATA{}(2)   字节序列数据。后续2字节定义再之后的数据长度。
        // 序列，Bytes 类型。
        // DATA{46af3fb481837fadbb421727f9959c2d} 入栈16个字节，视为16进制。
        // DATA{}(2) 指令最大可支持64kb数据，但目前上限为8kb。
        // 从脚本文本串解析为指令序列时，值的长度自动决定指令值（11-12）。
        // 注：
        // 类似Go语言中的[]byte类型，但支持相等比较。

13      TEXT{}(1)   文本数据。1字节定义再之后文本的字节数，UTF-8编码。
14      TEXT{}(2)   文本数据。与上类似，但由2字节定义长度。
        // 文本，String 类型。
        // 例：
        // TEXT{Hello} 入栈 Hello 文本串。
        // 从脚本文本串解析为指令序列时，文本字节长度自动决定指令值（13-14）。
        // 注：
        // 类似Go语言中的string类型，编码为UTF-8。

15      ~~~         （系统保留）
```


### 2. 交互指令

在脚本的执行环境中，存在一个数据的「缓存区」结构，它是与外部发生联系的一个中介区域。缓存区的数据为移动逻辑（即只有一份），如果发生了读取，数据就是被取走而不是被复制。对缓存区的写入和读取是非阻塞的，不影响脚本执行流。

导出和导入的缓存区是两个各自独立的区域，脚本中并不能先导出数据寄存，之后再导入同样的数据使用。

```go
指令值  指令字      说明
-----------------------------------------------------------
16      INPUT(1)    导入缓存区数据。
        // 返回：[]interface{}
        // 需配合取值指令使用，否则只是移除了缓存区条目。
        // 附参：读取的条目数，0值表示全部（显示可省略）。
        // 例：
        // @INPUT 导入缓存区全部数据展开到当前位置（实参空间）。
        // $INPUT 导入缓存区全部数据展开添加到当前局部变量域。
        // @INPUT[10] PUSH 将缓存区最后10项数据取出展开后入栈。

17      OUTPUT      导出实参序列到缓存区。
        // 不定参数指令。
        // 例：
        // @TOPS[4] OUTPUT 引用栈顶4项数据导出到缓存区。

18      BUFDUMP     缓存区数据转储。外部需注册回调以接收转储的数据。
        // 说明：
        // 回调无需在所有的节点中执行，这是一个私有逻辑。
        // 有需求的交易者应该已经在节点机上预作了部署，此可触发业务逻辑。
        //
        // 转储时包含一个前置的状态值：
        //  0   尚未执行通关指令（PASS, MOVE）
        //  1   通关成功（脚本继续或正常结束）
        // -1   通关失败（脚本终止，仅由系统自动调用）
        //
        // 注记：
        // 系统内置的默认签名验证包含了通关逻辑（PASS）及其状态。
        // 内置验证失败后转储的数据为nil（因为缓存区尚未初始化）。

19      ~~~         （系统保留）

20      PRINT(1)    控制台打印。多个实参条目用换行分隔。
        // 不定参数指令。
        // 附参：标记打印的字符串格式。
        //      0   => 智能格式，默认
        //      2   => 二进制
        //      8   => 八进制
        //      10  => 十进制
        //      16  => 十六进制
        // 例：
        // @TOPS[3] PRINT[16] 向控制台打印栈顶3项数据的十六进制表示。
        // @POP PRINT 弹出栈顶项并用智能格式打印（0可省略）。
```


### 3. 栈操作指令

对脚本栈中的条目执行简单的操作，如：入栈、弹出、引用（待复制）等。通常配合取值指令 `@` 和 `$` 使用（见后）。

```go
指令值  指令字      说明
-----------------------------------------------------------
21      NOP         无操作（占位）。
22      PUSH        数据入栈。
        // 不定参数指令。
        // 把前面的实参序列按顺序压入栈顶。无实参时无效果。
        // 例：
        //      $[0] $[1] $[2] PUSH
        // 引用局部变量域中前3个值为实参，复制压入栈顶。

23      POP         弹出栈顶条目。
        // 弹出的数据如果没有前置取值指令，则为丢弃，下同。
        // @POP 取值到当前位置（实参）
        // $POP 取值并添加到局部变量域
24      POPS(1)     弹出栈顶端多个条目。
        // 返回：[]interface{}
        // 一次性取出，保持栈内顺序。
        // 附参：弹出的条目数，0值表示全部（显示可省略）。
        // 例：
        // $POPS[10] 弹出栈顶10个条目添加到局部变量域。
        // POPS[10]  单纯移除栈顶端10个条目。
        // POPS      简单移除栈内全部条目（同 POPS[0]).

// 以下为目标引用。
// 需前置取值指令（@, $）使用，否则无效果。
25      TOP         引用栈顶项。
        // 引用栈顶项但并不取出。下同。
        // 例：
        // @TOP PUSH 与 DUP 效果相同。
26      TOPS(1)     引用栈顶端多个条目，顺序保持。
        // 返回：[]interface{}
        // 附参：引用条目数，0值表示全部条目（显示可省略）。
        // 例：
        // TOPS[1] 与 TOP 效果相同。

27      PEEK(1)     引用栈内任意条目，从栈顶开始逆向计数。
        // 附参：目标条目下标，从0开始计数。
        // 例：PEEK[0] 与 TOP 相同。

28      INDEX(1)    引用栈内任意条目，从栈底开始计数。
        // 附参：目标条目下标，从0开始计数。

29      GETS(1,1)   引用栈内多个条目，从栈底开始计数。
        // 返回：[]interface{}
        // 附参1：起始下标，栈底为0值。
        // 附参2：引用条目数，0值表示后续全部。
        // 注：
        // 附参视为uint8类型，最多支持255条数据。
```


### 4. 取值指令

截取后续指令的返回值到当前位置作为实参（@）或添加到局部变量域中（$），返回值不再入栈。`@` 主要用于不定参数指令的实参设置，`$` 则为程序编码提供一个有限的变量空间（当前域）。这是两个特殊的符号指令，需配合有返回值的指令使用。

```go
指令值  指令字      说明
-----------------------------------------------------------
30      @           值固化。截取后续指令的返回值到当前位置（实参化）。
        // 固化的值可以连续多个，它们的顺序与返回值的顺序一致。
        // 如果返回值是一个 []interface{} 类型，则会被自动展开。
        // 例1：
        //      @GETS[0,2] PRINT[16]
        // 获取栈底2个值展开为2个实参，向控制台打印它们的16进制表示。
        //
        // 例2：
        //      CASE{ @POP + @{99}, {1000} }
        // 测试表达式：
        //      @POP + @{99}
        //      取出栈顶值 + 99，结果值自动入栈。
        // 分支语句：
        //      {1000}
        //      取结果值与 SWITCH 标的值相等比较，是则将值 1000 入栈。
        //      注：结果值首先从实参空间查询，遵循二阶获取。
        //
        // 例3：
        //      CASE{ {100} @EQUAL, ENV{FromHeight} }
        // 测试表达式：
        //      {100} @EQUAL
        //      值100入栈，取栈顶2个值相等比较，结果当前保持。
        // 分支语句：
        //      ENV{FromHeight}
        //      取结果值与 SWITCH 标的值相等比较，是则将所属区块高度值入栈。

31      $           变量赋值。取后续指令的返回值添加到当前变量域中。
        // 因为取值能力的限制（见下），最多可添加128个值，多出视为错误。
        // 变量域中的值无法修改，它们是只添加的。这是有意的抑制复杂性设计。
        // 例：
        // $TOP 引用栈顶项复制添加到当前局部变量域中。
        // $INDEX[0] 引用栈底项复制添加到当前的局部变量域中。
        // $MO_MATH{Abs} 取栈顶值计算绝对值后，存入当前局部变量域中。
        //
        // 说明：
        // 同上 @ 指令，[]interface{} 类型的返回值会被自动展开。
        // 脚本顶层拥有一个局部性质的「脚本域」，用于脚本顶层的局部变量存储。

32      ${}(1)      当前变量域取值。按添加时的顺序引用变量的值作为实参。
        // 附参：局部变量域成员下标，值范围：[0-127]。
        // 这是 $ 赋值指令的逆指令，目标由位置下标决定。
        // 与 @ 指令的效果一样，提取的值也进入了当前的实参空间。
        // 例：
        // $[0]  取第1个添加进局部变量域中的值。
        // $[-1] 取最后一个添加到局部变量的值，可能更常用。
        // 注：负下标提供一种友好定位方式，指逆向计数。
        //
        // 作为补充，可以用$赋值时的名称来取回值，名称用花括号包围。
        // 因为存在名称重复覆盖的问题，这只是一种辅助语法，只宜「就近」使用。
        // 例：
        // $POP   取栈顶值添加到局部变量域。
        // ${POP} 从局部变量域取回上面添加的值。
        // ${MO_MATH.Abs} 模块指令与其成员可用句点连接。
        // 注意：
        // 重复名称的值添加没有问题，但取值则为同名的最后一次添加值。
        //
        // 注记：
        // 这里的表达都只是一种视觉显示，实际的指令附参是一个数值。
        // 指令序列的预解析（构造交易时）会将名称或负值对应到实际的下标值。
```


### 5. 结果指令

对脚本的执行实施决定性控制，即产生某种结果。除非已明确「通过」，否则失败退出或简单结束都不会转移信用。
也即：如果想要脚本代表的信用转移出去，必须执行通关指令（PASS, MOVE）。

> **注：**<br>
> 通关指令类似于「安检」的逻辑。关口可能有多个，一个无效则全部失败。<br>

```go
指令值  指令字      说明
----------------------------------------------------------
33      PASS        检查通过。实参值为TRUE时通过。
34      MOVE        转移通过。实参值为TRUE时，关联凭信检查通过（属性相关）。
        // 说明：
        // 以上两个指令用于通关检查，也称为通关指令。
        // 实参为 FALSE 或关联检查不合格时导致失败，脚本终止。
        // 注：
        // MOVE 专用于凭信，会触发关联检查，应当用在末端（逻辑上）。
        // PASS 通用，用于凭信脚本中只是简单通关检查。
        // 通关失败时会自动调用 BUFDUMP 指令，向系统外传递信息。
        // 注意：
        // 凭信中只有最后使用了 MOVE 指令，才会决定是否成功。

35      GOTO(4,32,2) 执行流跳转。跳转到某区块某交易的某序号的输出脚本。
        // 实参数据被传递到新脚本作为解锁参数入栈。
        // 新脚本继承当前的脚本环境，但有其自身的执行域（栈、缓存区、全局空间）。
        // 新脚本的执行结果即为当前脚本的结果（如PASS状态）。
        // 附参：(Height[4], TxID[32], OutIndex[2])
        // 含义：(理想块高度，交易ID，输出索引)
        // 例：
        //      @TOPS[10] GOTO[1000,0xcca750...,1]
        // 引用栈顶10项数据，跳转并传递到第1000号理想块的目标交易的第2项输出。
        //
        // 注：
        // 考虑安全和性能，跳转指令累计使用不可超过4次。
        // 累计使用指实际执行次数，而非代码中指令的数量（如分支中重复）。

36      JUMP        执行流跳转。
        // 功能类似GOTO，但由实参提供跳转目标（同GOTO附参）。
        // 跳转到的目标脚本保持当前脚本的环境和执行域，执行后自动返回。
        // 例：
        //     JUMP FN_HASH256 DATA{...} EQUAL PASS
        // 取栈顶3项目标定位跳转，返回后取栈顶值计算哈希摘要...。
        // 目标脚本可能往当前栈中添加了哈希源数据，因此返回后验证。
        //
        // 注：
        // 该指令类似于第三方脚本片段嵌入的逻辑。
        // 因为脚本的执行域也共享，所以实际上可通过全局空间传递信息。
        // 这是一个跳转指令，一个脚本中连续使用不得超过4次。

37      ~~~         （系统保留）

38      RETURN      结束脚本的执行并向系统返回实参。
        // 不定参数指令。
        // 系统对返回值目前仅仅是向控制台简单打印。
39      EXIT        结束脚本的执行。
        // return作用类似但不返回任何值。
```


### 6. 运算指令

有两种形式的运算指令：
1. 符号指令。操作数在符号两侧的常见形式，如：`{100} + {10}`。
2. 命名指令。普通指令，操作数作为实参在指令之前，如：`{100} {10} ADD`。

命名指令就像普通指令一样执行，结果自动入栈。符号指令要复杂一些，它们用于构造表达式。


#### 表达式

表达式是一个完整的运算单元，内部的操作数只能是值指令和直接取值的指令（如 `@INDEX[n]、$[n]、ENV{n}、VAR[n]`），而无法包含需要从实参空间取实参的函数类指令（如 `MUL、BOTH` 等）。作为必要的特性，表达式内的值指令不再自动入栈（如：`{100}` 与 `@{100}` 同）。

**这是一种严格受限的表达式逻辑**，因为函数实参是书写在函数（指令）之前而不是之后。

符号指令可用于构建复杂的表达式，整个表达式运算结束后，结果自动入栈，就像是存在一个「表达式指令」一样。

```go
指令值  指令字      说明
-----------------------------------------------------------
40      ()(1)       表达式封装或优先级分组。
        // 附参：包含的表达式长度，显示省略。
        // 顶层表达式的结果自动入栈，内部子表达式结果直接参与运算。
        // 例1：
        //      ( @INDEX[0] + @{100} )
        //      ( @INDEX[0] + {100} )
        // 创建一个表达式：引用栈底值加100。计算结果自动入栈。
        // 值{100}前的取值指令@可选，通常不再需要（表达式内取消了自动入栈）。
        //
        // 例2：
        //      ( {1.5} * (@INDEX[0] + {100}) )
        // 创建了一个复杂表达式：内部包含了加法的优先级分组（子表达式）。
        // 最外围的初始括号是必需的，它创建了顶层表达式实体。
        //
        // 注记：
        // 如果表达式中包含浮点数，整数会自动转换为float64计算。结果为浮点数。

41      ~~~         （系统保留）
        // 未来可能用于赋值操作或其它。

// 符号指令（双操作数）
42      *           乘
43      +           加
44      ~~~         （下用）
45      -           减
46      %           余（正负号跟随左操作数，同go语言）
47      /           除
48      ~~~         （下用）

49      <<          左移位
50      >>          右移位
51      &           位与：AND
52      |           位或：OR
53      ^           位异或（XOR）
        // 实参在指令前后两侧，可构造复杂表达式。
        // 例：
        //      ( {100} + @INDTX[0] * {1.5} / {2} )
        // 表达式内存在符号指令本身的优先级。优先级逻辑同go语言的设计。
        // 最后运算结果为浮点数（运行时float64）自动入栈。

// 命名指令（双操作数）
44      MUL         乘：MUL(x, y) => x*y
48      ADD         加：ADD(x, y) => x+y
54      SUB         减：SUB(x, y) => x-y
55      DIV         除：DIV(x, y) => x/y
56      MOD         模：MOD(x, y) => x%y
        // 实参：有序的两个操作数。
        // 例1：
        //      {32} {100} MUL
        // 入栈值32、入栈值100，取栈顶2项值计算乘法，结果入栈。
        //
        // 例2：
        //      @POP @{100} MUL
        // 弹出栈顶值为实参，值100为当前实参，相乘后结果入栈。
        // 注意：
        // @POP {100} MUL   错误。{100}的入栈不需要前置实参（参数错误）。
        // {100} @POP MUL   错误。{100}先入栈后被取值，单实参乘法运算无效。
        // 更好的：
        // ... {100} MUL 无需预先 POP 弹出。

// 命名指令（单操作数）
57      NEG         取负：-x
58      NOT         取反：!v （仅适用Bool类型）
        // 实参：目标操作数。
        // 例：
        //      @INDEX[0] PUSH NEG
        // 1. 取栈底值压入栈顶。
        // 2. 取栈顶值转为负值后入栈。

59      DUP         复制：返回2个值
        // 适用任意数据类型，但主要用于Bytes和String。
        // 例：
        //      <pubKey> DUP FN_HASH256 ...
        // 1. 公钥入栈。
        // 2. 取出栈顶公钥复制后双双入栈。
        // 3. 取出栈顶值计算哈希摘要（32字节）入栈。
```

**注意：**
> `ADD, +` 指令对于字符串（String）和序列（Bytes）是连接操作。


### 7. 比较指令

设计为类似函数的操作方式，指令需要的两个操作数由所取实参定义，结果自动入栈。

```go
指令值  指令字      说明
-----------------------------------------------------------
60      EQUAL       相等：==
61      NEQUAL      不相等：!=
62      LT          小于：<
63      LTE         小于等于：<=
64      GT          大于：>
65      GTE         大于等于：>=
        // 说明：
        // 上面的操作符只是一个类比表达，并不作为指令字名称。
```

**说明：**
> 序列（Bytes）类型的大小比较按从左到右的逐字节进行。<br>
> 布尔（Bool）类型仅支持 `EQUAL, NEQUAL` 比较。<br>
> 浮点（Float）类型应用于 `EQUAL, NEQUAL` 比较时可能需要小心。<br>


### 8. 转换指令

本设计中脚本里的值是准严格类型（类似Golang），大部分值需要显式类型转换。
实参转换后会自动入栈。

```go
指令值  指令字      说明
-----------------------------------------------------------
66      BOOL        转换为布尔值：Bool(x)
        // 实参：字符串、整数、浮点数。
        // 字符串：""  => FALSE; 非空串 => TRUE
        // 整数：  0   => FALSE; 非零值 => TRUE
        // 浮点数：0.0 => FALSE; 非零值 => TRUE
        // 例：
        // BOOL IF{ ... }
        // 取出栈顶项转换为Bool值入栈，IF测试栈顶值决定是否执行if块。

67      INT         转换为整数：Int(x)
        // 实参：布尔值、字符串、浮点数。

68      FLOAT       转换为浮点数：Float(x)
        // 实参：布尔值、整数、字符串。

        // 整数：  简单的类型转换
        // 字符串：简单浮点数或科学记数法表示

69      BYTES       转换为序列：Bytes(x)
        // 实参：字符串。
        // 字符串：即字符的UTF-8码值序列

70      STRING      转换为字符串：String(x)
        // 实参：布尔值、整数、序列、浮点数。
        // 布尔值：TRUE => "True"; FALSE => "False"
        // 整数：  十进制字符串
        // 序列：  视为UTF-8编码字符串
        // 浮点数：带小数点数值或科学记数法表示
```


### 9. 流程指令

不像普通编程语言中可以用花括号限定语句块的范围，或者使用END语句结束语法块，本设计中使用限定性很强的长度标定方式，即：表达式或语句块的长度由附属参数准确指定。

附参长度大多为1-2字节，系统假设了不同指令的适用长度。因为长度值的使用，结束指令就不再需要了。
显示时中括号包围的附参值省略。

> **注记：**
> 由附属参数指定表达式或语句块的长度，是一种严格限制的设计。
> 这不只是限制了代码块本身，隐含地，它对代码嵌套的层级也有效果（上层块受限），这可能促进代码逻辑的扁平化。

```go
指令值  指令字      说明
-----------------------------------------------------------
71      IF{}(1)     真值执行。
        // 实参值为TRUE时，执行语句块指令序列。
        // 附参：IF块指令序列长度（<256字节）。
        // 例：
        //      ... EQUAL IF{ @TEXT{Hello} PRINT }
        // 解：
        // EQUAL      取出栈顶两个值比较是否相等，结果入栈。
        // IF{...}    取出栈顶值如果为TRUE，执行语句块内指令序列。
        // @TEXT{...} 取花括号内的 Hello 为实参值。
        // PRINT      向控制台打印（智能格式）。
72      ELSE{}(1)   IF不满足时执行。
        // 与IF配对使用，参数同上。

73      SWITCH{}(2) 分支选择区。
        // 取实参为标的作为后续CASE比较值。
        // 附参：区域长度，2字节定义。
        // 提示：
        // 设标的值为TRUE，可模拟if/elseif/else的逻辑。
74      CASE{}(1,1) 条件分支。
        // 测试表达式结果与SWITCH标的值作相等比较，匹配则执行分支。
        // 附参1：测试表达式长度。
        // 附参2：分支语句块长度。
        // 注：
        // 每个CASE自然结束（无需break），也无贯穿逻辑。
        // 测试表达式的结果无需当前保持，系统取结果时依然为二阶查找。
75      DEFAULT{}(1) 默认分支。
        // 其它分支不可达时执行，通常在所有CASE块之后，可选。
        // 附参：分支语句块长度。

76      EACH{}(1)   迭代循环。
        // 不定参数指令。对实参序列迭代处理。
        // 附参：循环块长度。
        // 当前域变量：
        // - $[0]  // 当前条目数据 ${Value}
        // - $[1]  // 当前循环下标，从0开始 ${Index}
        // - $[2]  // 迭代条目数 ${Total}
        //
        // 注记：
        // 1字节的长度主要为限制负载，但也可促进逻辑的分解，便于审计。
77      BREAK       退出循环或SWITCH。

78-79   （系统保留）
        // 未来可能用于LOOP类指令。
```


### 10. 逻辑指令

实参需为布尔值类型，结果为布尔值，自动入栈。
注：逻辑非指令（NOT）被划归在前面运算指令的单操作数部分。

```go
指令值  指令字      说明
-----------------------------------------------------------
80      BOTH        两个都为真（&&）
        // 逻辑AND，取2个实参测试True值。
        // 例：
        // BOTH PASS 取栈顶两个值，都为真时通过。

81      EVERY       全部实参都为真（类 &&）
        // 不定参数指令。
        // 例：
        // @TOPS[3] EVERY  引用栈顶3个值，测试是否都为True。
        // @TOP $[1] EVERY 引用栈顶值和第2个局部变量的值，测试是否都为True。

82      EITHER      两个中任一为真（||）
        // 逻辑OR，取2个实参测试True值。
        // 例：
        // EITHER PASS 取栈顶两个值，任一为真即通过。

83      SOME(1)     全部实参部分为真（||, &&）
        // 不定参数指令。
        // 附参：为真的最低数量，默认值1。
        // 实参的个数与附参值对比：
        // - |附参值| >= 实参个数：  全为真，类似EVERY
        // - |附参值| < 实参个数：   部分为真
        // 如：
        // - SOME[2]：  2个以上为真（含2个）
        // - SOME[1]：  1个以上为真，默认
        // - SOME[0]：  无条件放行（可能无意义）
        // 例：
        // @TOP $[0] SOME 引用栈顶值和第1个局部变量的值，任一为True即可。
        // @POPS[3] SOME[2] 取出栈顶3个值，测试是否有2个以上为True。
```


### 11. 环境指令

脚本执行时，除了有一个顶层的系统环境之外，还有一个与当前交易和脚本校验相关的「脚本环境」：3个只读域（`ENV、IN、OUT`）。

另外，脚本执行时还有2个全局空间：`TIME、VAR`（执行域），以及和多重签名构造有关的 `MULSIG` 信息环境。

```go
指令值  指令字      说明
-----------------------------------------------------------
84      EVN{}(1)    取脚本环境里的值。
        // 附参：目标名称的标识值（码值），花括号内显示名称。
        // 以下定义都是针对当前正在验证的交易，验证输入脚本。
        // 参考：
        // - Height      理想块高度
        // - FromHeight  源交易所在区块高度
        // - RealHeight  打包进的区块高度
        // - FromID      来源交易ID
        // - TxID        当前交易ID（在验证）
        // - Timestamp   当前交易时间戳
        // - InTotal     输入项总数
        // - InAmount    输入总金额
        // - OutTotal    输出项总数
        // - OutAmount   输出总金额
        // - FromGOTO    是否从GOTO指令跳转过来（用于管理者交易）
        // - FromJUMP    是否从JUMP指令跳转过来（同上）
        // - FromSource  跳转来源指令序列（截止跳转位置，不含跳转指令本身）
        // - JumpCount   跳转计数（含GOTO和JUMP指令）
        // 例：
        // EVN{RealHeight} 获取交易所在区块高度压入栈中

85      IN{}(1)     取当前输入脚本的某个成员值。
        // 附参：输入项中的成员标识，花括号内名称表达。
        // 成员参考：
        // - Amount      币金数量
        // - Receiver    接收者（适用币金和凭信）
        // - Description 凭信描述
        // - Count       凭信转移计数
        // - Title       标签标题
        // - Content     标签内容
        // - Attachment  附件（ID）
        // - Script      脚本自身
        // - Datum       输出条目的完整数据序列
        // - Index       在前阶输出集中的序位
        // 例：
        // IN{Amount} 取输入项的币金数量。
        //
        // 注记：
        // Script 指当前的脚本段，如管理者脚本中引用就指该管理者脚本自身。

86      OUT{}(1)    取当前（在验证）交易的某个输出项的成员值。
        // 实参：目标输出项的序位（起始值0）。
        // 附参：输出项中的成员的标识，名称同 IN 指令。
        // 说明：
        // 成员参考同 IN 指令，略。
        // 即交易的输出可影响使用该输出的后续交易，比如限定接收者。
        // 例：
        // @{1} OUT{Description} 取当前交易第2项输出的描述数据。
        // @POP OUT{Receiver} 栈顶值为输出项下标，取该输出项的接收者。
        //
        // 注记：
        // 目标输出项序位设计为实参，因为序位应该由用户自己掌控。

87      SIGNS(1)    签名数量检查，返回布尔值或签名数量。
        // 合法签名的数量大于等于目标值时返回True，否则为False。
        // 附参：目标签名数。
        // 注：0为特殊值，表示返回验证合法的签名数量。
        // 例：
        // SIGNS[2] 传入的签名中有2位以上验证合法，True值入栈，否则False入栈。
        // SIGNS[0] 传入的签名中合法的签名数量本身入栈。
        // SIGNS[_] 同上。数字0并不是一个恰当的表意，下划线为占位符。
        // SIGNS[]  同上。省略简化但视觉友好（保留括号）。
        //
        // 传统的多签验证实际上可如下：
        //      SIGNS[2] PASS
        // 或者
        //      SIGNS[] {2} GTE PASS
        // 后一种语法中，可以支持从其它地方获取对比目标值{2}（如局部变量）。

88      MULSIG(1)   多重签名序位确认，返回布尔值。
        // 目标序位的签名合法时返回True，否则返回False。
        // 附参：目标序位。即签名者序号，从0开始。
        // 例：
        // MULSIG[2] 确认多重签名哈希列表中第3序位者参与签名。
        //
        // 也可以模拟2/3多重签名验证：
        //      MULSIG[0] MULSIG[1] MULSIG[2] @POPS[3] SOME[2] PASS
        // 不过这样的指令序列太累赘了。

89      TIME{}(1)   取全局时间对象的目标属性值。
        // 该对象存在于脚本的全局域内，花括号内为目标的标准名称。
        // 附参：定义取值的目标属性。
        // 属性：
        // - Stamp      时间戳（微秒）
        // - Year       年数（4位）
        // - Month      月次/年（1-12）
        // - YearDay    日次/年（1-365|366）
        // - Day        日次/月（1-31）
        // - WeekDay    日次/周（0-6）
        // - Hour       时数/日（0-23）
        // - Minute     分钟数/时（0-59）
        // - Second     秒数/分钟（0-59）
        // 例：
        // TIME{Stamp} 获取时间对象的时间戳值入栈。

90      VAR(1)      取全局变量的值。
        // 全局变量存在于脚本的全局域内，最多支持256个变量（0-255）。
        // 附参：目标变量位置下标。
        // 例：
        // @VAR[0] 取全局变量里首个变量值到当前实参空间。
        // VAR[10] 取全局变量里第11号位置的值入栈。
        // 注：
        // 全局变量由 SYS_VAR 指令赋值。
        // JUMP指令跳转的脚本共享同样的全局空间，彼此会相互影响。
```

> **附：**<br>
> 信息分层结构：`系统环境（SYS_ENV） > 脚本环境 > 执行域 > MULSIG > 脚本域 > 局部域`。


### 12. 模式指令

指令序列的模式匹配，主要用于脚本的预执行检测，判断是否符合特定的代码逻辑。

这在要求脚本有特定的功能，必须在实际执行前得到确认时有用。比如链间兑换，需要确认对方的预支付是完整有效的。一个逻辑完整的模式可以让中间件执行匹配测试并提取信息，实现无人工审核的自动化流程。

一个模式序列通常称为一个模型，一个拥有广泛共识的模型通常会被发布在一笔管理者交易中供分享。

> **注：**<br>
> 模型与目标脚本的匹配是完整匹配，即滤除掉可忽略指令，模式序列与目标指令序列需同时结束。<br>

```go
指令值  指令字          说明
-----------------------------------------------------------
91      MODEL<>{}(1,2)  创建模式匹配区。
        // 开启一个解析器子环境，对实参执行模式匹配测试。
        // 实参： 待测试的指令序列，Bytes类型。
        // 附参1：忽略指令清单长度，<>内显示逗号分隔的指令列表。
        // 附参2：模式区长度，{}内包含了模式匹配序列。
        // 返回： #指令收集的值集。
        // 注意：
        // - 忽略指令清单长度指占用的字节数，而不是指令的个数。
        // - 不便用名称表达的忽略指令（如值指令），可用指令值（数字）书写。
        // - 忽略是指整个指令，包含附参、关联数据和前置的取值实参。
        // 例：
        // 目标脚本：
        // DUP FN_HASH160 DATA{46af3fb4...} @TOP PRINT EQUAL PASS FN_CHECKSIG PASS
        // 注：添加了一条无用的打印公钥哈希的指令。
        // 匹配模式：
        // MODEL<PRINT, INPUT, OUTPUT, BUFDUMP> {
        //      DUP FN_HASH160 DATA[20]{?} #[4] EQUAL PASS FN_CHECKSIG PASS
        // }
        // @POP OUTPUT BUFDUMP
        // 说明：
        // #指令提取了公钥哈希，打印指令被忽略（连同 @TOP 取值实参）。
        // 模式指令执行完后，提取的公钥哈希返回并自动入栈。
        // @POP OUTPUT BUFDUMP 导出公钥哈希并触发注册的（如果有）外部应用。
        //
        // 附注：
        // - 如果一个未知脚本匹配该模式，就说明其支付逻辑正常，即便公钥哈希可能是错的。
        // - 如果模式匹配失败，系统会以类似PASS失败的方式处理。
        // - 可忽略指令在源脚本中可自由出现。它们也可出现在模式中，按普通的指令正常匹配。

// 以下为局部指令，仅在MODEL之内有效。
92      #(1)        提取前一个指令匹配目标的相关值。
        // 提取的值会加入一个队列，它们将作为MODEL指令的返回值。
        // 附参：取值范围类型，下面为二进制。
        // 001 - 指令本身。(1)
        // 010 - 单独附参。(2)
        // 100 - 关联数据。(4)
        // 011 - 指令和附参。(3)
        // 101 - 指令和关联数据。(5)
        // 110 - 附参和关联数据。(6)
        // 111 - 全部值。包含指令本身、附参和关联数据。(7)
        // 例：
        // MODEL<...> {
        //      DATA[32]{?} #4 提取前面DATA匹配目标的数据部分。
        // }
        // 注：
        // #[4] 是正常的附参显示方式，也可简写为 #4。

93      ~~~         （系统保留）

94      ?(1)        指令通配符。
        // 匹配当前位置的目标指令为任意指令，或用后续指令匹配某种任意值。
        // 附参：任意值标识。
        // （下为二进制）
        // 0 - 目标为任意指令（含附参和关联数据）。默认。(0)
        //
        // 用后续指令匹配，附参通配位：
        // 0000 0001 - 关联数据任意。(1)
        // 1000 0000 - 第1个附参任意，其它附参严格匹配。(128)
        // 0100 0000 - 第2个附参任意，其它附参严格匹配。(64)
        // 0010 0000 - 第3个附参任意，其它附参严格匹配。(32)
        // 0001 0000 - 第4个附参任意，其它附参严格匹配。(16)
        // 0000 1000 - 第5个附参任意，其它附参严格匹配。(8)
        // 0000 0100 - 第6个附参任意，其它附参严格匹配。(4)
        // 0000 0010 - 第7个附参任意，其它附参严格匹配。(2)
        // 注：
        // 最多支持7个附参的通配标记，多个通配位可同时设置（位或）。
        //
        // 例1：
        // ?0           目标位置为任意一个指令。?0 是 ?[0] 的简写。
        // ?            同上。默认的0值省略。
        // ?1 DATA[32]  匹配32字节任意序列。
        // DATA[32]{?}  同上。一种更友好的表达。
        //
        // 例2：
        // ?129 DATA    匹配任意长度任意字节序列（两个 DATA{} 值指令）。
        // DATA[?]{?}   同上。[?]表示附参通配，多个附参用逗号分隔。
        // TEXT[?]{?}   匹配任意长度任意文本串（两个 TEXT{} 值指令）。
        // 注：
        // ?后续的 DATA/TEXT 指令通常取附参值小的指令（如 DATA{}(1)，指令值11）。
        // 因为匹配的目标指令附参为任意，所以模式中该指令的附参值无意义。
        //
        // 例3：
        // IF[60]{?}    匹配if指令，关联的语句块长60字节，内容不限。
        // IF[?]{?}     匹配一个if指令，长度和内容都任意。
        // IF{...}      匹配一个if指令，内容进阶匹配。附参值不参与匹配。
        // IF[?]{...}   非法。无法明确内容长度。
        //
        // 例4：
        // ?1{}     匹配任意整数。此处用指令值3，附参值无意义。
        // {?}      同上。通常用此表达。
        // {?.}     匹配任意浮点数，此处用指令值8，附参值无意义。
        //
        // 例5：
        // GOTO[1000,?,0] 或 ?64 GOTO
        // 匹配GOTO指令，目标区块1000，交易ID任意，第1项输出（输出集内0下标）。
        //
        // GOTO[1000,?,?] 或 ?96 GOTO
        // 匹配GOTO指令，目标区块1000，交易ID和输出项位置任意。
        // 注：
        // 全部附参同为任意时，可简写。如：GOTO[?,?,?] 可简写为 GOTO[?]。
        //
        // 例6：
        // ?129 @DATA   匹配取值实参。取值指令（@,$）与跟随指令紧邻。
        // @DATA[?]{?}  同上。更友好。
        // TEXT[512]{?} 匹配512字节长任意文本。TEXT{}(2) 指令，附参严格匹配。
        // @TEXT[?]{?}  匹配取值实参。此为 TEXT{}(1) 指令，附参值无意义。
        // ?128 PRINT   匹配打印指令，打印格式不限。注：打印指令无关联数据。
        // PRINT[?]     同上。无关联数据的指令，模式的数据位忽略。

95      /../        正则模式匹配，适用于String类型。
        // 后续指令为TEXT{}，花括号内为 /../ 包围的正则表达式。
        // 例：
        //      TEXT{/Zip:[0-9]+/}
        // 后续TEXT{}包含的正则表达式与目标位置的TEXT{}值进行匹配。
        // 因为允许长度不同，附参值不会参与比较。匹配目标必须同为字符串类型。
        // 解析：
        // 指令文本解析为实际的指令值序列为（括号示意分隔）：
        //      [93][13](10){Zip:[0-9]+}
        //      [当前指令值][TEXT指令值](附参值){正则表达式}
        // 注：
        // 正则表达式语法请参考 https://golang.org/s/re2syntax

96      !{}(~,~)    值范围匹配，适用于整数。
        // 限定目标值在设定的范围之内，包括边界值。
        // 附参1：下限值，变长整数。
        // 附参2：上限值，变长整数。
        // 例：
        // !{0, 1024} 限定目标值：0 ≦ n ≦ 1024。
        // !{-128, 127} 匹配的目标是一个 {}(1)细整数指令（指令值3）。
        // 注：
        // 系统内部会将目标值转换为 int64 类型后比较。

97      !{}(8,8)    值范围匹配，适用于浮点数。
        // 限定目标值在设定的范围之内，包括边界值。
        // 附参1：下限值，8字节表达。
        // 附参2：上限值，8字节表达。
        // 例：
        // !{0, 1.0} 限定目标值大于等于0，小于等于1.0。
        // 注：
        // 两个范围值必须至少有一个是浮点数表示。
        // 系统内部会将目标值转换为 float64 类型后比较。

98      !BOOL       任意布尔值匹配。
        // 布尔值TRUE或FALSE仅能匹配确定的值，这是一个通配。
        // 例：
        // !BOOL 匹配的目标必须是TRUE或FALSE。

99      ...         任意指令段匹配
        // 匹配任意目标指令序列，包括其附参和关联数据。
        // 非贪婪匹配，后续其它模式指令段与目标指令序列中的首个匹配段匹配。
        // 注：
        // 通配只限于当前域内，子域是一个整体单元（不会进入）。
        // 若置于模式序列的头部或末尾，可以实现前端或后端局部匹配。
        // 例：
        // 目标脚本：
        //      TIME{Stamp} {<expireTime>} GT   // 比较时间戳
        //      @TOP PRINT                      // 栈顶值引用打印
        //      IF{                             // 如果为真
        //          @POP PRINT                  // 弹出栈顶值并打印
        //      }
        //      POP TRUE                        // 弹出栈顶值丢弃，入栈真值
        //      IF{
        //          TEXT{Hello in IF}           // 必然会到这里
        //          @TOP PRINT                  //A: 引用栈顶值打印
        //      }
        //      TEXT{done!}                     // 入栈 done!
        //      @TOP PRINT                      //B: 与IF内的A相同
        //      POPS                            // 弹出栈内数据
        // 匹配模型：
        // MODEL<>{
        //      TIME{Stamp} {<expireTime>} GT
        //      ...                             // 1
        //      IF{
        //          ...                         // 2
        //          @TOP PRINT                  // A
        //      }
        //      TEXT{done!}
        //      @TOP PRINT                      // B
        //      POPS                            // 终点严格匹配
        // }
        // 1.说明：
        // 通配 @TOP PRINT、第一个 IF{} 和 POP TRUE 等指令。
        // 因为后面的 IF{} TEXT{done!} @TOP PRINT 才完整匹配，找到停止点。
        //
        // 2.说明：
        // 通配 TEXT{Hello in IF} 指令，不会跳过内层A匹配到外面的B位置。
        // 一是因为域限制，二是非贪婪匹配的要求。这符合直觉的理解。
        //
        // 匹配模型：
        // MODEL<>{
        //      TIME{Stamp} {<expireTime>} GT
        //      ...                             // 1
        //      IF[?]{?}
        //      ...                             // 2
        // }
        // 说明：
        // 1处的匹配截止到第一个 IF 指令，即通配首个「@TOP PRINT」指令段。
        // 2处的匹配截止到目标脚本终点。这是必须的，否则整个模型匹配失败。
        //
        // 注记：
        // 如果将嵌套的不同层级的域构造成树，这应该是一个广度优先的查询算法。
        // 如果不同域内有模式取值（#），则需要用深度优先的方式汇集。
```


#### 附：模式匹配包含的内容

1. **位置匹配**：滤除掉可以忽略的指令，位置必须一一对应。位置记数不包含可忽略指令。
2. **指令匹配**：指令相同即为匹配。如果指令为值指令，则也隐含了类型匹配逻辑。
3. **附参匹配**：在指令匹配的前提下，附参参与匹配。可前置 `?` 指令包含通配逻辑。
4. **数据匹配**：在指令匹配的前提下，指令关联的数据参与匹配。可前置 `?` 指令表达通配逻辑。
5. **正则匹配**：`/../` 指令表达正则模式匹配，仅对String目标数据有效。
6. **范围限定**：用于整数和浮点数的范围匹配。两个8字节的附参可涵盖低值指令。
7. **布尔匹配**：任意布尔值匹配（TRUE或FALSE）。

> **注：**<br>
> 模式中流程指令（`IF{}`、`SWITCH{}` 等）在有进阶匹配的内容时，其附参不参与匹配。<br>

### 14. 系统指令

执行系统级功能的指令。值区间：`[106-113]`，8个。

```go
指令值  指令字          说明
-----------------------------------------------------------
106     SYS_AWARD       兑奖指令。
        // 检查当前交易所在区块之后240个区块的兑奖槽，计算奖金兑现。
        // 规则：
        //      确认数   兑现比例
        //      ----------------
        //      1        20%
        //      2        40%
        //      3        60%
        //      4        80%
        //      5        100%
        // 说明：
        // 因为是统计之后的区块的兑奖确认数，所以奖励必须超过一天才能兑现。
        // 当前铸造者需要分析前面240个区块的兑奖槽并计算-241号区块的奖金截留。
        // 用户能够取走的奖金和将要被截留的部分是各自计算，但应互为契合。
        //
        // 注记：
        // 兑奖指令仅适用Coinbase交易，可能是被强制执行而非在输出脚本中书写。

107-109 （待定）

110     SYS_ENV{}(1)    获取系统级环境值。
        // 附参：目标名称的标识值（码值）。
        // 项目：
        // - BlockHeight 区块链当前最新高度
        // - LimitStack  栈高度上限（256）
        // - LimitScope  局部变量域上限（128）
        // - ……
        // 例：
        // SYS_ENV{BlockHeight} 获取区块链当前最新高度。

111     SYS_OUT{}(1,1)  获取脚本所在交易某个输出项的成员值。
        // 附参1：目标输出项的序位（起始值0）。
        // 附参2：输出项中的成员的标识，花括号内名称表达。
        // 说明：
        // 脚本所在交易指脚本作为输出时的源交易，即当前输入所在的前阶交易。
        // 这是一个系统级指令，因为输入源所在交易与当前验证交易关联性较弱。
        // 成员参考：
        // （与 IN/OUT 指令相同）
        // 例：
        // SYS_OUT[1]{Script} 取前阶交易第2项输出的脚本数据。
        // SYS_OUT[0]{Description} 取前阶交易第1项输出的描述数据。
        // 注记：
        // 目标输出项序位设计为附参，因为它们必然已知。
        // 如果输出项下标指向脚本所在输出，则与 IN 指令效果相同。

112     SYS_TIME        设置脚本全局时间对象。
        // 实参：目标时间值。
        // 支持微秒级UNIX时间戳（Int）和RFC3339格式串（String）。
        // 另外也支持如下标识值：
        // -1 取当前交易创建的时间（在验证），默认。
        // -2 取当前时间（脚本执行时）。
        // -3 取脚本所在源交易的创建时间。
        // -4 取交易将要打包进的区块（当前最后高度+1）的创建时间。
        // -5 取脚本所在源交易所在区块的创建时间。
        // 例：
        // @TEXT{2006-01-02T15:04:05+08:00} SYS_TIME // RFC3339格式串
        // @{1136185445000000} SYS_TIME // 微秒级时间戳
        // 注：
        // 默认的全局时间对象在脚本执行时创建，取交易的时间戳时间设置。
        // 配合 TIME 指令使用。通常无需手动设置。

113     SYS_VAR(1)      设置全局变量值
        // 把实参值设置到全局变量空间。
        // 用法与局部变量域指令 $ 不同，此为设置模式，最多支持256个变量。
        // 附参：目标变量位置下标。
        // 例：
        //      @(100 + @TOP) SYS_VAR[255]
        // 栈顶值加100后赋值到全局变量域的最后一个变量位置。
```
