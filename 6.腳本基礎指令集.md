## 基礎指令集

> **註**：下面的代碼並非Go語言代碼，僅為顯示友好。


### 1. 值指令

在腳本中作為一個實際的值存在，直接自動入棧。

指令的附參指示了數據的具體長度（字節數），如果書寫時已經寫出了全部數據，則可以省略附參（中括號的部分）。

值區間：`[0-23]`，24個。

```go
指令值  指令名      說明
-----------------------------------------------------------
0       NIL         空值
        // 卽Go語言中的 nil 值。
        // 如果一個腳本中隻有一個NIL，表示腳本不可執行（失敗）。
        // 可視為一個無腳本標識符。

1       TRUE        眞值（True）
2       FALSE       假值（False）
        // 類型：Bool，布爾值。
        // 在源碼中書寫指令名，解析為指令值。
        // 例：
        // TRUE 解析創建的腳本中值為1，運行時自動入棧true。

3       {}(1)       int8，1字節：[-128, 127]
4       {}(1)       byte/uint8，1字節：[0, 255]
5       {}(2)       int16，2字節：[-0x8000, 0x7fff]
6       {}(2)       uint16，2字節：[0, 0xffff]
7       {}(4)       int32/rune，4字節：[-0x80000000, 0x7fffffff]
8       {}(4)       uint32，4字節：[0, 0xffffffff]
9       {}(8)       int64，8字節整數：[-1<<63, 1<<63 - 1]
        // 類型：Int，整數。
        // 附參：指令的關聯數據。
        // 從源碼解析為指令時，數據值的大小自動決定使用哪個指令。
        // 例：
        // {12756}  入棧正整數，指令選用int16（指令值5）。
        //          腳本中指令序列：首字節值5，後2字節為值 12756
        // {0xfff0} 入棧正整數，指令選用uint16（指令值6）。
        //          腳本中指令序列：首字節值6，後2字節為值 0xfff0（65520）
        // {'\n'}   支持單字節字符表達，解析為 byte 類型。
        //          腳本中指令序列：首字節值4，後1字節值 10（換行符ASCII碼值）。
        //
        // 支持千分位的書寫方式。
        // 如：{12,345}  實為整數12345。
        //
        // 註記：
        // 有符號整數和無符號整數有一些重疊，通常優先采用有符號整數指令（不要求）。

10      {}(1+n)     大整數，變長字節。
        // 類型：BigInt。
        // 附參：首字節標記實際數據占用的字節數。
        // 註：也卽Go語言中 math/big 包裏的Int類型。
        // 例：
        // {0x8000ffff 8000ffff 01}
        // 超出了int64的表示範圍。通常用16進製書寫，允許值內空格友好。

11      {}(4)       浮點數（4字節，float32）
12      {}(8)       浮點數（8字節，float64）
        // 類型：Float，浮點數。
        // 附參：指令的關聯數據。
        // 源碼書寫時必須包含小數點，支持科學記數法。
        // 解析為指令時，值的大小自動決定使用哪個指令。
        // 例：
        // {3.14159} 入棧浮點數 3.14159。指令值為11。
        // {2.1606e108} 超出float32容量。指令值為12。
        // 註記：
        // 繫統實現可能會統一轉換到float64類型後使用。

13      DATA{}(1)   字節序列數據。1字節附參。
14      DATA{}(2)   字節序列數據。2字節附參。
        // 類型：Bytes，字節序列。
        // 附參：後續實際數據的長度（字節數）。
        // 源碼書寫時使用16進製字符串，解析時值的長度自動決定指令值（若未指定附參）。
        // DATA{}(2) 指令2字節附參最大可表示64kb數據。
        // 例1：
        // DATA{46af3fb481837fadbb421727f9959c2d} 入棧16字節值，按16進製解析。
        // 解析後的指令序列：
        // [13|16|0x46|0xaf|0x3f|...] 共18字節，首字節指令值，次字節數據長度。
        // 例2:
        // DATA[8]{46af3fb4} 入棧8字節，不足者用0值補充。
        // 解析後的指令序列：
        // [13|8|0x46|0xaf|0x3f|0xb4|0|0|0|0] 共10字節，首字節指令值，次字節數據長度。
        // 註：
        // 花括號內的16進製字符串無需前置 0x 標識。
        // 其它指令的解析類此：首字節指令值，後續為附參（如果有）和關聯數據（如果有）。

15      TEXT{}(1)   文本數據。1字節附參。
16      TEXT{}(2)   文本數據。2字節附參。
        // 類型：String，字符串。
        // 附參：後續實際字符串 `UTF-8` 編碼字節數。
        // 從源碼解析為指令序列時，文本的字節長度自動決定使用哪個指令（15|16）。
        // 例：
        // TEXT{Hello}      入棧 Hello 文本串。
        // TEXT{ "Hello" }  同上，可用引號包圍字符串。
        // TEXT[5]{Hai}     明確聲明數據長度為5字節，多餘字節為0值。
        // 註：
        // 花括號內的字符串無需引號包圍，此時括號內前後空白也是數據。
        // 如果需要添加空白友好視覺，可用引號把字符串括起來。
        // 註意：
        // 通常無需明確指定附參值大小，對於寬字節字符來說，這反而容易出問題。

17      /.../(1)    正則表達式。
        // 類型：RegExp。
        // 附參：表達式占用字節數。
        // 雙斜綫之內為表達式的字符串形式。
        // 註：
        // 運行時會解析為一個正則表達式對象。

18      TIME{}(~)   時間表達式。
        // 類型：Date。
        // 附參：UNIX時間戳。變長正整數。
        // 花括號內書寫為標準格式的時間表達式（RFC3339）。
        // 例：
        // TIME{2006-01-02}
        // TIME{2020-11-24T15:04:05+08:00}
        // 註：
        // 解析的UNIX時間戳精確到毫秒。

19-21   （繫統保留）

22      CODE{}(1)   代碼/指令序列。
        // 類型：Script。
        // 關聯數據為一段指令序列。
        // 源碼中書寫為指令名，解析後是一個指令值及其附參值的字節序列（Bytes）。
        // 附參：序列長度（解析結果字節數）。
        // 例：
        // CODE{
        //      TOP FN_HASH256 DATA{<hashResult>} EQUAL PASS
        // }
        // 入棧一段哈希屏障驗證代碼。
        // 註：
        // 附參無需書寫，將由解析器解析後自動設置。

23      （保留未用）
```


### 2. 取值指令

截取後面一個跟隨指令的返迴值，讓它們不再自動入棧。依指令的不同，截取的值可能放入實參空間（`@`），或者添加到當前的局部域中（`&`）。另外還有一個附參替代指令，它是一個標識，表示後面緊跟的指令的附參從實參獲取（指令本身有附參的特性）。最後是從前面存儲的局部域中取值的指令（`&{}`）。

這是幾個用特殊符號表示的指令，功能也是特定的。作為一種顯示方便，符號與後面的跟隨指令無需用空格分隔。

值區間：`[24-28]`，5個。

```go
指令值  指令字      說明
-----------------------------------------------------------
24      @           實參創建。
        // 後一個指令的返迴值被攔截放入實參空間。
        // 因為實參空間也為棧頂入值/取值的邏輯，所以也可以理解為放入當前位置。
        // 例1：
        //      @PEEKS[0,2] PRINT[16]
        // 獲取棧底2個值展開為2個實參，向控製臺打印它們的16進製表示。
        // 例2：
        //      @POPS[3] OUTPUT BUFDUMP
        // 彈出棧頂3項數據展開，輸出到緩存區並轉儲（供外部使用）。
        // 註：
        // 如果後面是附參替代指令~（見下），會簡單跳過它。

25      ~           附參替代
        // 指示後一個指令的附參將從實參獲取。
        // 此時後面指令的附參不再有用，所以實際上這些附參會被移除（沒有了）。
        // 這是一種通用的取值渠道切換機製。
        // 註：
        // 大部分附參隻有1字節長，最大值255，從實參取值後可突破這個局限。
        // 該指令應當域後面的指令緊鄰，空格是可選的。
        // 例：
        // @POPS[3]   彈出棧頂3項到當前位置（實參空間）。
        // @POPS[0]   彈出數據棧全部條目到當前位置。
        // @POPS[]    同上。中括號內的0值可省略，但括號必須保留。
        // {3} @~POPS 取實參值3為彈出數量，彈出值放到當前位置。
        // {3} @POPS  同上。沒有中括號表示附參缺失，故省略前置的指令名是可行的。
        // @POPS      自動取棧頂項為實參表示彈出數量，註意與 @POPS[] 的區別。
        // @~POPS     同上，更明確一些。
        //
        // 例外：
        // 不適用表達語法邏輯的流程指令，以及附參用於描述關聯數據的值指令。
        // 解析器會提示源碼中的這種錯誤，最終的指令值序列（交易腳本）裏不會有這種情況。
        // 另外：
        // 考慮安全性原因，不支持引用其它指令的擴展類指令（執行未知指令）。

26      &           局域存值。
        // 取後一個指令的返迴值添加到當前局部域中。
        // 局部變量域空間有限，最多可添加128個值。
        // 與實參創建指令@相同，如果後面是附參替代指令~，會簡單跳過。
        // 例：
        // &TOP       引用棧頂項（復製）添加到當前局部域中。
        // &POPS[3]   彈出棧頂3項自動展開添加到局部域中（同時添加3個）。
        // {3} &~POPS 從實參取彈出數量定義，彈出棧頂3項展開添加到當前局部域。
        // {3} &POPS  同上，中括號缺失表示附參缺失，需從實參取值。
        // &POPS      自動從棧頂取彈出數量定義，然後再彈出指定數量的條目。
        // &~POPS     同上。更明確一些。

27      &{}(1)      局域取值。
        // 按添加時的順序引用局部域的值。
        // 附參：局部域成員下標，支持負數從末尾算起。
        // 這可視為&的逆指令。與@一樣，提取的值進入當前實參區。
        // 例：
        // &[0]   取第1個添加進局部域中的值，放到當前位置（實參區）。
        // &[-1]  取最後一個添加到局部的值放到當前位置。
        // {3} ~& 取實參3為目標位置，取第4個添加進局部域的值（卽 &[3]）到當前位置。
        // 註意混淆：
        // ~& @POPS[3]  自動取棧頂值為目標位置，取出局部域中的值，之後再彈出棧頂3個值。
        // ~&POPS[3]    前段同上，後段 POPS[3] 沒有意義（取出又原樣返迴）。
        // ~& POPS[3]   同上，這才是含義準確的寫法。
        // &POPS[3]     彈出棧頂3項展開添加到當前局部域（&實為存值指令:26）。
        //
        // 作為補充，可以用&賦值時的名稱來取迴值，名稱用花括號包圍。
        // 因為存在名稱重復覆蓋的問題，這隻是一種輔助語法，隻適合「就近」使用。
        // 例：
        // &POP   彈出棧頂項添加到當前局部域。
        // &{POP} 從局部域獲取上面添加的值。
        // &MO_MATH.Abs   取棧頂值計算絕對值後，添加到當前局部域。
        // &{MO_MATH.Abs} 從局部域中獲取上面添加的值。
        // 註：
        // 模塊的方法調用表示為用句點連接模塊名和方法名。

28      （保留未用）
```


### 3. 棧操作指令

對腳本數據棧中的條目執行直接的操作，如：入棧、彈出、引用、切片、映射等。通常會配合取值指令 `@` 或 `&` 使用。

> **註：**
> 與其它指令不同，棧操作指令如果需要實參，將不經由實參區而直接讀取數據棧。

值區間：`[29-43]`，15個。

```go
指令值  指令字      說明
-----------------------------------------------------------
29      NOP         無操作。
        // 實參：不定數量。
        // 返迴值：無。
        // 無任何操作，但會讀取實參（清空實參區）。
        // 因為不會失敗，所以效果類似：TRUE PASS。
        // 例：
        // @POPS[3] NOP 彈出棧頂3項到當前位置，讀取清空實參區。
        // @INPUT[] NOP 讀取導入緩存區到當前位置，清空實參區。
        // NOP          無任何行為，執行繼續。
        // 註記：
        // 不定數量實參允許0個實參（繫統不會自動從數據棧取值）。

30      PUSH        數據入棧。
        // 把前面的實參序列按順序壓入數據棧。
        // 實參：不定數量。
        // 返迴值：無。
        // 例：
        //      &[0] &[1] &[3] PUSH
        // 引用前3個局部變量為實參，順序壓入數據棧。
        // 註：
        // 如果實參空間為空，則沒有任何結果。


// 下面的指令無需實參，
// 直接提取（彈出）棧頂條目。
31      POP         棧頂項彈出。
        // 返迴值：棧頂項。
        // 如果沒有前置取值指令，返迴值會自動重新入棧。
        // 例：
        // POP  彈出棧頂項又返迴入棧，沒有意義。
        // @POP 彈出棧頂項到當前位置（進實參區）。
        // &POP 彈出棧頂項添加到當前局部域存儲。

32      POPS(1)     棧頂多項彈出。
        // 附參：彈出的條目數。
        // 附參為uint8類型，0值表示彈出全部。
        // 返迴值：
        // 彈出的條目集，保持棧內原有底頂序。不定數量。
        // 例：
        // @POPS[4]     彈出棧頂4項展開到當前位置（實參區）。
        // &POPS[4]     彈出棧頂4項展開添加到局部域。
        // @POPS[]      彈出數據棧全部條目到當前位置，中括號為必需。
        // @POPS        取出棧頂項為彈出數量，彈出棧內全部。

33      SLICE(1)    棧頂切片。
        // 取出棧頂多個條目封裝為一個切片。
        // 如果沒有前置取值指令，返迴值會自動入棧（單值不展開）。
        // 附參：取棧條目數。
        // 返迴值：數據切片（單值）。
        // 例：
        // @SLICE[4]  取出棧頂4項打包為一個切片到當前位置。
        // SLICE[4]   同上封裝為切片，但返迴值會被繫統自動入棧。
        // &SLICE[4]  同上封裝為切片，作為單值添加到當前局部域。
        // 註：
        // 這實際上是一個數據封裝指令，變多值為集合單值。
        // 連續的取值可以將棧內數據分別封裝為多個集合組，
        // 如：
        //      @SLICE[3] @SLICE[2] PUSH
        // 將棧頂5項打包為兩個分片，註意先打包的3項最後先入棧。

34      SPREAD      切片展開。
        // 取出棧頂項展開返迴。
        // 如果前置取值指令，會展開壓入實參空間或局部域。
        // 返迴值：展開的條目，不定數量。
        // 例：
        // &SPREAD  取棧頂項展開壓入當前局部域。
        // @SPREAD  取棧頂項展開到當前位置（實參區）。
        // SPREAD   將棧頂項展開後返迴入棧。
        // 註：
        // 棧頂項需要為可展開類型，比如一個切片條目。
        // 連續的取值可以將棧內多個切片逐一展開到目標空間。

35      LOPS(1)     移除棧頂多項。
        // 附參：移除的條目數。
        // 附參為uint8類型，0值表示移除全部（清空數據棧）。
        // 返迴值：無。
        // 例：
        // LOPS[3]  移除棧頂3項（廢除）。
        // LOPS     移除數量由當前實參（取棧頂項）決定。
        // LOPS[]   移除棧內全部數據。
        // @POPS[] NOP  同上。
        // 註：
        // 用於移除棧頂多餘的數據以方便後續處理，
        // 可視作是 POPS 和 NOP 復合指令的簡捷版。


// 下兩個指令實參用途特殊。
36      MAP{}(1)    集合映射。
        // 迭代每一個成員執行子語句塊，返迴值構造為一個集合返迴。
        // 操作目標為棧頂項，需為一個切片或可迭代集合。
        // 附參：子語句塊長度。
        // 實參：不定數量，任意類型。
        // 語法：
        // 子語句塊擁有一個局部私有的數據棧，與腳本的全局數據棧無關。
        // 不定數量的實參序列作為私有數據棧的初始導入條目。
        // 私有數據棧在各個迭代之間是共享的，因此這是一個微型公共空間。
        // 局部域變量：
        // - &[0]  // 當前條目數據 &{Value}
        // - &[1]  // 當前循環下標，從0開始 &{Index}
        // - &[2]  // 迭代集本身 &{Origin}
        // 註：
        // 當前局部域中前3個位置被占用並可由指令更新（對用戶隻讀）。
        // 限製：
        // 迭代循環次數小於256（len(data) < 256）。
        // 例：
        // RANGE[10,3]      // 創建了一個切片[10,11,12]入棧
        // {200} {300}      // 簡單入棧2個值
        // @POPS[2]         // 彈出為實參，將進入MAP的私有數據棧
        // MAP{             // 將迭代前面創建的切片
        //      &{Index} @POP PRINT
        //      &{Index} &{Value} PRINT
        //      @( &[0] * 2 ) RETURN
        // }
        // 將打印共3組值：
        // - 0 300      // 第1次
        // - 0 10
        // - 1 200      // 第2次
        // - 1 11
        // - 2 NIL      // 第3次，棧內已無值
        // - 2 12
        // 最後的結果集：[20, 22, 24] 自動進入上層數據棧。

37      FILTER{}(1) 集合過濾。
        // 迭代每一個成員執行子語句塊，提取返迴眞值的源成員。
        // 操作目標為棧頂項，需為可迭代對象。
        // 附參：子語句塊長度。
        // 實參：不定數量，任意類型。
        // 語法：同上MAP。
        // 局部域變量：同上MAP。
        // 限製：同上，迭代循環次數小於256。
        // 例：
        // RANGE[10,4]      // 創建：[10,11,12,13]
        // FILTER{
        //      ( &[0] % 2 )    // 註：&[0] 沒有簡寫形式（&0）
        //      @BOOL RETURN
        // }
        // 最後的結果：奇數集 [11, 13] 自動進入上層數據棧。
        // 註：
        // 依然可以前置取值指令截取返迴值：@FILTER{...}


// 以下為引用方式，不取出棧條目。
// 如果無前置取值指令，返迴值自動入棧類似於克隆。
38      TOP         引用棧頂項。
        // 返迴值：棧頂項。
        // 例：
        // @TOP 引用棧頂項到當前位置。
        // TOP  引用棧頂項並返迴之，自動入棧。註：同 DUP[2]
        // &TOP 引用棧頂項存入當前局部域。

39      TOPS(1)     引用棧頂端多個條目。
        // 附參：引用條目數，0值無意義。
        // 附參uint8類型，受限於255數量。
        // 如果無前置取值指令，相當於多值克隆。
        // 返迴值：棧頂多項，不定數量。
        // 例：
        // TOPS[1] 與 TOP 效果相同。
        // TOPS[3] 引用棧頂3項返迴。相當於克隆棧頂3項，保持原順序。
        // TOPS    取出棧頂項作為數量，據此引用之後的棧頂各條目。
        // TOPS[]  無引用返迴空，無意義。

40      PEEK(1)     引用棧內任意位置條目。
        // 附參：目標條目下標。
        // 附參int8類型，棧底為0，支持負數從棧頂算起。
        // 返迴值：引用目標項。
        // 例：
        // PEEK[-1] 與 TOP 相同。
        // PEEK[0]  引用棧底項（效果：克隆棧底項放到棧頂）。
        // @PEEK[]  同上引用，放入當前位置。
        // &PEEK[]  同上引用，放入當前局部域。

41      PEEKS(1,1)  引用棧內任意位置段條目。
        // 附參1：起始位置下標。
        // int8類型，棧底為0，支持負值從棧頂算起。
        // 附參2：引用條目數。
        // uint8類型，最多引用255條。0值表示目標位置之後全部。
        // 返迴值：目標位置段條目，不定數量。
        // 例：
        // @PEEKS[0, 5]  引用棧底5項數據到當前位置。
        // @PEEKS[-5, 0] 引用棧頂5項到當前位置，同：@TOPS[5]
        // PEEKS[0, 5]   引用棧底5項，效果為克隆棧底5項到棧頂。
        // PEEKS         取出棧頂2項為附參替代，據此引用。
        // PEEKS[0, 0]   沒有實際意義。
        // PEEKS[]       合法，但同上沒有意義。
        // 例：
        //      PEEKS[0,5] SLICE[5]
        // 效果：打包棧底5項為一個切片放到棧頂（單項）。
        //
        //      @PEEKS[0,5] SLICE[5]
        // 註意：與上不同，此為連續取值。
        // 1. 引用棧底5項到當前位置（展開）。
        // 2. 打包棧頂5項為一個切片到當前位置（單值）。

42-43   （保留未用）
```


### 4. 交互指令

### 5. 結果指令

對腳本的執行實施檢査和控製，如：通過、失敗、退出或跳轉等。

除非腳本就隻有一個 `NIL` 值，否則腳本執行中如果沒有碰上失敗，就視為驗證通過。

值區間：`[50-57]`，8個。

```go
指令值  指令字      說明
----------------------------------------------------------
50      PASS        檢査通過（安檢）。
        // 實參值為TRUE時為通過，否則為失敗。
        // 通過失敗會導致腳本非通過結束，信用轉移失敗，腳本終止。
        // 實參：布爾值，單個。
        // 參考：
        // 通關失敗應當觸發一個通知機製（如BUFDUMP），以便外部可以了解情況。

51      FAIL        驗證失敗。
        // 實參值為TRUE時失敗。
        // 這是 NOT PASS 的簡捷版，但表意更清晰。
        // 實參：布爾值，單個。

52      GOTO(4,32,2) 執行流跳轉（獨立）。
        // 跳轉到某個區塊的某筆交易的某個序號的輸出腳本。
        // - 實參數據（如果有）被傳遞到新的腳本作為解鎖參數使用。
        // - 新腳本繼承當前的校驗域，但有其自身的腳本域（數據棧、全局空間、緩存區）。
        // - 新腳本的執行結果卽為當前腳本的結果（PASS狀態）。
        // 附參1：區塊高度（理想塊），uint32類型
        // 附參2：交易ID，32字節哈希字節序列
        // 附參3：輸出項索引，uint16類型。
        // 實參：任意，不定數量。
        // 例：
        //      @TOPS[10] GOTO[1000, 0xcca750..., 1]

        // 註：實參也可能是一個 CODE 的指令值序列。
        //
        // 安全性：
        // 考慮安全性和運行性能，繫統的實現中連續跳轉應當不超過一定次數。

        // 跳轉定位與GOTO指令相同。

        // 例：
        //     JUMP FN_HASH256 DATA{...} EQUAL PASS
```

### 6. 流程指令

不像普通編程語言中可以用花括號限定語句塊的範圍，本設計中使用限定性的長度標定方式：表達式或語句塊的長度由附參確定。附參的字節數限定了語句塊的長度，這是一種嚴格限製性的設計。因為長度值的使用，結束指令就不再需要了。

語句塊的最終長度由解析器解析計算而來，書寫時無需包含中括號的附參明示。指令的子語句塊卽為該指令的關聯數據。

> **註：**
> 因為頂層語句塊的長度有限，所以實際上代碼的嵌套層級也會受限。這可能是有益的。

值區間：`[58-65]`，8個。

```go
指令值  指令字      說明
-----------------------------------------------------------
58      IF{}(1)     眞值執行。
        // 實參值為TRUE時執行子語句塊。設置IF狀態域。
        // 附參：子語句塊長度，uint8類型。
        // 實參：布爾值，單值。
        // IF狀態域：賦值為實參值的邏輯反，由ELSE使用。
        // 例：
        // ... EQUAL                // 比較結果入棧。
        // IF{                      // 取出棧頂值，為TRUE則執行子語句塊
        //      TEXT{Hello!} PRINT  // 打印 Hello!
        // }
        // 條件值在IF指令之前就計算好了，此僅簡單比較卽可。
        // 狀態域存值是為了後面可能有的ELSE語句自我判斷。
        // 參考：
        // IF和ELSE之間可能可以允許其它語句，隻要是平級的，後來的IF會覆蓋之前的IF存值。

59      ELSE{}(1)   IF不滿足時執行。
        // 檢査IF狀態域，如果為眞則執行。
        // 附參：指令塊長度，uint8類型。
        // 實參：無。
        // IF狀態域：覆蓋賦值為FALSE。
        // 註：
        // IF狀態域覆蓋賦值之後，後面的ELSE就會無用（也是錯誤的）。
        // 應當與IF指令配對使用。

60      SWITCH{}(2) 分支選擇區。
        // 選擇區開啓。設置SWITCH狀態域。
        // 附參：區域長度，uint16類型。
        // 實參：任意值，單值。
        // SWITCH狀態域：賦值為實參值，由後面的CASE使用。
        // 提示：
        // 若標的值為TRUE，可模擬if/elseif/else的邏輯。

61      CASE{}(1)   條件分支。
        // 取實參值與SWITCH狀態域的值比較：
        // 眞：執行子語句塊。
        // 假：跳過子語句塊，繼續後續平級語句塊。
        // 附參：語句塊長度。
        // 實參：任意類型，單值。
        // CASE塊執行之後卽退出SWITCH結構。也可主動退出（BREAK）。
        // 例1：
        // @POP                     // 將被放入SWITCH狀態域
        // SWITCH{
        //      @POP + @{99}        // CASE條件先計算。此處結果入棧
        //      CASE{               // 取棧頂值與SWITCH狀態域的值比較，相等則執行子語句塊
        //          {1000}          // 將值 1000 入棧
        //          BREAK           // 退出SWITCH，可選（會自然退出）
        //      }
        //      {100} @EQUAL        // 下一個CASE的條件。此處結果當前保持
        //      CASE{               // 取實參值與...比較...
        //          ENV{FromHeight}
        //      }
        //      DEFAULT{...}        // 默認分支，抵達則無條件執行
        // }
        // 註記：
        // CASE指令的條件段與指令本身處於平級，這樣讓條件計算更靈活。
        // 已經執行的CASE子塊會直接退出SWITCH結構，因此不會帶來多餘的計算。

62      DEFAULT{}(1) 默認分支。
        // 無條件執行。
        // 附參：語句塊長度。
        // 說明：
        // 其它CASE分支未執行時執行，隻能在所有CASE之後。可選。

63      EACH{}(1)   迭代循環。
        // 對集合（切片）成員逐一迭代處理。
        // 附參：循環塊長度。
        // 實參：切片或可迭代集合。
        // 局部域變量：
        // - &[0]  // 當前條目數據 &{Value}
        // - &[1]  // 當前循環下標，從0開始 &{Index}
        // - &[2]  // 迭代條目總數 &{Total}
        // 限製：
        // 迭代循環次數小於256（len(data) < 256）。

64      BREAK       主動退出EACH或SWITCH。
        // 實參：無。
        // 參考：
        // 本指令可能用panic來實現（上層捕獲）。

65      （保留未用）
```

### 8. 運算指令

有兩種形式的運算指令：
1. 符號指令。操作數在符號兩側的常見形式，如：`{100} + {10}`。僅用於表達式內，由表達式指令包含。
2. 命名指令。普通指令，操作數作為實參在指令之前，如：`{100} {10} ADD`。適用表達式之外的逐項執行邏輯（結果自動入棧）。

值區間：`[78-101]`，24個。


#### 表達式

表達式是一個完整的運算單元，在表達式中使用的操作數隻能是值指令和單純取值類的指令（如 `{123}、POP、PEEK、ENV{n}、VAR[x]、&[n]` 等），無法包含需要從實參空間取實參的功能性指令，因為沒有辦法知道當前實參是用於前面的符號指令還是後面的命名指令。

這是一種**嚴格受限的表達式**邏輯，如果表達式中需要命名指令的值，可以提前執行指令讓結果入棧，然後在表達式中簡單引用或取值。

表達式是一個獨立的小環境，最終的計算結果作為表達式指令的結果返迴而自動入棧，但內部的指令的返迴值則沒有自動入棧的邏輯。


#### 指令清單

```go
指令值  指令字      說明
-----------------------------------------------------------
78      ()(1)       表達式封裝（根）。
        // 內部可以包含子表達式，由優先級分組指令（見下）封裝。
        // 內部的計算不涉及實參空間，因此取值無需@指令（但&取值可用）。
        // 附參：表達式長度。
        // 實參：無。
        // 返迴值：任意類型，單值。
        // 例：
        //      ( PEEK[0] + {100} )
        // 棧底值加100，返迴值自動入棧。
        // 例：
        //      ( {1.5} * (PEEK[0] + {100}) )
        // 包含加法子表達式的復合表達式。最後的結果返迴後會自動入棧。

79      ()(1)       優先級分組（子表達式）。
        // 僅存在於根表達式之內，卽：( ...() ) 括號內部的括號。
        // 附參：子表達式長度。
        // 實參：無。
        // 返迴值：任意類型，由根表達式內部處理。


// 符號指令（雙操作數）
// 註：僅用於表達式內部。
80      *           乘
81      +           加
82      -           減/負
83      %           模（正負號跟隨左操作數，同Golang）
84      /           除

85      <<          左移位
86      >>          右移位
87      &           位與：AND
88      |           位或：OR
89      ^           位異或（XOR）
        // 實參在指令前後兩側，遵循普通的運算符優先級。
        // 例：
        // ( {100} + PEEK[0] * {1.5} / {2} )
        // 註：運算符之間的空格是可選的。


// 命名指令（雙操作數）
// 註：無法用於表達式內部。
90      MUL         乘：MUL(x, y) => x*y
91      ADD         加：ADD(x, y) => x+y
92      SUB         減：SUB(x, y) => x-y
93      MOD         模：MOD(x, y) => x%y
94      DIV         除：DIV(x, y) => x/y
        // 實參：有序的兩個操作數。
        // 返迴值：相關類型，單值。
        // 例：
        // {32} {100} MUL  入棧32和100，取棧頂2項相乘
        // @POP @{100} MUL 先取出到實參區後計算。
        // 另：
        // @POP {100} MUL  錯誤。實參區實際上隻有1個值（@POP）。

95      DIVMOD      除並求餘。
        // 返迴值：商和餘數，2個值。
        // 運算式：(n, m) => n % m


// 命名指令（單操作數）
// 註：無法用於表達式內部。
96      NEG         取負：-x  // Int, Float
97      NOT         取反：!v  // Bool
        // 實參：目標操作數。
        // 例：
        // @PEEK[0] NEG   取棧底項置負後返迴
        // {123} BOOL NOT 結果為False

98      DUP(1)      復製
        // 同一數據可復製多次。
        // 附參：復製次數。uint8類型，0值沒有意義。
        // 返迴值：復製的條目，不定數量。
        // 例：
        // DUP[1]       取出棧頂項復製1個返迴（實際上沒做啥）。
        // DUP[2]       取出棧頂項復製為2個返迴。效果同 TOP。

        // 註：
        // 適用任意數據類型，但主要用於Bytes和String。

```

**註意：**
> `ADD, +` 指令對於字符串（`String`）和序列（`Bytes`）是連接操作。


### 9. 比較指令

兩個或多個操作數，返迴一個布爾值（單值）。

值區間：`[102-109]`，8個。

```go
指令值  指令字      說明
-----------------------------------------------------------
102     EQUAL       相等（a == b）
103     NEQUAL      不相等（a != b）
104     LT          小於（a < b）
105     LTE         小於等於（a <= b）
106     GT          大於（a > b）
107     GTE         大於等於（a >= b）
        // 實參：兩個操作數。
        // 運算式：(a, b) => a op b

108     WITHIN(1,1) 範圍判斷。
        // 附參1：最小值（包含）。
        // 附參2：最大值（包含）。
        // 實參：1個待比較值。
        // 例：
        // ENV{RealHeight} WITHIN[0, 239]
        // 取當前交易所在眞實區塊高度，檢査是否為第一天收錄。
        // 註：
        // 本指令和下面的同名指令是對小整數的一個友好支持。
        // 如果需要更大的值範圍，可使用附參替代技朮。
        // 參考：
        // 實現可能適用任意類型的值範圍判斷（如果可比較）。

109     WITHIN(2,2) 範圍判斷。
        // 同上，附參取值稍大。
        // 例：
        // WITHIN[256, 0xffff]     棧頂值是否為必須2個字節表示的值。
        // {256} {0xffff} WITHIN   同上，但由實參定義範圍。
        // {256} {0xffff} ~WITHIN  同上，更明確的寫法。
        // 註記：
        // 繫統會因附參替代指令而先取出需要的數據，之後才是實參取值。
        // 如果實參空間有值，提取替代附參時同樣是一次性的邏輯。
        // 因此：
        // @{256} @{0xffff} WITHIN            正確。一次性取出2個附參。
        // &[0] @{256} @{0xffff} WITHIN       錯誤。取出的附參數量不對。
        // &[0] PUSH @{256} @{0xffff} WITHIN  正確。待比較值 &[0] 已入棧。
        // 註：
        // 解析器取WITHIN指令的值可能是前一個（108），這沒有區別。
```

**說明：**
> 序列（`Bytes`）類型的大小比較按從左到右的逐字節進行。
> 布爾（`Bool`） 類型僅適用 `EQUAL, NEQUAL` 比較。


### 10. 邏輯指令

實參需為布爾（`Bool`）類型，返迴值為布爾值（`TRUE` 或 `FALSE`）。

值區間：`[110-113]`，4個。

```go
指令值  指令字      說明
-----------------------------------------------------------
110     BOTH        兩個都為眞（&&）
        // 邏輯AND，是否兩個值都為TRUE。
        // 實參：2個Bool值。
        // 例：
        // BOTH PASS  取棧頂兩個值，都為TRUE時通過。

111     EVERY       全部為眞（類 &&）
        // 邏輯AND，全部實參都為TRUE時為眞。
        // 實參：不定數量Bool類型值。
        // 例：
        // @TOPS[3] EVERY  引用棧頂3個值，檢査是否都為TRUE。
        // @POPS[2] &[0] EVERY  提取棧頂2項，和首個局部域的值，檢査是否都為TRUE。
        // EVERY  實參區無值，返迴TRUE。

112     EITHER      兩個中任一為眞（||）
        // 邏輯OR，兩個中是否有其一為TRUE。
        // 實參：2個Bool值。
        // 例：
        // EITHER PASS 提取棧頂兩個值，任一為眞卽通過。

113     SOME(1)     一部分為眞（||, &&）
        // 邏輯OR，全部中特定數量的值為TRUE。
        // 附參：為眞的最低數量。
        // 實參：不定數量Bool類型值。
        // 如：
        // - SOME[2]  2個以上為眞（含2個）
        // - SOME[1]  至少1個為眞，較常用
        // - SOME[0]  無條件放行。無意義，類似：NOP
        // 例：
        //      @POPS[3] &[0] SOME[2]
        // 提取棧頂3項和局部域值，檢査是否至少有2個為TRUE。
        //      @POPS[3] SOME[0]
        // 彈出棧頂3項，檢査並無條件通過：實參必須為Bool類型。
        // 例：
        // SOME[0]  實參區無值，返迴TRUE
        // SOME[1]  實參區無值，返迴FALSE
```


### 11. 模式指令

指令序列的模式匹配，用於驗證目標腳本是否符合特定的邏輯序列。在排除掉可忽略的指令後，目標指令序列需與模式序列完整匹配。如果匹配失敗，繫統會以 `FAIL` 的方式處理。

一個模式序列通常稱為一個模型。一個擁有廣泛共識的模型可能會被發布在一筆管理者交易中供任意分享（作為 `GOTO` 目標）。

模式匹配很有用處：比如在鏈間兌換時確認對方的預支付腳本是否有效、在選舉子鏈中對投票內容的格式做驗證（以免選票無效）、甚至由中間件執行匹配並提取信息來執行鏈外的業務，實現無人工審核的自動化流程。

總的來說，模式匹配的目的是讓某件事在 **下一步行動前** 得到確認。

值區間：`[114-125]`，12個。

```go
指令值  指令字          說明
-----------------------------------------------------------
114     MODEL<>{}(1,2)  創建模式匹配區。
        // 開啓一個解析子環境，等待對實參字節序列執行模式匹配。
        // 附參1：可忽略指令清單長度，<>內為逗號分隔的指令列表。
        // 附參2：模式區長度，{}內包含了模式匹配序列。
        // 實參： 待測試的指令序列（Bytes），不定數量。
        // 如果實參區有多個值，會被自動合並為一個字節序列。
        // 註：因為屬於不定數量實參，所以不會自動從數據棧獲取目標。
        // 返迴值：#指令收集的值，不定數量。
        // 註意：
        // - 可忽略指令清單長度指占用的字節數，而非指令的個數。
        // - 不方便用名稱表達的可忽略指令（如值指令），可用指令值表示。
        // - 忽略是指整個指令，包含附參、關聯數據和前置的實參區取值。
        // - 可忽略指令也可出現在模式區內，此時視為普通指令處理。
        // 例：
        // 目標腳本：
        //      TOP FN_HASH160 DATA{<address>}
        //      @TOP PRINT                      // 將被忽略
        //      EQUAL PASS FN_CHECKSIG PASS
        // 模型：
        //      MODEL<PRINT, INPUT, OUTPUT>
        //      {
        //          TOP FN_HASH160 DATA[20]{?} #[4]  // 換行僅為顯示友好
        //          EQUAL PASS FN_CHECKSIG PASS      // 嚴格匹配（原樣）
        //      }
        //      @POP OUTPUT BUFDUMP     // 模型之外，正常代碼
        // 說明：
        // - #指令提取了公鈅哈希（假設20字節長）。
        // - 打印指令被忽略（連同 @TOP 實參取值）。
        // - 模式匹配結束後，返迴提取的公鈅哈希（自動入棧）。
        //
        // 附註：
        // 如果一個未知腳本與該模式匹配，就說明其支付邏輯正常（公鈅任意）。


// 以下為局部指令，僅在MODEL{}之內有效。
115     #(1)        取值指示。
        // 針對前一個指令匹配的目標，取目標的不同部分的值。
        // 附參：取值標識。
        // 0001 - 指令本身。(1)
        // 0010 - 附參部分。(2)
        // 0100 - 關聯數據。(4)
        // 1000 - 取值實參（如果有）。(8)
        // 可合並標識：
        // 0011 - 指令和附參。(3)
        // 0101 - 指令和關聯數據。(5)
        // 0110 - 附參和關聯數據。(6)
        // 0111 - 指令整體：指令、附參和關聯數據。(7)
        // 1111 - 目標整體，含取值實參的值。(15)
        // 例：
        // MODEL<...> {
        //      DATA[32]{?} #4  提取前面DATA匹配目標的數據段。
        // }
        // #4 是 #[4] 的簡寫形式。
        // 註：
        // 提取的值會放入一個內部隊列，作為MODEL最終的返迴值。

116     $(1)        正則匹配取值。
        // 針對前一個正則匹配目標，取匹配值或子匹配式的值。
        // 附參：正則表達式或子表達式的取值序位。
        // 例：
        // MODEL<...> {
        //      TEXT{/[a-zA-Z]+/} $[0]  取目標字符串中匹配的單詞
        // }
        // 例：
        // TEXT{/[a-zA-Z]+/} $0         同前。
        // TEXT{/Version-(\d+)/} $[1]   提取主版本號（首個子表達式匹配項）
        // TEXT{/Version-(\d+)/} $1     同上。簡寫形式
        // 註：
        // 上面示例中省略了上級封裝指令 MODEL<>{}。
        // 指令 TEXT{/../} 詳見後。

117     ?           指令通配。
        // 目標指令可為任意指令（含附參和關聯數據）。
        // 例：
        // ?    當前目標位置為任意一個指令。

118     ?(1)        指令局部通配。
        // 用跟隨指令來和目標作比較，附參指定了通配的部分。
        // 附參：位置標識。
        // 0000 0001 - 關聯數據任意。(1)
        // 1000 0000 - 第1個附參任意，其它附參嚴格匹配。(128)
        // 0100 0000 - 第2個附參任意，其它附參嚴格匹配。(64)
        // 0010 0000 - 第3個附參任意，其它附參嚴格匹配。(32)
        // 0001 0000 - 第4個附參任意，其它附參嚴格匹配。(16)
        // 0000 1000 - 第5個附參任意，其它附參嚴格匹配。(8)
        // 0000 0100 - 第6個附參任意，其它附參嚴格匹配。(4)
        // 0000 0010 - 第7個附參任意，其它附參嚴格匹配。(2)
        // 註：
        // 最多支持7個附參的通配標記，各個通配位可任意同時設置。
        //
        // 例：
        // ?[1] DATA[32]  跟隨指令的關聯數據任意（但必須為32字節長）。
        // ?1 DATA[32]    同上。
        // DATA[32]{?}    同上。更友好的表達，由解析器智能分辨。
        // TEXT[300]{?}   匹配300字節長任意文本。將被解析為 TEXT{}(2) 指令。
        // 註意：
        // DATA[?]{?}     不確定，依賴於解析器生成的指令：DATA{}(1) 或 DATA{}(2)
        // !Bytes         替代上面的需求，匹配任意字節序列（見後）。
        //
        // 例：
        // ?64 PEEKS[-3]  首個附參確定，第二個附參任意。卽下標位置確定，引用數量任意。
        // PEEKS[-3, ?]   同上，含義更清晰。
        // (?)            目標是一個運算表達式，同：!CalExp
        // ({10} + (?))   目標是表達式內的一個優先級分組指令。
        //
        // 例：
        // IF[60]{?}    匹配if指令，關聯的語句塊長60字節，內容不限。
        // IF[?]{?}     匹配一個if指令，長度和內容都任意。
        // IF[?]{...}   匹配一個if指令，內容等待進階匹配。
        // IF{...}      同上。簡化寫法。
        //
        // 例：
        // @ ?1 DATA[32]    匹配實參取值，任意32字節序列。
        // @ DATA[32]{?}    同上。
        // @ !Bytes         匹配兩個DATA{}指令取值。
        // & POP        匹配 &POP 復合指令，空格是可選的。
        // &[?]         匹配任意取局部域成員指令。
        // ?128 &       同上。

119     !{TYPE}(1)  類型匹配。
        // 目標為特定類型的值指令。
        // 附參：類型名。
        // - Bool   布爾值
        // - Int    整數（7個）
        // - BigInt 大整數
        // - Float  浮點數（2個）
        // - Bytes  字節序列（2個）
        // - String 字符串（2個）
        // - RegExp 正則表達式（/.../）
        // - Date   日期/時間（TIME{}）
        // - Script 腳本代碼（CODE{}）
        // - CalExp 運算表達式（(...)）
        // 例：
        // !{Bool}  目標必須是布爾類型（TRUE|FALSE）。
        // !Bool    同上，可省略花括號。
        // !Bytes   匹配兩個 DATA{} 指令。
        // !String  匹配兩個 TEXT{} 指令。
        // !Script  匹配 CODE{} 指令。
        // !Int     匹配任意整數。
        // !Float   匹配任意浮點數。
        // !CalExp  目標是一個運算表達式：(...)。

120     !{}(~,~)    整數值範圍匹配。
        // 限定目標值在設定的範圍之內，包括邊界值。
        // 附參1：下限值，變長整數。
        // 附參2：上限值，變長整數。
        // 例：
        // !{0, 1024} 限定目標值：0 ≦ n ≦ 1024。
        // !{0, 127}  目標應當是一個 {}(1) 整數指令（不要求）。
        // 註：
        // 整數有多個指令表達，範圍匹配不要求目標具體是哪個指令。
        // 本指令暫不支持大整數（BigInt）範圍。

121     !{}(8,8)    浮點數值範圍匹配。
        // 限定目標值在設定的範圍之內，包括邊界值。
        // 附參1：下限值，8字節表達。
        // 附參2：上限值，8字節表達。
        // 例：
        // !{0, 1.0}  限定目標值大於等於0，小於等於1.0。
        // 註：
        // 兩個邊界值必須至少有一個是浮點數表示，否則會識別為整數範圍。

122     TEXT{/../}(1)   正則模式匹配，適用String類型。
        // 目標需為TEXT{}指令，雙斜綫包圍待用的正則表達式。
        // 附參：匹配表達式的長度。
        // 例：
        //      TEXT{/Zip:[0-9]+/}
        // 目標TEXT{}的內容是否匹配這裏的正則表達式。
        // 註：
        // 正則表達式語法請參考 https://golang.org/s/re2syntax

123     ...         任意指令段匹配。

        //      SYS_TIME{Stamp} {<expireTime>} GT
        //      @TOP PRINT              // 1
        //      IF{
        //          @POP PRINT
        //      }
        //      TRUE
        //      IF{
        //          TEXT{Hello in IF}   // 2
        //          @TOP PRINT          // A
        //      }
        //      TEXT{done!}
        //      @TOP PRINT              // B

        // MODEL<>{
        //      SYS_TIME{Stamp}         // 換行沒問題
        //      {<expireTime>} GT
        //      ...                     // 1
        //      IF{
        //          ...                 // 2
        //          @TOP PRINT          // A
        //      }
        //      TEXT{done!}
        //      @TOP PRINT              // B
        //      LOPS[]                  // 終點完整匹配
        // }
        // 1.
        // 第一個...適配「@TOP PRINT IF{...} TRUE」。
        // 2.
        // 第二個...適配「TEXT{Hello in IF}」，到A為止，不會跨層到B。
        // 註：非貪婪和同域同時約束。
        //
        // 模型：
        // MODEL<>{
        //      SYS_TIME{Stamp}
        //      {<expireTime>} GT
        //      ... IF{?} ...       // IF匹配第二個IF（貪婪）
        // }
        // 第二個...是必須的，匹配到腳本結束，否則整個匹配失敗（完整匹配要求）。

        // 參考：
        // 如果將嵌套構造為樹結構，是一個廣度優先的査詢。
```

#### 附：模式匹配包含的內容

1. **位置匹配**：濾除掉可以忽略的指令後，位置一一對應。卽：位置記數不包含可忽略指令。
2. **指令匹配**：指令相同卽為匹配。如果指令為值指令，則也隱含了類型匹配邏輯。
3. **附參匹配**：在指令匹配的前提下，附參參與匹配。
4. **數據匹配**：在指令匹配的前提下，指令關聯的數據也參與匹配。
5. **正則匹配**：`TEXT{/../}` 可對文本目標進行正則匹配。
6. **範圍限定**：整數和浮點數的範圍匹配，內部的細分類型被忽略。
7. **類型匹配**：對目標的數據類型執行匹配，值任意。

> **註：**
> 流程類指令（`IF{}`、`SWITCH{}` 等）在有子內容匹配時，附參會被忽略。


### 12. 環境指令

腳本的運行處在一個逐層嵌套的環境裏：`繫統環境 > 交易全局 > 校驗域 > 腳本域 > 局部域`，其中前者是後者的父域。

1. **繫統環境**。包含整個區塊鏈繫統的一些基本信息。通常為隻讀。
2. **交易全局**。覆蓋整個交易，可觸及其它兄弟輸出和當前輸入自身。
3. **校驗域**。  當前輸出腳本的校驗運行，包含 `GOTO` 跳轉到的遠端腳本。
4. **腳本域**。  當前腳本範圍，包含 `JUMP` 嵌入的子腳本。
5. **局部域**。  當前腳本頂層的存儲區、以及嵌套語法塊的子域。都是私有邏輯，無法互訪。

### 擴展指令區

最後的8個指令空間用於擴展指令區（`[11111xxx]`）。

指令的擴展籍由附參實現，主要包含了2個預置的引用指令：附參定位了擴展的指令本身，而該擴展指令是一個完整的邏輯，可以包含其自身的附參。這種擴展方式實際上是一種 **附參遞進** 的邏輯，理論上可以無限遞進深層嵌套（類似XML樹）。

值區間：`[248-255]`，8個。

```go
指令值  指令字          說明
-----------------------------------------------------------
248     EX_BASE1(1)     基礎擴展引用（1字節空間）。
        // 1字節的附參可包含256個擴展指令。

249     EX_BASE2(2)     基礎擴展引用（2字節空間）。
        // 2字節的附參可包含64k個擴展指令。

250-252 （待定）

253     EX_PRIV1(1)     私有擴展引用（1字節空間）。
        // 私有範疇，公共節點簡單忽略（不執行）。
        // 通常由有特定需求的應用使用。
        // 註：
        // 如果在模式語句塊內使用，則遵循正常的模式匹配邏輯，
        // 這不會導致指令運行。

254     EX_PRIV2(2)     私有擴展引用（2字節空間）。
        // 同上。較大擴展空間容量（64k）。

255     （繫統保留）
```
