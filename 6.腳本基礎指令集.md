## 基礎指令集

> **註**：下面的代碼並非Go語言代碼，僅為顯示友好。


### 1. 值指令

在腳本中作為一個實際的值存在，自動入棧。通常稱它們為實體值。
註：顯示時省略中括號包圍的附參值。

```go
指令值  指令字      說明
-----------------------------------------------------------
0       NIL         空值
        // 卽Go語言中的 nil 值。

1       TRUE        眞值（True）
2       FALSE       假值（False）
        // 布爾值，Bool 類型。

3       {}(1)       字節整數（byte/uint8），1字節：[0, 255]
4       {}(2)       小整數（int16），2字節：[-0x8000, 0x7fff]
5       {}(4)       中整數（int32），4字節：[-0x80000000, 0x7fffffff]
6       {}(~)       變長整數，Varint：[-1<<63, 1<<63-1]
        // 整數，Int 類型（int64）。
        // 附參值卽為指令的關聯數據，應當在{}內表達。
        // 從腳本的文本解析為指令序列時，值的大小自動決定指令的值（3-6）。
        // 值存儲采用大端序結構（big-endian unsigned integer）。
        // 例：
        // {12756} 入棧正整數 12756，指令值應該為4（2字節）。
        // 註：
        // 4字節的中整數和變長整數有一些重合，優先采用4字節記錄。
        // 內部運算時可能會統一轉換到int64類型。

7       {}(1+n)     大整數（BigInt），變長字節。
        // 首字節標記實際數據占用的字節數，結構類似後面的 DATA{}(1)。
        // 值存儲采用大端序（big-endian）結構，同上。
        // 註：
        // Go語言中卽為math/big包裏的Int類型。

8       {}(4)       浮點數（4字節，float32）
9       {}(8)       浮點數（8字節，float64）
        // 浮點數，Float 類型。
        // 附參卽為關聯的數據值，表達在{}內。
        // 作為文本顯示時必須包含小數點，支持科學記數法。
        // 從腳本文本解析到指令序列時，值的大小自動決定指令值（8-9）。
        // 例：
        // {3.14159} 入棧浮點數 3.14159，指令值為8。
        // {2.1606e108} 超出float32容量，指令值為9。
        // 註：
        // 內部計算時會統一轉換到 float64 類型。

10      NL          換行符。可用於批量解鎖參數的分隔。
        // 註記：
        // 非直接査找方式切分，數據仍需解析。
        // 因為解析可跳過指令關聯數據的檢査，所以效率也不低。

11      DATA{}(1)   字節序列數據。後續1字節定義再之後的數據長度。
12      DATA{}(2)   字節序列數據。後續2字節定義再之後的數據長度。
        // 序列，Bytes 類型。
        // DATA{46af3fb481837fadbb421727f9959c2d} 入棧16個字節，視為16進製。
        // DATA{}(2) 指令最大可支持64kb數據，但目前上限為8kb。
        // 從腳本文本串解析為指令序列時，值的長度自動決定指令值（11-12）。
        // 註：
        // 類似Go語言中的[]byte類型，但支持相等比較。

13      TEXT{}(1)   文本數據。1字節定義再之後文本的字節數，UTF-8編碼。
14      TEXT{}(2)   文本數據。與上類似，但由2字節定義長度。
        // 文本，String 類型。
        // 例：
        // TEXT{Hello} 入棧 Hello 文本串。
        // 從腳本文本串解析為指令序列時，文本字節長度自動決定指令值（13-14）。
        // 註：
        // 類似Go語言中的string類型，編碼為UTF-8。

15      ~~~         （繫統保留）
```


### 2. 交互指令

在腳本的執行環境中，存在一個數據的「緩存區」結構，它是與外部發生聯繫的一個中介區域。緩存區的數據為移動邏輯（卽隻有一份），如果發生了讀取，數據就是被取走而不是被復製。對緩存區的寫入和讀取是非阻塞的，不影響腳本執行流。

導出和導入的緩存區是兩個各自獨立的區域，腳本中並不能先導出數據寄存，之後再導入同樣的數據使用。

```go
指令值  指令字      說明
-----------------------------------------------------------
16      INPUT(1)    導入緩存區數據。
        // 返迴：[]interface{}
        // 需配合取值指令使用，否則隻是移除了緩存區條目。
        // 附參：讀取的條目數，0值表示全部（顯示可省略）。
        // 例：
        // @INPUT 導入緩存區全部數據展開到當前位置（實參空間）。
        // $INPUT 導入緩存區全部數據展開添加到當前局部變量域。
        // @INPUT[10] PUSH 將緩存區最後10項數據取出展開後入棧。

17      OUTPUT      導出實參序列到緩存區。
        // 不定參數指令。
        // 例：
        // @TOPS[4] OUTPUT 引用棧頂4項數據導出到緩存區。

18      BUFDUMP     緩存區數據轉儲。外部需註冊迴調以接收轉儲的數據。
        // 說明：
        // 迴調無需在所有的節點中執行，這是一個私有邏輯。
        // 有需求的交易者應該已經在節點機上預作了部署，此可觸發業務邏輯。
        //
        // 轉儲時包含一個前置的狀態值：
        //  0   尚未執行通關指令（PASS, MOVE）
        //  1   通關成功（腳本繼續或正常結束）
        // -1   通關失敗（腳本終止，僅由繫統自動調用）
        //
        // 註記：
        // 繫統內置的默認簽名驗證包含了通關邏輯（PASS）及其狀態。
        // 內置驗證失敗後轉儲的數據為nil（因為緩存區尚未初始化）。

19      ~~~         （繫統保留）

20      PRINT(1)    控製臺打印。多個實參條目用換行分隔。
        // 不定參數指令。
        // 附參：標記打印的字符串格式。
        //      0   => 智能格式，默認
        //      2   => 二進製
        //      8   => 八進製
        //      10  => 十進製
        //      16  => 十六進製
        // 例：
        // @TOPS[3] PRINT[16] 向控製臺打印棧頂3項數據的十六進製表示。
        // @POP PRINT 彈出棧頂項並用智能格式打印（0可省略）。
```


### 3. 棧操作指令

對腳本棧中的條目執行簡單的操作，如：入棧、彈出、引用（待復製）等。通常配合取值指令 `@` 和 `$` 使用（見後）。

```go
指令值  指令字      說明
-----------------------------------------------------------
21      NOP         無操作（占位）。
22      PUSH        數據入棧。
        // 不定參數指令。
        // 把前面的實參序列按順序壓入棧頂。無實參時無效果。
        // 例：
        //      $[0] $[1] $[2] PUSH
        // 引用局部變量域中前3個值為實參，復製壓入棧頂。

23      POP         彈出棧頂條目。
        // 彈出的數據如果沒有前置取值指令，則為丟棄，下同。
        // @POP 取值到當前位置（實參）
        // $POP 取值並添加到局部變量域
24      POPS(1)     彈出棧頂端多個條目。
        // 返迴：[]interface{}
        // 一次性取出，保持棧內順序。
        // 附參：彈出的條目數，0值表示全部（顯示可省略）。
        // 例：
        // $POPS[10] 彈出棧頂10個條目添加到局部變量域。
        // POPS[10]  彈出棧頂端10個條目並丟棄。
        // POPS      彈出棧內全部條目（同 POPS[0])並丟棄.

// 以下為目標引用。
// 需前置取值指令（@, $）使用，否則無效果。
25      TOP         引用棧頂項。
        // 引用棧頂項但並不取出。下同。
        // 例：
        // @TOP PUSH 與 DUP 效果相同。
26      TOPS(1)     引用棧頂端多個條目，順序保持。
        // 返迴：[]interface{}
        // 附參：引用條目數，0值表示全部條目（顯示可省略）。
        // 例：
        // TOPS[1] 與 TOP 效果相同。

27      PEEK(1)     引用棧內任意條目，從棧頂開始逆向計數。
        // 附參：目標條目下標，從0開始計數。
        // 例：PEEK[0] 與 TOP 相同。

28      INDEX(1)    引用棧內任意條目，從棧底開始計數。
        // 附參：目標條目下標，從0開始計數。

29      GETS(1,1)   引用棧內多個條目，從棧底開始計數。
        // 返迴：[]interface{}
        // 附參1：起始下標，棧底為0值。
        // 附參2：引用條目數，0值表示後續全部。
        // 註：
        // 附參視為uint8類型，最多支持255條數據。
```


### 4. 取值指令

截取後續指令的返迴值到當前位置作為實參（@）或添加到局部變量域中（$），返迴值不再入棧。`@` 主要用於不定參數指令的實參設置，`$` 則為程序編碼提供一個有限的變量空間（當前域）。這是兩個特殊的符號指令，需配合有返迴值的指令使用。

```go
指令值  指令字      說明
-----------------------------------------------------------
30      @           值固化。截取後續指令的返迴值到當前位置（實參化）。
        // 固化的值可以連續多個，它們的順序與返迴值的順序一致。
        // 如果返迴值是一個 []interface{} 類型，則會被自動展開。
        // 例1：
        //      @GETS[0,2] PRINT[16]
        // 獲取棧底2個值展開為2個實參，向控製臺打印它們的16進製表示。
        //
        // 例2：
        //      CASE{ @POP + @{99}, {1000} }
        // 測試表達式：
        //      @POP + @{99}
        //      取出棧頂值 + 99，結果值自動入棧。
        // 分支語句：
        //      {1000}
        //      取結果值與 SWITCH 標的值相等比較，是則將值 1000 入棧。
        //      註：結果值首先從實參空間査詢，遵循二階獲取。
        //
        // 例3：
        //      CASE{ {100} @EQUAL, ENV{FromHeight} }
        // 測試表達式：
        //      {100} @EQUAL
        //      值100入棧，取棧頂2個值相等比較，結果當前保持。
        // 分支語句：
        //      ENV{FromHeight}
        //      取結果值與 SWITCH 標的值相等比較，是則將所屬區塊高度值入棧。

31      $           變量賦值。取後續指令的返迴值添加到當前變量域中。
        // 因為取值能力的限製（見下），最多可添加128個值，多出視為錯誤。
        // 變量域中的值無法脩改，它們是隻添加的。這是有意的抑製復雜性設計。
        // 例：
        // $TOP 引用棧頂項復製添加到當前局部變量域中。
        // $INDEX[0] 引用棧底項復製添加到當前的局部變量域中。
        // $MO_MATH{Abs} 取棧頂值計算絕對值後，存入當前局部變量域中。
        //
        // 說明：
        // 同上 @ 指令，[]interface{} 類型的返迴值會被自動展開。
        // 腳本頂層擁有一個局部性質的「腳本域」，用於腳本頂層的局部變量存儲。

32      ${}(1)      當前變量域取值。按添加時的順序引用變量的值作為實參。
        // 附參：局部變量域成員下標，值範圍：[0-127]。
        // 這是 $ 賦值指令的逆指令，目標由位置下標決定。
        // 與 @ 指令的效果一樣，提取的值也進入了當前的實參空間。
        // 例：
        // $[0]  取第1個添加進局部變量域中的值。
        // $[-1] 取最後一個添加到局部變量的值，可能更常用。
        // 註：負下標提供一種友好定位方式，指逆向計數。
        //
        // 作為補充，可以用$賦值時的名稱來取迴值，名稱用花括號包圍。
        // 因為存在名稱重復覆蓋的問題，這隻是一種輔助語法，隻宜「就近」使用。
        // 例：
        // $POP   取棧頂值添加到局部變量域。
        // ${POP} 從局部變量域取迴上面添加的值。
        // ${MO_MATH.Abs} 模塊指令與其成員可用句點連接。
        // 註意：
        // 重復名稱的值添加沒有問題，但取值則為同名的最後一次添加值。
        //
        // 註記：
        // 這裏的表達都隻是一種視覺顯示，實際的指令附參是一個數值。
        // 指令序列的預解析（構造交易時）會將名稱或負值對應到實際的下標值。
```


### 5. 結果指令

對腳本的執行實施決定性控製，卽產生某種結果。除非已明確「通過」，否則失敗退出或簡單結束都不會轉移信用。
也卽：如果想要腳本代表的信用轉移出去，必須執行通關指令（PASS, MOVE）。

> **註：**<br>
> 通關指令類似於「安檢」的邏輯。關口可能有多個，一個無效則全部失敗。<br>

```go
指令值  指令字      說明
----------------------------------------------------------
33      PASS        檢査通過。實參值為TRUE時通過。
34      MOVE        轉移通過。實參值為TRUE時，關聯憑信檢査通過（屬性相關）。
        // 說明：
        // 以上兩個指令用於通關檢査，也稱為通關指令。
        // 實參為 FALSE 或關聯檢査不合格時導致失敗，腳本終止。
        // 註：
        // MOVE 專用於憑信，會觸發關聯檢査，應當用在末端（邏輯上）。
        // PASS 通用，用於憑信腳本中隻是簡單通關檢査。
        // 通關失敗時會自動調用 BUFDUMP 指令，向繫統外傳遞信息。
        // 註意：
        // 憑信中隻有最後使用了 MOVE 指令，才會決定是否成功。

35      GOTO(4,32,2) 執行流跳轉。跳轉到某區塊某交易的某序號的輸出腳本。
        // 實參數據被傳遞到新腳本作為解鎖參數入棧。
        // 新腳本繼承當前的腳本環境，但有其自身的執行域（棧、緩存區、全局空間）。
        // 新腳本的執行結果卽為當前腳本的結果（如PASS狀態）。
        // 附參：(Height[4], TxID[32], OutIndex[2])
        // 含義：(理想塊高度，交易ID，輸出索引)
        // 例：
        //      @TOPS[10] GOTO[1000,0xcca750...,1]
        // 引用棧頂10項數據，跳轉並傳遞到第1000號理想塊的目標交易的第2項輸出。
        //
        // 註：
        // 考慮安全和性能，跳轉指令累計使用不可超過4次。
        // 累計使用指實際執行次數，而非代碼中指令的數量（如分支中重復）。

36      JUMP        執行流跳轉。
        // 功能類似GOTO，但由實參提供跳轉目標（同GOTO附參）。
        // 跳轉到的目標腳本保持當前腳本的環境和執行域，執行後自動返迴。
        // 例：
        //     JUMP FN_HASH256 DATA{...} EQUAL PASS
        // 取棧頂3項目標定位跳轉，返迴後取棧頂值計算哈希摘要...。
        // 目標腳本可能往當前棧中添加了哈希源數據，因此返迴後驗證。
        //
        // 註：
        // 該指令類似於第三方腳本片段嵌入的邏輯。
        // 因為腳本的執行域也共享，所以實際上可通過全局空間傳遞信息。
        // 這是一個跳轉指令，一個腳本中連續使用不得超過4次。

37      ~~~         （繫統保留）

38      RETURN      結束腳本的執行並向繫統返迴實參。
        // 不定參數指令。
        // 繫統對返迴值目前僅僅是向控製臺簡單打印。
39      EXIT        結束腳本的執行。
        // return作用類似但不返迴任何值。
```


### 6. 運算指令

有兩種形式的運算指令：
1. 符號指令。操作數在符號兩側的常見形式，如：`{100} + {10}`。
2. 命名指令。普通指令，操作數作為實參在指令之前，如：`{100} {10} ADD`。

命名指令就像普通指令一樣執行，結果自動入棧。符號指令要復雜一些，它們用於構造表達式。


#### 表達式

表達式是一個完整的運算單元，內部的操作數隻能是值指令和簡單取值的指令（如 `@INDEX[n]、$[n]、ENV{n}、VAR[n]`），而無法包含需要從實參空間取實參的函數類指令（如 `MUL、BOTH` 等）。作為必要的特性，表達式內的值指令不再自動入棧（如：`{100}` 與 `@{100}` 同）。

表達式遵循正常的視覺順序邏輯，它無法與需要從實參空間取值的命名指令混用，因為沒有簡單的辦法知道當前實參是用於前面的符號指令還是後面的命名指令，所以這是一種**嚴格受限的表達式**邏輯。如果表達式中需要命名指令的值，應當提前執行該指令讓結果入棧，然後在表達式中簡單取值。

表達式可以是復雜的（包含子表達式），整個表達式運算結束後，結果自動入棧。


#### 指令清單

```go
指令值  指令字      說明
-----------------------------------------------------------
40      ()(1)       表達式封裝或優先級分組。
        // 附參：包含的表達式長度，顯示省略。
        // 頂層表達式的結果自動入棧，內部子表達式結果直接參與運算。
        // 例1：
        //      ( @INDEX[0] + @{100} )
        //      ( @INDEX[0] + {100} )
        // 創建一個表達式：引用棧底值加100。計算結果自動入棧。
        // 值{100}前的取值指令@可選，通常不再需要（表達式內取消了自動入棧）。
        //
        // 例2：
        //      ( {1.5} * (@INDEX[0] + {100}) )
        // 創建了一個復雜表達式：內部包含了加法的優先級分組（子表達式）。
        // 最外圍的初始括號是必需的，它創建了頂層表達式實體。
        //
        // 註記：
        // 如果表達式中包含浮點數，整數會自動轉換為float64計算。結果為浮點數。

41      ~~~         （繫統保留）
        // 未來可能用於賦值操作或其它。

// 符號指令（雙操作數）
42      *           乘
43      +           加
44      -           減
45      %           模（正負號跟隨左操作數，同go語言）
46      /           除

47      <<          左移位
48      >>          右移位
49      &           位與：AND
50      |           位或：OR
51      ^           位異或（XOR）
        // 實參在指令前後兩側，可構造復雜表達式。
        // 例：
        //      ( {100} + @INDTX[0] * {1.5} / {2} )
        // 表達式內存在符號指令本身的優先級。優先級邏輯同go語言的設計。
        // 最後運算結果為浮點數（運行時float64）自動入棧。

// 命名指令（雙操作數）
52      MUL         乘：MUL(x, y) => x*y
53      ADD         加：ADD(x, y) => x+y
54      SUB         減：SUB(x, y) => x-y
55      MOD         模：MOD(x, y) => x%y
56      DIV         除：DIV(x, y) => x/y
        // 實參：有序的兩個操作數。
        // 例1：
        //      {32} {100} MUL
        // 入棧值32、入棧值100，取棧頂2項值計算乘法，結果入棧。
        //
        // 例2：
        //      @POP @{100} MUL
        // 彈出棧頂值為實參，值100為當前實參，相乘後結果入棧。
        // 註意：
        // @POP {100} MUL   錯誤。{100}的入棧不需要前置實參（參數錯誤）。
        // {100} @POP MUL   錯誤。{100}先入棧後被取值，單實參乘法運算無效。
        // 更好的：
        // ... {100} MUL 無需預先 POP 彈出。

// 命名指令（單操作數）
57      NEG         取負：-x
58      NOT         取反：!v （僅適用Bool類型）
        // 實參：目標操作數。
        // 例：
        //      @INDEX[0] PUSH NEG
        // 1. 取棧底值壓入棧頂。
        // 2. 取棧頂值轉為負值後入棧。

59      DUP         復製：返迴2個值
        // 適用任意數據類型，但主要用於Bytes和String。
        // 例：
        //      <pubKey> DUP FN_HASH256 ...
        // 1. 公鈅入棧。
        // 2. 取出棧頂公鈅復製後雙雙入棧。
        // 3. 取出棧頂值計算哈希摘要（32字節）入棧。
```

**註意：**
> `ADD, +` 指令對於字符串（String）和序列（Bytes）是連接操作。


### 7. 比較指令

設計為類似函數的操作方式，指令需要的兩個操作數由所取實參定義，結果自動入棧。

```go
指令值  指令字      說明
-----------------------------------------------------------
60      EQUAL       相等：==
61      NEQUAL      不相等：!=
62      LT          小於：<
63      LTE         小於等於：<=
64      GT          大於：>
65      GTE         大於等於：>=
        // 說明：
        // 上面的操作符隻是一個類比表達，並不作為指令字名稱。
```

**說明：**
> 序列（Bytes）類型的大小比較按從左到右的逐字節進行。<br>

### 9. 流程指令

不像普通編程語言中可以用花括號限定語句塊的範圍，或者使用END語句結束語法塊，本設計中使用限定性很強的長度標定方式，卽：表達式或語句塊的長度由附屬參數準確指定。

附參長度大多為1-2字節，繫統假設了不同指令的適用長度。因為長度值的使用，結束指令就不再需要了。
顯示時中括號包圍的附參值省略。

> **註記：**
> 由附屬參數指定表達式或語句塊的長度，是一種嚴格限製的設計。
> 這不隻是限製了代碼塊本身，隱含地，它對代碼嵌套的層級也有效果（上層塊受限），這可能促進代碼邏輯的扁平化。

```go
指令值  指令字      說明
-----------------------------------------------------------
71      IF{}(1)     眞值執行。
        // 實參值為TRUE時，執行語句塊指令序列。
        // 附參：IF塊指令序列長度（<256字節）。
        // 例：
        //      ... EQUAL IF{ @TEXT{Hello} PRINT }
        // 解：
        // EQUAL      取出棧頂兩個值比較是否相等，結果入棧。
        // IF{...}    取出棧頂值如果為TRUE，執行語句塊內指令序列。
        // @TEXT{...} 取花括號內的 Hello 為實參值。
        // PRINT      向控製臺打印（智能格式）。
72      ELSE{}(1)   IF不滿足時執行。
        // 與IF配對使用，參數同上。

73      SWITCH{}(2) 分支選擇區。
        // 取實參為標的作為後續CASE比較值。
        // 附參：區域長度，2字節定義。
        // 提示：
        // 設標的值為TRUE，可模擬if/elseif/else的邏輯。
74      CASE{}(1,1) 條件分支。
        // 測試表達式結果與SWITCH標的值作相等比較，匹配則執行分支。
        // 附參1：測試表達式長度。
        // 附參2：分支語句塊長度。
        // 註：
        // 每個CASE自然結束（無需break），也無貫穿邏輯。
        // 測試表達式的結果無需當前保持，繫統取結果時依然為二階査找。
75      DEFAULT{}(1) 默認分支。
        // 其它分支不可達時執行，通常在所有CASE塊之後，可選。
        // 附參：分支語句塊長度。

76      EACH{}(1)   迭代循環。
        // 不定參數指令。對實參序列迭代處理。
        // 附參：循環塊長度。
        // 當前域變量：
        // - $[0]  // 當前條目數據 ${Value}
        // - $[1]  // 當前循環下標，從0開始 ${Index}
        // - $[2]  // 迭代條目數 ${Total}
        //
        // 註記：
        // 1字節的長度主要為限製負載，但也可促進邏輯的分解，便於審計。
77      BREAK       退出循環或SWITCH。

78-79   （繫統保留）
        // 未來可能用於LOOP類指令。
```


### 10. 邏輯指令

實參需為布爾值類型，結果為布爾值，自動入棧。
註：邏輯非指令（NOT）被劃歸在前面運算指令的單操作數部分。

```go
指令值  指令字      說明
-----------------------------------------------------------
80      BOTH        兩個都為眞（&&）
        // 邏輯AND，取2個實參測試True值。
        // 例：
        // BOTH PASS 取棧頂兩個值，都為眞時通過。

81      EVERY       全部實參都為眞（類 &&）
        // 不定參數指令。
        // 例：
        // @TOPS[3] EVERY  引用棧頂3個值，測試是否都為True。
        // @TOP $[1] EVERY 引用棧頂值和第2個局部變量的值，測試是否都為True。

82      EITHER      兩個中任一為眞（||）
        // 邏輯OR，取2個實參測試True值。
        // 例：
        // EITHER PASS 取棧頂兩個值，任一為眞卽通過。

83      SOME(1)     全部實參部分為眞（||, &&）
        // 不定參數指令。
        // 附參：為眞的最低數量，默認值1。
        // 實參的個數與附參值對比：
        // - |附參值| >= 實參個數：  全為眞，類似EVERY
        // - |附參值| < 實參個數：   部分為眞
        // 如：
        // - SOME[2]：  2個以上為眞（含2個）
        // - SOME[1]：  1個以上為眞，默認
        // - SOME[0]：  無條件放行（可能無意義）
        // 例：
        // @TOP $[0] SOME 引用棧頂值和第1個局部變量的值，任一為True卽可。
        // @POPS[3] SOME[2] 取出棧頂3個值，測試是否有2個以上為True。
```


### 11. 環境指令

腳本執行時，除了有一個頂層的繫統環境之外，還有一個與當前交易和腳本校驗相關的「腳本環境」：3個隻讀域（`ENV、IN、OUT`）。

另外，腳本執行時還有2個全局空間：`TIME、VAR`（執行域），以及和多重簽名構造有關的 `MULSIG` 信息環境。

```go
指令值  指令字      說明
-----------------------------------------------------------
84      EVN{}(1)    取腳本環境裏的值。
        // 附參：目標名稱的標識值（碼值），花括號內顯示名稱。
        // 以下定義都是針對當前正在驗證的交易，驗證輸入腳本。
        // 參考：
        // - Height      理想塊高度
        // - FromHeight  源交易所在區塊高度
        // - RealHeight  打包進的區塊高度
        // - FromID      來源交易ID
        // - TxID        當前交易ID（在驗證）
        // - Timestamp   當前交易時間戳
        // - InTotal     輸入項總數
        // - InAmount    輸入總金額
        // - OutTotal    輸出項總數
        // - OutAmount   輸出總金額
        // - FromGOTO    是否從GOTO指令跳轉過來（用於管理者交易）
        // - FromJUMP    是否從JUMP指令跳轉過來（同上）
        // - FromSource  跳轉來源指令序列（截止跳轉位置，不含跳轉指令本身）
        // - JumpCount   跳轉計數（含GOTO和JUMP指令）
        // 例：
        // EVN{RealHeight} 獲取交易所在區塊高度壓入棧中

85      IN{}(1)     取當前鎖定腳本的某個成員值。
        // 因為該腳本是在後續交易作為輸入時才執行驗證，所以命名為 IN。
        // 它實際上是對當前輸出腳本自身的目標進行取值。
        //
        // 附參：成員標識，花括號內名稱表達。
        // 成員參考：
        // - Amount      幣金數量
        // - Receiver    接收者（適用幣金和憑信）
        // - Description 憑信描述
        // - Count       憑信轉移計數
        // - Title       證據標題
        // - Content     證據內容
        // - Attachment  附件（ID）
        // - Script      腳本自身
        // - Datum       輸出條目的完整數據序列
        // - Index       在輸出集中的序位
        // 例：
        // IN{Amount} 取當前輸出項的幣金數量。
        //
        // 註：
        // Script 指當前的腳本段，如管理者腳本中引用就指該管理者腳本自身。
        // IN 的含義並不是指當前交易的輸入項，這些輸入項是自由的，也不應當被約束。

86      OUT{}(1)    取後階（正在驗證）交易的某個輸出項的成員值。
        // 實參：目標輸出項的序位（起始值0）。
        // 附參：輸出項中的成員的標識，名稱同 IN 指令。
        // 說明：
        // 成員參考同 IN 指令，略。
        // 卽當前交易的輸出（鎖定）定義可影響使用該輸出的後續交易，比如限定接收者。
        // 例：
        // $[0] OUT{Receiver} 取首個局部變量值為輸出項下標，取該輸出項的接收者。
        // @INDEX[0] OUT{Description} 取棧底值為輸出項下標，取該輸出項的描述數據。
        //
        // 註記：
        // 目標輸出項序位設計為實參，是因為後階交易的輸出應當由用戶自己掌控。

87      SIGNS(1)    簽名數量檢査，返迴布爾值或簽名數量。
        // 合法簽名的數量大於等於目標值時返迴True，否則為False。
        // 附參：目標簽名數。
        // 註：0為特殊值，表示返迴驗證合法的簽名數量。
        // 例：
        // SIGNS[2] 傳入的簽名中有2位以上驗證合法，True值入棧，否則False入棧。
        // SIGNS[0] 傳入的簽名中合法的簽名數量本身入棧。
        // SIGNS[_] 同上。數字0並不是一個恰當的表意，下劃綫為占位符。
        // SIGNS[]  同上。省略簡化但視覺友好（保留括號）。
        //
        // 傳統的多簽驗證（2/n）實際上可如下：
        //      SIGNS[2] PASS
        // 或者
        //      SIGNS[] {2} GTE PASS
        // 後一種語法中，可以支持從其它地方獲取對比目標值{2}（如局部變量）。

88      MULSIG(1)   多重簽名序位確認，返迴布爾值。
        // 目標序位的簽名合法時返迴True，否則返迴False。
        // 附參：目標序位。卽簽名者序號，從0開始。
        // 例：
        // MULSIG[2] 確認多重簽名哈希列表中第3序位者參與簽名。
        //
        // 也可以模擬2/3多重簽名驗證：
        //      MULSIG[0] MULSIG[1] MULSIG[2] @POPS[3] SOME[2] PASS
        // 不過這樣的指令序列太累贅了。

89      TIME{}(1)   取全局時間對象的目標屬性值。
        // 該對象存在於腳本的全局域內，花括號內為目標的標準名稱。
        // 附參：定義取值的目標屬性。
        // 屬性：
        // - Stamp      時間戳（微秒）
        // - Year       年數（4位）
        // - Month      月次/年（1-12）
        // - YearDay    日次/年（1-365|366）
        // - Day        日次/月（1-31）
        // - WeekDay    日次/週（0-6）
        // - Hour       時數/日（0-23）
        // - Minute     分鈡數/時（0-59）
        // - Second     秒數/分鈡（0-59）
        // 例：
        // TIME{Stamp} 獲取時間對象的時間戳值入棧。

90      VAR(1)      取全局變量的值。
        // 全局變量存在於腳本的全局域內，最多支持256個變量（0-255）。
        // 附參：目標變量位置下標。
        // 例：
        // @VAR[0] 取全局變量裏首個變量值到當前實參空間。
        // VAR[10] 取全局變量裏第11號位置的值入棧。
        // 註：
        // 全局變量由 SYS_VAR 指令賦值。
        // JUMP指令跳轉的腳本共享同樣的全局空間，彼此會相互影響。
```

> **附：**<br>
> 信息分層結構：`繫統環境（SYS_ENV） > 腳本環境 > 執行域 > MULSIG > 腳本域 > 局部域`。


### 12. 模式指令

指令序列的模式匹配，主要用於憑信輸出腳本中驗證輸入數據，或幣金腳本中預執行測試（判斷是否符合預定的代碼邏輯）。

比如鏈間兌換，需要確認對方的預支付是否完整有效，然後才能啓動自己的實際支付（卽在實際執行前得到某件事的確認）。一個邏輯完整的模式還可以讓中間件執行匹配測試並提取信息，實現無人工審核的自動化流程。

又如在選舉子鏈中，投票權的憑信花費需要包含對投票內容的格式驗證（以免選票無效），此時的模式指令直接包含在輸出腳本裏。

模型與目標腳本的匹配是完整匹配，卽濾除掉可忽略指令後，模式序列與目標指令序列需同時結束。如果模式匹配失敗，繫統會以類似 `PASS` 失敗的方式處理。


> **提示：**<br>
> 一個模式序列通常稱為一個模型，一個擁有廣泛共識的模型通常會被發布在一筆管理者交易中供分享。

```go
指令值  指令字          說明
-----------------------------------------------------------
91      MODEL<>{}(1,2)  創建模式匹配區。
        // 開啓一個解析器子環境，對實參執行模式匹配測試。
        // 實參： 待測試的指令序列，Bytes類型。
        // 附參1：被忽略指令清單長度，<>內顯示逗號分隔的指令列表。
        // 附參2：模式區長度，{}內包含了模式匹配序列。
        // 返迴： #指令收集的值集。
        // 註意：
        // - 忽略指令清單長度指占用的字節數，而不是指令的個數。
        // - 不便用名稱表達的忽略指令（如值指令），可用指令值（數字）書寫。
        // - 忽略是指整個指令，包含附參、關聯數據和前置的取值實參。
        // 例：
        // 目標腳本：
        // DUP FN_HASH160 DATA{46af3fb4...} @TOP PRINT EQUAL PASS FN_CHECKSIG PASS
        // 註：添加了一條無用的打印公鈅哈希的指令。
        // 匹配模式：
        // MODEL<PRINT, INPUT, OUTPUT, BUFDUMP> {
        //      DUP FN_HASH160 DATA[20]{?} #[4] EQUAL PASS FN_CHECKSIG PASS
        // }
        // @POP OUTPUT BUFDUMP
        // 說明：
        // #指令提取了公鈅哈希，打印指令被忽略（連同 @TOP 取值實參）。
        // 模式指令執行完後，提取的公鈅哈希返迴並自動入棧。
        // @POP OUTPUT BUFDUMP 導出公鈅哈希並觸發註冊的（如果有）外部應用。
        //
        // 附註：
        // - 如果一個未知腳本與該模式匹配，就說明其支付邏輯正常，卽便公鈅哈希可能是錯的。
        // - 可忽略指令在源腳本中可自由出現。它們也可出現在模式中，按普通的指令正常匹配。

// 以下為局部指令，僅在MODEL之內有效。
92      #(1)        提取前一個指令匹配目標的相關值。
        // 提取的值會加入一個隊列，它們將作為MODEL指令的返迴值。
        // 附參：取值範圍類型，下面為二進製。
        // 001 - 指令本身。(1)
        // 010 - 單獨附參。(2)
        // 100 - 關聯數據。(4)
        // 011 - 指令和附參。(3)
        // 101 - 指令和關聯數據。(5)
        // 110 - 附參和關聯數據。(6)
        // 111 - 全部值。包含指令本身、附參和關聯數據。(7)
        // 例：
        // MODEL<...> {
        //      DATA[32]{?} #4 提取前面DATA匹配目標的數據部分。
        // }
        // 註：
        // #[4] 是正常的附參顯示方式，也可簡寫為 #4。

93      ~~~         （繫統保留）

94      ?(1)        指令通配符。
        // 匹配當前位置的目標指令為任意指令，或用後一個指令來和目標匹配。
        // 附參：任意值標識。
        // 0 - 目標為任意指令（含附參和關聯數據）。默認。
        //
        // 用後一個指令匹配。附參標識（二進製）：
        // 0000 0001 - 關聯數據任意。(1)
        // 1000 0000 - 第1個附參任意，其它附參嚴格匹配。(128)
        // 0100 0000 - 第2個附參任意，其它附參嚴格匹配。(64)
        // 0010 0000 - 第3個附參任意，其它附參嚴格匹配。(32)
        // 0001 0000 - 第4個附參任意，其它附參嚴格匹配。(16)
        // 0000 1000 - 第5個附參任意，其它附參嚴格匹配。(8)
        // 0000 0100 - 第6個附參任意，其它附參嚴格匹配。(4)
        // 0000 0010 - 第7個附參任意，其它附參嚴格匹配。(2)
        // 註：
        // 最多支持7個附參的通配標記，多個通配位可同時設置（位或）。
        //
        // 例1：
        // ?0           目標位置為任意一個指令。?0 是 ?[0] 的簡寫。
        // ?            同上。默認的0值可省略書寫。
        // ?1 DATA[32]  匹配32字節任意序列。
        // DATA[32]{?}  同上。一種人性化的表達，花括號內表示關聯數，任意。
        //              註意解析文本的時候先識別後面的 ? 字符卽可。
        //
        // 例2：
        // ?129 DATA    匹配任意長度任意字節序列（首尾兩位置位）。
        // DATA[?]{?}   同上。[?]表示附參通配。另：若有多個附參可用逗號分隔的序列來表示。
        // ?129 TEXT    匹配任意長度任意文本串（TEXT{} 的附參和值都為任意）。
        // TEXT[?]{?}   同上，更友好。
        // 註記：
        // DATA{} 和 TEXT{} 實際上各有兩個指令（指令值 11,12 和 13,14），
        // 如果此處要求匹配的附參為任意，則這兩個指令本身在模式中的附參值就沒有意義了。
        // 此時 ? 之後跟隨的 DATA{} 指令采用11號就足夠了（更節省空間）。
        //
        // 例3：
        // IF[60]{?}    匹配if指令，關聯的語句塊長60字節，內容不限。
        // IF[?]{?}     匹配一個if指令，長度和內容都任意。
        // IF{...}      匹配一個if指令，內容進階匹配。附參值不參與匹配。
        // IF[?]{...}   非法。無法明確內容長度。
        //
        // 例4：
        // ?1{}     匹配任意整數。此處用指令值3，附參值無意義。
        // {?}      同上。通常用此表達。
        // {?.}     匹配任意浮點數，此處用指令值8，附參值無意義。
        //
        // 例5：
        // GOTO[1000,?,0] 或 ?64 GOTO
        // 匹配GOTO指令，目標區塊1000，交易ID任意，第1項輸出（輸出集內0下標）。
        //
        // GOTO[1000,?,?] 或 ?96 GOTO
        // 匹配GOTO指令，目標區塊1000，交易ID和輸出項位置任意。
        // 註：
        // 全部附參同為任意時，可簡寫。如：GOTO[?,?,?] 可簡寫為 GOTO[?]。
        //
        // 例6：
        // ?129 @DATA   匹配取值實參。取值指令（@,$）與跟隨指令緊鄰。
        // @DATA[?]{?}  同上。更友好。
        // TEXT[512]{?} 匹配512字節長任意文本。TEXT{}(2) 指令，附參嚴格匹配。
        // @TEXT[?]{?}  匹配取值實參。此為 TEXT{}(1) 指令，附參值無意義。
        // ?128 PRINT   匹配打印指令，打印格式不限。註：打印指令無關聯數據。
        // PRINT[?]     同上。無關聯數據的指令，模式的數據位忽略。

95      /../        正則模式匹配，適用於String類型。
        // 後續指令為TEXT{}，花括號內為 /../ 包圍的正則表達式。
        // 例：
        //      TEXT{/Zip:[0-9]+/}
        // 後續TEXT{}包含的正則表達式與目標位置的TEXT{}值進行匹配。
        // 因為允許長度不同，附參值不會參與比較。匹配目標必須同為字符串類型。
        // 解析：
        // 指令文本解析為實際的指令值序列為（括號示意分隔）：
        //      [93][13](10){Zip:[0-9]+}
        //      [當前指令值][TEXT指令值](附參值){正則表達式}
        // 註：
        // 正則表達式語法請參考 https://golang.org/s/re2syntax

96      !{}(~,~)    整數值範圍匹配。
        // 限定目標值在設定的範圍之內，包括邊界值。
        // 附參1：下限值，變長整數。
        // 附參2：上限值，變長整數。
        // 例：
        // !{0, 1024} 限定目標值：0 ≦ n ≦ 1024。
        // !{0, 127} 匹配的目標是一個 {}(1)字節整數指令（指令值3）。
        // 註：
        // 繫統內部可能會將目標值轉換為 int64 類型後比較。

97      !{}(8,8)    浮點數值範圍匹配。
        // 限定目標值在設定的範圍之內，包括邊界值。
        // 附參1：下限值，8字節表達。
        // 附參2：上限值，8字節表達。
        // 例：
        // !{0, 1.0} 限定目標值大於等於0，小於等於1.0。
        // 註：
        // 兩個範圍值必須至少有一個是浮點數表示。
        // 繫統內部可能會將目標值轉換為 float64 類型後比較。

98      !BOOL       任意布爾值匹配。
        // 布爾值TRUE或FALSE僅能匹配確定的值，這是一個通配。
        // 例：
        // !BOOL 匹配的目標必須是TRUE或FALSE。

99      ...         任意指令段匹配
        // 匹配任意目標指令序列，包括其附參和關聯數據。
        // 非貪婪匹配，後續其它模式指令段與目標指令序列中的首個匹配段匹配。
        // 註：
        // 通配隻限於當前域內，子域是一個整體單元（不會進入）。
        // 若置於模式序列的頭部或末尾，可以實現前端或後端局部匹配。
        // 例：
        // 目標腳本：
        //      TIME{Stamp} {<expireTime>} GT   // 比較時間戳
        //      @TOP PRINT                      // 棧頂值引用打印
        //      IF{                             // 如果為眞
        //          @POP PRINT                  // 彈出棧頂值並打印
        //      }
        //      POP TRUE                        // 彈出棧頂值丟棄，入棧眞值
        //      IF{
        //          TEXT{Hello in IF}           // 必然會到這裏
        //          @TOP PRINT                  //A: 引用棧頂值打印
        //      }
        //      TEXT{done!}                     // 入棧 done!
        //      @TOP PRINT                      //B: 與IF內的A相同
        //      POPS                            // 彈出棧內數據
        // 匹配模型：
        // MODEL<>{
        //      TIME{Stamp} {<expireTime>} GT
        //      ...                             // 1
        //      IF{
        //          ...                         // 2
        //          @TOP PRINT                  // A
        //      }
        //      TEXT{done!}
        //      @TOP PRINT                      // B
        //      POPS                            // 終點嚴格匹配
        // }
        // 1.說明：
        // 通配 @TOP PRINT、第一個 IF{} 和 POP TRUE 等指令。
        // 因為後面的 IF{} TEXT{done!} @TOP PRINT 才完整匹配，找到停止點。
        //
        // 2.說明：
        // 通配 TEXT{Hello in IF} 指令，不會跳過內層A匹配到外面的B位置。
        // 一是因為域限製，二是非貪婪匹配的要求。這符合直覺的理解。
        //
        // 匹配模型：
        // MODEL<>{
        //      TIME{Stamp} {<expireTime>} GT
        //      ...                             // 1
        //      IF[?]{?}
        //      ...                             // 2
        // }
        // 說明：
        // 1處的匹配截止到第一個 IF 指令，卽通配首個「@TOP PRINT」指令段。
        // 2處的匹配截止到目標腳本終點。這是必須的，否則整個模型匹配失敗。
        //
        // 註記：
        // 如果將嵌套的不同層級的域構造成樹，這應該是一個廣度優先的査詢算法。
        // 如果不同域內有模式取值（#），則需要用深度優先的方式匯集。
```


#### 附：模式匹配包含的內容

1. **位置匹配**：濾除掉可以忽略的指令後，位置一一對應。卽：位置記數不包含可忽略指令。
2. **指令匹配**：指令相同卽為匹配。如果指令為值指令，則也隱含了類型匹配邏輯。
3. **附參匹配**：在指令匹配的前提下，附參參與匹配。可前置 `?` 指令包含通配邏輯。
4. **數據匹配**：在指令匹配的前提下，指令關聯的數據參與匹配。可前置 `?` 指令表達通配邏輯。
5. **正則匹配**：`/../` 指令表達正則模式匹配，限於對 TEXT{} 目標數據測試。
6. **範圍限定**：用於整數和浮點數的範圍匹配。註：兩個8字節的附參可涵蓋低值指令。
7. **布爾匹配**：任意布爾值匹配（TRUE 或 FALSE）。

> **註：**<br>
> 模式中流程指令（`IF{}`、`SWITCH{}` 等）在有進階匹配的內容時，其附參不參與匹配。<br>
